Create `src/lib/builder/generator.ts` -- the AI-powered website generation engine:

1. `generateWebsite(request: GenerationRequest, orgId: string, supabase: SupabaseClient): Promise<{ html: string; css: string; project: WebsiteProject }>`:
   - If `request.template_id` is provided, load the template via `getTemplate()` and use it as a base. Replace template variables with values from the request (business_name, colors, etc.) using string replacement on CSS variables.
   - If no template, generate from scratch using Anthropic SDK:
     - Import Anthropic from '@anthropic-ai/sdk'
     - System prompt: "You are an expert web designer. Generate a complete, responsive, single-page HTML website. The HTML must be self-contained with all styles in a <style> block. Use modern CSS (flexbox, grid, CSS variables). The site must look professional and be mobile-responsive. Use placeholder images from https://placehold.co/. Do NOT use any external CSS frameworks or JavaScript frameworks. Return ONLY the HTML content, no markdown fences."
     - User prompt built from: business_name, industry, description, colors (if provided), page sections requested
     - Use model 'claude-sonnet-4-20250514' for speed (website gen is mostly creative, not reasoning-heavy)
     - Parse the response to extract the HTML content
   - Create a `website_projects` row in Supabase with status 'preview', the generated html_content, and metadata storing the generation request
   - Create a `website_revisions` row with version=1
   - Generate a URL-safe slug from the business_name (lowercase, hyphens, no special chars)
   - Return the generated HTML, CSS, and the created project record

2. `reviseWebsite(projectId: string, instruction: string, orgId: string, supabase: SupabaseClient): Promise<{ html: string; css: string; project: WebsiteProject }>`:
   - Load the existing project from website_projects
   - Load the current HTML content
   - Call Anthropic with system prompt: "You are an expert web designer. You will receive an existing HTML website and a revision instruction. Apply the requested changes to the HTML. Maintain the overall structure and responsiveness. Return ONLY the complete updated HTML, no markdown fences."
   - User message: "Current website HTML:\n```\n{current_html}\n```\n\nRevision request: {instruction}"
   - Update the website_projects row with new html_content
   - Create a new website_revisions row with incremented version and change_summary = instruction
   - Return updated HTML, CSS, and project

3. `getProjectHtml(projectId: string, orgId: string, supabase: SupabaseClient): Promise<string | null>`:
   - Simple helper to load html_content from website_projects for a given project

Create `src/lib/builder/index.ts`:
   - Re-export everything from './types'
   - Re-export everything from './templates'
   - Re-export everything from './generator'
