<objective>
Build procedural memory system — stored workflows that BitBit has learned, with trigger pattern matching.

Purpose: When the user says "email the AWU client," BitBit triggers the stored procedure ("use M365, never Gmail") instead of reasoning from scratch.
Output: procedural_memories table, creation/triggering logic, tests.
</objective>

**Schema from spec:**
procedural_memories: id, org_id, name, trigger_pattern, steps (JSONB[]), success_count, last_used_at, source (observed|explicit|consolidation), is_active

**Creation paths:**
- Explicit: user says "remember how to do X" → agent creates via tool
- Observed: sleep consolidation detects repeated tool sequences (3+) → creates procedure
- Consolidation: promotes from memory_patterns when confidence > 0.75

**Usage:** Proactive recall checks trigger_pattern against current message. If matched, injects steps as "## Known Procedure".
