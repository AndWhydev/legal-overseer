  <action>
Create the 5-stage sleep consolidation pipeline:

```typescript
export async function runSleepConsolidation(
  supabase: SupabaseClient,
  orgId: string,
): Promise<SleepConsolidationReport>
```

**Stage 1: SUMMARIZE** — Per-entity daily digest
- Query entity_nodes that have new event_tuples or edges created today
- For each (batch of 10): call Haiku with entity name + today's events/edges
- Haiku generates 2-3 sentence state summary
- Update entity_nodes.properties.daily_summary and .last_summarized

**Stage 2: RESOLVE CONFLICTS** — Temporal precedence
- Query entity_edges grouped by (source_id, target_id, relation_type) where count > 1 AND all have valid_until IS NULL
- For each conflict: keep the edge with most recent valid_from, set valid_until=now() on older ones
- Log each resolution with reason

**Stage 3: DISCOVER RELATIONSHIPS** — Latent edge detection
- Query event_tuples from today grouped by org_id
- Find entity pairs that co-occur in events (same day, or referenced in same event) but have no direct edge
- For pairs with 2+ co-occurrences: call Haiku to evaluate if meaningful relationship exists
- If Haiku says yes (confidence > 0.7): createEdge with source='consolidation'
- Limit to 5 new discoveries per run

**Stage 4: PRUNE** — Noise reduction
- Query memory_palace_entries created today with confidence < 0.3
- Check if any have entity links (entity_ids array not empty)
- Archive (is_active=false) those with no entity links AND confidence < 0.3
- Don't delete — just archive

**Stage 5: GENERATE MORNING BRIEFING**
- Query entities with: event_tuples where occurred_at within next 48h (deadlines)
- Query entities with: edges where relation_type contains 'blocked' and valid_until IS NULL
- Query: recent discoveries from Stage 3
- Query: approval_queue items pending
- Compile into structured briefing object:
  ```typescript
  {
    urgent: [{entity, reason, deadline}],
    followUps: [{entity, reason, daysSince}],
    discoveries: [{entity1, entity2, relationship}],
    pendingApprovals: number,
    generatedAt: string,
  }
  ```
- Store in organisations settings: UPDATE organisations SET settings = jsonb_set(settings, '{morning_briefing}', briefingJson)

Each stage is idempotent, wrapped in try/catch, logs progress. If one stage fails, the next runs.

Return SleepConsolidationReport with counts per stage.

All LLM calls use models.fast (Haiku).
