**Anthropic contextual retrieval technique:**
At ingestion time, for each chunk, send chunk + full message to Haiku with prompt:
"Give a short context to situate this chunk: who is speaking, what entity/project is discussed, when. Answer only with the context."
Prepend the ~50-100 token context to the chunk before embedding.
Cost: ~$1/million tokens at ingestion. Zero query-time cost.

**Current chunker flow:**
chunkText(content, metadata) → creates TextChunk[] with metadata prepend (From/Subject/Date/Channel)
embedding-service.ts calls chunkText, then embeds via Voyage, upserts to Pinecone.

**New flow:**
chunkText → contextualizeChunks (Haiku enrichment) → embed → upsert
</context>

```typescript
  chunks: TextChunk[],

): Promise<TextChunk[]>
```

Implementation:
1. For each chunk, call Haiku with:
   ```
   <message>
   From: {metadata.sender}
   Subject: {metadata.subject}  
   Channel: {metadata.channel}
   
   {fullMessageText}
   </message>
   
   <chunk>{chunk.text}</chunk>
   
   Give a short context (1-2 sentences) to situate this chunk within the message.
   Include: who is speaking/writing, what entity or project is discussed, approximate date context.
   Answer only with the context, nothing else.
   ```

2. Prepend the context (50-100 tokens) to chunk.text:
   `contextPrefix + "\n\n" + chunk.text`

3. Process chunks in parallel (Promise.all) for speed. Batch if >10 chunks.

4. If Haiku call fails for a chunk, keep the original chunk unchanged (graceful degradation).

5. Skip contextualization for very short chunks (<50 chars) — not worth the API call.

Use the AI provider pattern from the codebase — check src/lib/ai/provider.ts for how to call Haiku. Use generateText (not generateObject — we just want a string).

Export a config constant: CONTEXTUALIZE_ENABLED = process.env.CONTEXTUALIZE_CHUNKS !== 'false' (enabled by default, can be disabled via env var for cost control).
