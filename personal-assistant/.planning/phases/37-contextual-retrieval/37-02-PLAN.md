  <action>
Create a script that queues existing channel_messages for re-embedding:

1. Query channel_messages that have pinecone_ids but were embedded before today (created_at < today)
2. For each message (in batches of 50):
   - Insert into embedding_jobs with status='pending' and metadata flag: { recontextualize: true }
   - The existing process-embeddings cron will pick these up
   - Since embedding-service.ts now calls contextualizeChunks, the re-embedded chunks will get context prefixes
3. Log progress: "Queued X messages for re-embedding (batch Y/Z)"
4. Add a --limit flag (default 500) to control batch size per run
5. Add a --dry-run flag to preview without queueing

Run via: `npx tsx scripts/backfill-contextual-embeddings.ts --limit 100`

Make it idempotent: check if an embedding_job already exists for each message (by message_id) before queueing.

Note: This is a gradual backfill. Run with --limit 100 repeatedly, or let the cron process them over days. No need to re-embed everything at once.
  </action>
