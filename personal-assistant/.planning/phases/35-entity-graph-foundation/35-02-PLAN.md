<feature>
  <name>Entity Extraction Pipeline</name>
  <files>src/lib/knowledge-graph/entity-extractor.ts, src/lib/knowledge-graph/__tests__/entity-extractor.test.ts</files>
  <behavior>
    Given a text message with metadata (sender, channel, timestamp):
    1. Extract entity mentions → [{name, type, aliases}]
    2. Extract relationships → [{source, target, relation_type, properties}]
    3. Extract SVO events → [{subject, verb, object, occurred_at}]
    4. Resolve entities against existing entity_nodes (find or create)
    5. Create edges and event_tuples via graph-queries.ts
    
    Cases:
    - "Steve agreed to the $700 Phase 2 proposal" → 
      entities: [Steve(person), Phase 2(project)]
      events: [(Steve, agreed_to, $700 Phase 2 proposal, timestamp)]
      edges: [(Steve, discussed, Phase 2)]
    - "Maya's website rebuild is blocked waiting for hosting credentials" →
      entities: [Maya(person), website rebuild(project)]
      events: [(website rebuild, blocked_by, hosting credentials, timestamp)]
      edges: [(Maya, owns, website rebuild)]
    - Simple greeting "Hey how's it going" → no entities, no events (empty extraction)
  </behavior>
  <implementation>
    Use Haiku via AI SDK for extraction. Single prompt that returns structured JSON:
    {entities: [...], relationships: [...], events: [...]}
    
    Prompt should include:
    - Message text + metadata (sender, channel, date)
    - List of known entity names (from org's entity_nodes) for resolution
    - Instruction to output JSON only, no explanation
    
    After extraction:
    - For each entity: findOrCreateEntity()
    - For each relationship: createEdge()  
    - For each event: createEventTuple()
    
    The function should be fire-and-forget safe (never throw, log errors).
    Export: extractAndPopulateGraph(supabase, orgId, text, metadata)
  </implementation>
</feature>
