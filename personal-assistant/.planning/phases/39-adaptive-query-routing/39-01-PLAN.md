**From spec — classification:**
Score query on 4 signals:
  +1 entity count > 1
  +1 temporal markers ("last week", "in February", "recently")
  +1 query length > 50 chars
  +1 relational keywords ("about", "between", "related to")
0-1 → SIMPLE, 2 → MODERATE, 3-4 → COMPLEX

**Three paths:**
SIMPLE: BM25 only, no graph, no rerank, topK=5, 500 tokens, ~20ms
MODERATE: hybrid BM25+dense, 1-hop graph, no rerank, topK=10, 1500 tokens, ~80ms
COMPLEX: hybrid + multi-hop graph + Voyage rerank, topK=20, 3000 tokens, ~150ms
</context>

  <name>Adaptive Query Router</name>
  <files>src/lib/rag/query-router.ts, src/lib/rag/__tests__/query-router.test.ts</files>
  <behavior>
    classifyQuery(query: string) returns { complexity: 'simple'|'moderate'|'complex', score: number, signals: string[] }
    getRetrievalConfig(complexity) returns { useGraph, useRerank, topK, tokenBudget, searchModes }

    Cases:
    - "What is Steve's email?" → SIMPLE (1 entity, short, no temporal, no relational)
    - "What did we discuss about pricing last week?" → MODERATE (temporal + relational)
    - "How does Steve's Phase 2 feedback relate to Maya's project delay?" → COMPLEX (2 entities + relational + long)
    - "hey" → SIMPLE (score 0)
    - "" → SIMPLE (score 0)
  </behavior>
  <implementation>
    Pure functions, no DB calls, no LLM — just string analysis.
    
    Temporal markers regex: /\b(last|this|next|yesterday|today|tomorrow|ago|week|month|year|january|february|...|recently|lately)\b/i
    Relational keywords regex: /\b(about|between|related|relate|affect|impact|connection|how does|compared)\b/i
    Entity detection: count words starting with uppercase that aren't sentence starters (simple heuristic)
    
    Integration:
    - search_memory handler: call classifyQuery, use getRetrievalConfig to set topK, decide whether to run graph search
    - proactive recall: defaults to MODERATE config
  </implementation>
</feature>
