Add a new exported function graphSearch():

Input: supabase, orgId, query (string), options (entityIds?, timeRange?, limit?)
Process:
1. If entityIds provided: getNeighborhood for each → collect connected entities + edges
2. Get event_tuples for matched entities within timeRange
3. If query provided and entity_nodes have text_embedding: vectorSearchEntities via match_entity_nodes RPC
4. Format results as array of {content, score, source, metadata}

- For edges: "{source.name} {relation_type} {target.name} (since {valid_from})"
- For events: "{subject.name} {verb} {object_text} ({occurred_at})"
- For vector matches: "{entity.name} ({entity_type}): {properties summary}"

Return type matches existing search result format so it can merge with Pinecone/sparse results.
