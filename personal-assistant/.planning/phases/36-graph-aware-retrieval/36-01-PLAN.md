2. Parallel: graph walk (1-hop edges) + vector search (pgvector) + event lookup (recent 30d)

4. Budget: 1500 tokens
5. Format as <memory-context> block

<feature>
  <name>Graph-Aware Proactive Recall</name>
  <files>src/lib/memory-palace/proactive-recall.ts, src/lib/memory-palace/__tests__/proactive-recall.test.ts</files>
  <behavior>
    Given entity_node IDs (resolved from contact mentions):
    1. Fetches graph neighborhood (1-hop edges + neighbors) for each entity
    2. Fetches recent event_tuples (last 30 days) for each entity
    3. Optionally runs pgvector similarity if a query embedding is available
    4. Scores and ranks results by blended relevance
    5. Formats into a <memory-context> block within 1500-token budget
    6. Falls back gracefully to existing memory_palace_entries if graph is empty
    
    Cases:
    - Entity with graph data → returns neighborhood + events + memories, scored by relevance
    - Entity with no graph data → falls back to confidence-ordered memory_palace_entries (current behavior)
    - No entities mentioned → returns empty (no proactive recall)
    - Multiple entities → merges results, deduplicates, respects 1500 token budget
  </behavior>
  <implementation>
    Rewrite proactive-recall.ts:
    
    1. Update constants: MAX_RECALL_TOKENS = 1500, remove per-entity limits
    2. New main function: graphAwareRecall(supabase, orgId, entityNodeIds, queryEmbedding?)
    3. For each entity_node_id:
       a. getNeighborhood(supabase, orgId, entityId) → edges + neighbors
       b. getEntityEvents(supabase, orgId, entityId, {from: 30daysAgo}) → recent events
       c. Existing: memory_palace_entries query (keep as fallback)
    4. Score each item: 0.4*relevance + 0.3*confidence + 0.2*recency + 0.1*edge_weight
       - relevance: 1.0 for direct edges, 0.7 for 2-hop, 0.5 for memory palace matches
       - confidence: from entity_edges.confidence or memory_palace_entries.confidence
       - recency: exp(-0.01 * age_in_days)
       - edge_weight: 1.0 for direct, 0.5 for indirect
    5. Sort by score DESC, fit within 1500 tokens
    6. Format as XML block with entity context sections
    
    Also update context-assembler.ts:
    - Resolve entity mentions to entity_node IDs (via getEntityByAlias)
    - Pass entity_node IDs to new graphAwareRecall() instead of contact IDs
    - The proactive recall section stays in the same position in the prompt
  </implementation>
</feature>
