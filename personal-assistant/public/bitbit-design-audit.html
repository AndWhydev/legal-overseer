1 — Gold Standard Reference

What "done" looks like

The Chat and Inbox pages both achieve full compliance. They share these patterns: zero Tailwind for visual design, zero shadcn imports, all typography within the 14px/16px constraint, all spacing on the 4px grid, glass-morphic card surfaces with consistent token usage, spring/snap easing on all animations, and monochrome palette with orange as the sole accent color.

Chat Page Patterns

  • CSS class namespace bb-chat__* for all chat-specific styles
  • Message bubbles: 14px body, max-width: min(560px, 74%)
  • User bubble: borderRadius: 18px 18px 4px 18px
  • Glass card: rgba(15, 20, 30, 0.35) + blur(24px) saturate(1.3)
  • Spring easing: cubic-bezier(0.34, 1.56, 0.64, 1)
  • Staggered reveals: 30-60ms per item on whispers, chips, pipeline stages
  • Pipeline stages: status-driven borders (idle/active/done/error)
  • Code blocks: shiki highlighting, 13px mono, copy button, max-height 400px
  • Artifact panel: clamp(400px, 50vw, 700px) wide, spring slide-in
  • Monochrome + orange only. No decorative colors

Inbox Page Patterns

  • Zero Tailwind violations — all inline React.CSSProperties
  • Zero shadcn imports — custom components throughout
  • Data-attribute styling: data-unread, data-important, data-level, data-focused, data-selected
  • Message rows: 48px height, flex with 160px sender col
  • Glass pill filters: borderRadius: 20px, backdrop blur
  • Priority gradient tints via linear-gradient(135deg, ...)
  • Animations: expandIn (200ms snap), collapseOut (180ms ease-out), groupChildIn (220ms snap), AI shine sweep (0.6s)
  • Z-index stack: dropdowns 100, snooze 150, toasts 200, overlay 9999
  • Stagger: calc(var(--stagger) * 35ms) for group children
  • Monochrome palette. Category tags use semantic colors at full opacity

3 — The 12 Semantic Rules

These rules are the semantic contract for BitBit's visual language. Every component, every page, every future feature must satisfy all 12. They are derived from the patterns in the compliant Chat and Inbox pages. When in doubt: fontSize: 16, padding: 16, borderRadius: 12, fontWeight: 400.

01

Two Sizes Only

16px for primary body, titles, card headers, and labels. 14px for secondary text, captions, metadata, and helper text. Nothing else. No 10, 11, 12, 13, 18, 20, 24, or 38px. No clamp(). No text-xs/text-sm.

fontSize: 16 fontSize: 14 fontSize: 12 fontSize: 18 text-xs
02

Two Weights Only

400 (regular) for body text. 500 (medium) for labels, headings, and emphasis. No 300, 600, or 700. No font-bold or font-semibold.

fontWeight: 400 fontWeight: 500 fontWeight: 600 fontWeight: 700 font-bold
03

4px Spacing Grid

Every spacing value — padding, margin, gap — must be divisible by 4. Common: 4, 8, 12, 16, 20, 24, 32, 40. Forbidden: 3, 5, 6, 7, 10, 14, 15, 18, 22.

padding: 16 gap: 12 padding: 10 gap: 6 px-1.5
04

Radius Vocabulary

Five allowed values: 8 (buttons, chips), 12 (small cards), 16 (standard cards), 24 (modals, large surfaces), 9999 (pills). Nothing else. No 0, 3, 4, 6, 10, 14, 18, or 20.

borderRadius: 8 borderRadius: 12 borderRadius: 16 borderRadius: 6 borderRadius: 10 borderRadius: 20
05

Inline Styles for Design

All colors, backgrounds, borders, shadows, and typography must use React.CSSProperties inline styles. Tailwind is permitted for layout only: flex, grid, gap, items-center. Never bg-[...], text-[...], border-[...], rounded-[...].

style={{ background: '...' }} className="bg-amber-500/15" className="text-xs"
06

No shadcn

Do not import Card, CardHeader, CardTitle, CardContent, Button, Badge, Input, Tabs, TabsList, TabsTrigger, TabsContent, Select, or any other shadcn/ui component. Build with styled <div>, <button>, <span>, <input> using the glass card pattern.

<div style={glassCard}> import { Card } from '@/ui/card'
07

Monochrome + Orange

Whites, grays, and dark navy for all surfaces and text. #FF5A1F orange for CTAs and active states only. Status colors (green/yellow/red) are allowed only for semantic indicators and must use muted 12% opacity backgrounds. No blue, purple, cyan, or pink decorative accents.

#FF5A1F accent blue accent purple accent
08

Glass Card Pattern

Every card surface: rgba(15,20,30,0.6) background, blur(20px) saturate(1.2) backdrop, 1px solid rgba(255,255,255,0.03) border, inset 0 1px 0 rgba(255,255,255,0.05) box shadow. Consistent everywhere — no flat cards, no solid backgrounds, no missing backdrop blur.

09

40px Touch Targets

Minimum interactive element height is 40px. Buttons: height: 40 explicit, or padding: '12px 16px' minimum to guarantee it. No 28px, 30px, or 32px buttons. Primary CTAs may go 48px on spacious layouts.

height: 40 padding: '12px 20px' padding: '8px 16px' (=~30px)
10

Noise Over Glow

Wherever there is a glow effect or color gradient, apply an SVG fractalNoise overlay with mix-blend-mode: overlay. This adds texture and prevents the "AI dashboard" flat-glow look. Place as a sibling div inside a position: relative container.

11

Spring Animations

Two easing curves: cubic-bezier(0.34, 1.56, 0.64, 1) for spring (overshoot), cubic-bezier(0.16, 1, 0.3, 1) for snap (fast settle). Stagger: 30-60ms per item. Durations: 150ms fast, 200ms normal, 300ms slow. No linear or ease-in-out.

cubic-bezier(0.34, 1.56, 0.64, 1) ease-in-out linear
12

Data-Attribute Styling

Use data-* attributes for state-driven style changes: data-active, data-selected, data-unread, data-expanded, data-level. This keeps JSX clean and enables CSS-driven state styling rather than complex ternary expressions in inline styles. Reference: inbox-tab.tsx.

Next Steps

This document serves as the semantic contract for all BitBit UI work going forward. Each redesign task in the queue above should reference these 12 rules. The Chat and Inbox pages are the living reference implementations — when in doubt, look at how they solve the same pattern. The goal is not just visual consistency, but a reduction in cognitive load: fewer decisions to make, fewer values to remember, fewer ways to get it wrong.
