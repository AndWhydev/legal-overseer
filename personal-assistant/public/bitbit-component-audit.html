4 — Top 5 Consolidation Opportunities

1. Replace 14 native <select> → GlassDropdown

Files: scenario-planner-ui, team-management, channel-config-drawer, ad-scripts-tab, reports-tab, admin-tab, campaign-create-panel, template-editor-panel, upload-modal, portal-management, portal-requests, and 3 more. All use inconsistent inline styles. The shared GlassDropdown component exists and is proven.

2. Unify 3 toggle implementations → 1 shared Toggle

settings-tab, notification-preferences, and invoice-template-editor each implement their own iOS-style toggle switch with different heights (24px), backgrounds, and focus styles. Consolidate into a single Toggle component.

3. Extend ListRow to 10+ custom implementations

A shared ListRow component exists but is only used in ~3 files. 10+ files implement custom row patterns: leads-list-view, invoice-list, swarm-list, role-activity-feed, team-management, api-key-management, notification-preferences, contacts-tab, attention-view, role-detail-view.

4. Token migration: creator-studio-tab (42 hardcoded rgba)

Worst offender. 42 inline rgba() values that should use C.bgCard, C.bgHover, C.textPrimary etc. from design-tokens.ts. Each replacement is ~1 line but the file needs a full pass.

5. S.card adoption: 6% → target 90%

16 of 17 card patterns still use inline glass styles instead of S.card / S.cardLight / S.cardFlush from design tokens. Each adoption replaces 6-8 lines of inline style with a single spread: style={S.card}.

5 — Execution Plan
Phase 1

Token Migration Sprint

Replace all hardcoded rgba with S/C imports across 29 tabs. Highest impact, mechanical work — ideal for parallel agents.

Phase 2

Component Consolidation

Replace 14 native selects with GlassDropdown. Unify 3 toggles into 1. Extend ListRow to cover 10+ custom row patterns.

Phase 3

Page-by-Page Redesign

Each page gets fresh Opus agent eyes. Independent redesigns where needed. Focus on the C-graded tabs first.

Phase 4

Animation Pass

Apply bb-stagger, bb-lift, spring easing to all lists and card grids. Currently 2/29 tabs have animations.

Phase 5

Light Mode Verification

Test every page in midnight + light + aurora themes. Fix any remaining hardcoded dark-mode values.

Phase 6

Accessibility Pass

WCAG 2.2 audit, keyboard navigation, screen reader testing. Focus on interactive components.
