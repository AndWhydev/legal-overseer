# AWU Agency Policies

All agents operating under the AWU organization must follow these rules.

## Pricing

- Minimum project rate: $500
- Hourly rate: $90-150/hr depending on complexity
- Always include GST in quotes for Australian clients
- Payment terms default: 20% deposit, balance on completion
- Alternative terms: 7-day, 14-day, or 30-day net (per client agreement)
- Never discount more than 15% without Andy's approval
- Always include a "Recommended" tier in multi-tier proposals to anchor pricing
- Indian team benchmark: ~$150/task — our value prop is reliability + quality, not price

## Communication

- Response SLA: 2 hours during business hours (8am-6pm AEST)
- Lead response SLA: 2 minutes (automated acknowledgment)
- Always CC Andy on first contact with new clients
- Use andy@allwebbedup.com.au for client-facing relationship comms
- Use tor@allwebbedup.com.au for technical comms
- Never share internal pricing, cost structures, or margins with clients
- Never reveal AI agent involvement unless explicitly approved

## Client Handling

- Andy is the face — all relationship management flows through him
- Tor handles technical execution and delivery
- Never contact AWU clients directly without Andy's knowledge
- Escalate any client complaint immediately to Andy (WhatsApp)
- If a client asks about timelines, check current workload before committing
- Each client gets a dedicated Asana project from template
- Weekly status updates to active clients (automated by Client Comms agent)

## Confidentiality

- Client work created for AWU belongs to AWU
- Don't share one client's data, strategy, or pricing with another client
- NDA required for projects >$5k
- Contractor agreement terms (Jan 2026) apply to all AWU work
- AWU clients cannot be taken or worked with directly outside AWU

## Quality

- All deliverables reviewed before client handoff
- Screenshots/recordings of completed work for verification team
- Test in staging before going live
- Backup before making changes to live sites
- Code changes via Code Snippets when filesystem is read-only (WordPress hosting)

## Invoicing

- Invoice on completion of each milestone or project phase
- Include bank details on every invoice
- PDF format with AWU branding (or Torkay branding for Tor's direct work)
- Track all invoices in BitBit — never rely on email alone
- CRITICAL: Never send duplicate invoices — always check sent history first
- Chase overdue invoices: Day 1 (friendly), Day 7 (firm), Day 14 (final notice)

## Escalation Rules

- Client complaint → Andy immediately (WhatsApp)
- Legal/threat language → Andy + hold all actions
- Budget overrun >10% → Andy before proceeding
- Scope change requested by client → Andy before committing
- Technical blocker >24 hours → Tor reports to Andy with options
- Payment >30 days overdue → Andy decides next steps
