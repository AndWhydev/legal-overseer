## Critical Integration Issues

### 1. Role Domain Modules Not Registered at Cron Runtime

**Severity:** Critical
**Affects:** All scheduled role execution (finance, comms, sales)

The cron route `role-tick/route.ts` imports `runScheduledRoles` from `role-scheduler`, which imports `role-runtime` and `role-registry`. But no module in the cron execution path imports the domain role files (`finance-role.ts`, `comms-role.ts`, `sales-role.ts`) that call `registerRole()`.

Auto-registration fires only on module import. At runtime, `getRole(roleConfig.role_type)` returns `undefined` for all roles. The role-tick cron fires every 5 minutes and silently skips all role execution.

**Fix:** Add `import '@/lib/roles/finance/finance-role'` (and comms, sales) to `role-tick/route.ts` or `role-scheduler.ts`.

### 2. Intelligence Widgets Call Wrong API Endpoint

**Severity:** Major
**Affects:** Dashboard intelligence display

`IntelligenceWidgets` calls `fetch('/api/roles/status')` but never parses intelligence data from the response. All four widgets (Revenue Radar, Client Health, Cash Flow, Capacity) hardcode `gatheringData: true` regardless of API response. The purpose-built `/api/intelligence/[metric]` route is never called by any UI component.

**Fix:** Wire `IntelligenceWidgets` to fetch from `/api/intelligence/{metric}` endpoints.

### 3. SEO/Tender Scheduled Ticks Missing

**Severity:** Moderate
**Affects:** SEO-03, SEO-04, TNDR-03, TNDR-04

SEO and tender tools are registered as chat-invokable tool groups but have no scheduled execution. Requirements specify "runs on scheduled tick" for both, but no cron or role evaluate() calls these tools proactively.

**Fix:** Either add dedicated cron routes or integrate into an existing role's `evaluate()` loop.
