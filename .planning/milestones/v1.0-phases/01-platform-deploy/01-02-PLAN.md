<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Configure bitbit.com.au domain and DNS</name>
  <action>
    Andy must configure the domain:

    1. In Vercel Dashboard → Project → Settings → Domains → Add `bitbit.com.au`
    2. Vercel will show required DNS records (typically CNAME to `cname.vercel-dns.com`)
    3. Go to domain registrar (Namecheap, Cloudflare, or wherever bitbit.com.au is registered)
    4. Add DNS record: CNAME `bitbit.com.au` → `cname.vercel-dns.com`
    5. If apex domain (no www), may need A record instead — follow Vercel's instructions
    6. Wait for DNS propagation (usually 5-30 minutes)
    7. Vercel will auto-provision SSL certificate

    Also add Supabase auth redirect URL:
    1. Go to Supabase Dashboard → Authentication → URL Configuration
    2. Add `https://bitbit.com.au` to Redirect URLs
    3. Set Site URL to `https://bitbit.com.au`
  </action>
  <verify>
    ```bash
    curl -s -o /dev/null -w "%{http_code}" https://bitbit.com.au/
    ```
    Returns 200 or 302. SSL valid (no certificate errors).
  </verify>
  <done>bitbit.com.au resolves to Vercel deployment with valid SSL. Supabase auth redirect configured.</done>
</task>
