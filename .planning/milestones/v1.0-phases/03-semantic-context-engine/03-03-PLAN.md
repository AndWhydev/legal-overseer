  <action>
    Create `personal-assistant/src/lib/context/assembler.ts`:
    - Uses `createClient` from `@/lib/supabase/server`
    - Import types from `./types`

    `assembleEntityBriefing(orgId, entityType: EntityType, entityId: string, options?: { maxTimelineEvents?: number, maxMemories?: number })`:
    - Queries entity_relationships WHERE (entity_a_type = type AND entity_a_id = id) OR (entity_b_type = type AND entity_b_id = id), ordered by strength DESC, limit 20
    - Queries entity_timeline WHERE entity_type = type AND entity_id = id, ordered by occurred_at DESC, limit options.maxTimelineEvents ?? 15
    - Queries semantic_memories WHERE entity_ids @> ARRAY[entityId]::uuid[] AND is_active = true, ordered by confidence DESC, limit options.maxMemories ?? 10
    - Returns `EntityBriefing { entity: { type, id }, relationships: RelationshipEdge[], timeline: TimelineEntry[], memories: MemoryEntry[] }`

    `assembleContext(orgId, query: string)`:
    - Uses `resolveEntityRanked` from ./entity-resolver to find entities mentioned in query
    - For each resolved entity (max 3), calls assembleEntityBriefing
    - Returns `ContextBriefing { resolvedEntities: ResolvedEntity[], briefings: EntityBriefing[], summary: string }`
    - `summary` is a plain-text formatted string suitable for injection into a system prompt (entity name, type, key relationships, recent events, memories — budgeted to ~2000 chars per entity)

    Add these interfaces to `types.ts`:
    - `EntityBriefing`, `ContextBriefing`, `RelationshipEdge`, `TimelineEntry`, `MemoryEntry`, `ResolvedEntity`

    Create `personal-assistant/src/lib/context/cross-reference.ts`:
    - `crossReference(orgId, entityType: EntityType, entityId: string)`:
      - Finds all related entity IDs from entity_relationships
      - For contact entities: queries tasks table for tasks where the contact is linked (via entity_relationships), queries invoices where client_contact_id matches
      - Returns `CrossReference { relatedTasks: TaskRef[], deadlines: Deadline[], financialSignals: FinancialSignal[], waitingFor: WaitingFor[] }`
    - `getRelatedTasks(orgId, contactId)` — tasks linked to contact via entity_relationships with status != 'archived'
    - `getFinancialSignals(orgId, contactId)` — invoices for contact: total outstanding, overdue count, last payment date
    - `getDeadlines(orgId, entityId)` — tasks with target dates approaching (next 14 days)

    Add `CrossReference`, `TaskRef`, `Deadline`, `FinancialSignal`, `WaitingFor` to types.ts.
  </action>

  </verify>
