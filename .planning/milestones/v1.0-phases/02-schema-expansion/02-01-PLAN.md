Create migration `005_entity_relationships.sql` for the entity relationship graph.

The entity_relationships table models directed edges between any two business entities (contacts, tasks, invoices, projects, channels, messages). This is the core graph structure for "Sezer works_on White House RE" and "Invoice AWU-001 bills_for White House RE".

Schema:
```sql
CREATE TABLE entity_relationships (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id uuid REFERENCES organizations ON DELETE CASCADE NOT NULL,
  entity_a_type text NOT NULL,  -- 'contact', 'task', 'invoice', 'project', 'channel_message'
  entity_a_id uuid NOT NULL,
  entity_b_type text NOT NULL,
  entity_b_id uuid NOT NULL,
  relationship_type text NOT NULL,  -- 'works_on', 'bills_for', 'assigned_to', 'mentioned_in', 'related_to', 'manages', 'created_by'
  strength float NOT NULL DEFAULT 1.0,  -- 0.0-1.0, decays over time or strengthens with more evidence
  metadata jsonb DEFAULT '{}',  -- flexible: { project_name: "White House RE", role: "primary_contact" }
  last_evidence_at timestamptz DEFAULT now(),  -- when was this relationship last confirmed
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);
```

Add CHECK constraint: `entity_a_type IN ('contact', 'task', 'invoice', 'project', 'channel_message', 'goal')`.
Same for entity_b_type.
Add CHECK: `relationship_type IN ('works_on', 'bills_for', 'assigned_to', 'mentioned_in', 'related_to', 'manages', 'created_by', 'client_of', 'vendor_for', 'part_of')`.

Indexes:
- `idx_entity_rel_org` on (org_id)
- `idx_entity_rel_a` on (org_id, entity_a_type, entity_a_id) -- find all relationships FROM an entity
- `idx_entity_rel_b` on (org_id, entity_b_type, entity_b_id) -- find all relationships TO an entity
- `idx_entity_rel_type` on (org_id, relationship_type) -- find all relationships of a type
- Unique constraint on (org_id, entity_a_type, entity_a_id, entity_b_type, entity_b_id, relationship_type) to prevent duplicates

**006_entity_timeline.sql** — Unified timeline per entity across all channels.

Every event that affects an entity (task created, message received, invoice sent, status changed) writes a row here. This answers "what happened to Sezer this week?" across all channels.

```sql
CREATE TABLE entity_timeline (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id uuid REFERENCES organizations ON DELETE CASCADE NOT NULL,
  entity_type text NOT NULL,  -- same CHECK as entity_relationships
  entity_id uuid NOT NULL,
  event_type text NOT NULL,  -- 'message_received', 'message_sent', 'task_created', 'task_updated', 'invoice_sent', 'invoice_paid', 'status_changed', 'mention', 'note_added'
  event_data jsonb NOT NULL DEFAULT '{}',  -- { subject: "RE: Website changes", from: "sezer@...", snippet: "..." }
  channel_source text,  -- 'gmail', 'whatsapp', 'asana', 'manual', 'system'
  related_entity_type text,  -- optional: the other entity involved
  related_entity_id uuid,    -- optional: FK to the other entity
  occurred_at timestamptz NOT NULL DEFAULT now(),  -- when the event actually happened (not when recorded)
  created_at timestamptz DEFAULT now()
);
```

CHECK on entity_type: same set as entity_relationships.
CHECK on event_type: `IN ('message_received', 'message_sent', 'task_created', 'task_updated', 'task_completed', 'invoice_created', 'invoice_sent', 'invoice_paid', 'invoice_overdue', 'status_changed', 'mention', 'note_added', 'relationship_created', 'contact_created', 'contact_updated')`.

Indexes:
- `idx_timeline_entity` on (org_id, entity_type, entity_id, occurred_at DESC) -- main query: timeline for an entity
- `idx_timeline_org_time` on (org_id, occurred_at DESC) -- org-wide timeline
- `idx_timeline_channel` on (org_id, channel_source, occurred_at DESC) -- filter by channel
- `idx_timeline_event_type` on (org_id, event_type) -- find all events of a type

No updated_at trigger needed (timeline entries are append-only, not updated).

**007_semantic_memories.sql** — Learnable facts with confidence, supersession, and source tracking.

This replaces the flat `memory_entries` table with a richer structure. Each memory links to specific entities and source events, can be superseded by newer information, and has a confidence score.

```sql
CREATE TABLE semantic_memories (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id uuid REFERENCES organizations ON DELETE CASCADE NOT NULL,
  entity_ids uuid[] NOT NULL DEFAULT '{}',  -- which entities this memory is about
  category text NOT NULL DEFAULT 'general',  -- 'preference', 'relationship', 'financial', 'behavioral', 'factual', 'general'
  content text NOT NULL,  -- "Steve always pays within 48hrs", "Sezer prefers email for invoices"
  confidence float NOT NULL DEFAULT 1.0 CHECK (confidence >= 0.0 AND confidence <= 1.0),
  source_events uuid[] DEFAULT '{}',  -- references to entity_timeline.id entries that produced this memory
  superseded_by uuid REFERENCES semantic_memories ON DELETE SET NULL,  -- if a newer memory replaces this one
  is_active boolean NOT NULL DEFAULT true,  -- false when superseded
  extracted_by text,  -- 'reflection_agent', 'manual', 'system'
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);
```

CHECK on category: `IN ('preference', 'relationship', 'financial', 'behavioral', 'factual', 'general', 'procedural')`.

Indexes:
- `idx_memories_org` on (org_id, is_active) WHERE is_active = true -- active memories for an org
- `idx_memories_entities` GIN on (entity_ids) -- find memories about specific entities
- `idx_memories_source` GIN on (source_events) -- trace which events produced a memory
- `idx_memories_category` on (org_id, category) WHERE is_active = true

Add updated_at trigger using existing `update_updated_at()` function.
