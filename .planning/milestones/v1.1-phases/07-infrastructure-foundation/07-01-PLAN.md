<objective>
Refactor all agent and context modules to accept a Supabase client as a parameter (dependency injection) instead of calling `createClient()` internally. This eliminates module-level Supabase coupling, enabling multi-tenant support and testability.

Purpose: INFR-01 requires all tools receive the Supabase client from execution context. Currently 14 files import `createClient` from `@/lib/supabase/server` and call it inside functions. This creates tight coupling to the Next.js cookie-based client and prevents the agent runtime (Phase 8) from passing a service-role client for background operations.

Output: All agent/context/channel modules accept `SupabaseClient` as first parameter. Callers (API routes, server actions) create the client once and pass it down.
</objective>
