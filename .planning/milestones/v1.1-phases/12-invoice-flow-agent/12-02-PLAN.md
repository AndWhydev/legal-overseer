  <action>
    Create invoice PDF generation and sending modules.

    In `invoice-pdf.ts`:
    - `generateInvoicePdf(invoice, orgSettings)` — Generate a branded PDF invoice as a Buffer/Uint8Array.
    - Use a simple HTML-to-string approach (no heavy PDF library needed for v1 — generate clean HTML invoice template and return as string for email body, with a note that PDF attachment uses a lightweight lib like `pdfkit` or `jspdf` if available, otherwise return HTML).
    - Template includes: org name/logo placeholder, invoice number, date, due date, client details, line items table, subtotal, GST (10%), total, payment terms, bank details placeholder.
    - Payment terms configurable: 7-day, 14-day, 30-day (from org settings or invoice metadata).
    - Return `{ html: string, subject: string }` for email sending.

    In `invoice-sender.ts`:
    - `queueInvoiceSend(supabase, orgId, invoiceId, agentConfigId)` — Create approval queue entry with action_type invoice_send, include invoice summary in action_summary. Always routes through approval (confidence_score = 0, routing_decision = ask).
    - `processApprovedInvoiceSends(supabase, orgId)` — Find approved invoice_send approvals, generate PDF/HTML, update invoice status to sent, update sent_via and pdf_url fields. Follow lead-acknowledgment pattern for processing approved items.
    - `checkOverdueInvoices(supabase, orgId)` — Find invoices where status=sent and due_date < today, update to overdue. Increment reminder_count. Queue notification approval for overdue alerts.
    - Status lifecycle: draft (created) → sent (after approval + email) → viewed (future webhook) → overdue (past due_date) → paid (manual or webhook update).

    Add vitest tests for:
    - PDF/HTML generation includes all required fields
    - Approval queue integration creates correct action_type
    - Overdue detection finds past-due invoices
    - Status transitions are valid (no skip from draft to paid)
    - Processing approved sends updates invoice status
