  <action>
    Create the Invoice Flow agent module following established agent patterns (sentry, lead-swarm).

    In `023_invoice_flow.sql`, extend `invoices` table with:
    - `project_reference text` (the project/work description for duplicate matching)
    - `source_intent text` (original NL command for audit trail)
    - `created_by text DEFAULT manual` (track agent vs manual creation)
    - Add unique partial index on (org_id, client_contact_id, project_reference, total) WHERE status != cancelled for duplicate detection

    In `invoice-flow.ts`, implement:
    - `parseInvoiceIntent(text: string)` — Extract contact name, project/work description, amount (if specified), and payment terms from natural language. Return structured `InvoiceIntent` with parsed fields.
    - `resolveInvoiceEntities(supabase, orgId, intent)` — Use `searchContacts` from shared-tools and `resolveEntityRanked` from entity-resolver to resolve contact name to contact ID. Search for related projects/tasks via cross-reference to determine work scope and suggest amount if not specified.
    - `detectDuplicateInvoice(supabase, orgId, contactId, projectRef, total)` — Query invoices table for matching (org, contact, project_reference, total) where status is not cancelled. Return boolean + existing invoice if duplicate found.
    - `createInvoiceFromIntent(supabase, orgId, intent, agentConfigId)` — Orchestrate: parse → resolve entities → check duplicates → create invoice via shared-tools createInvoice. If duplicate detected, queue approval with action_type invoice_duplicate_override. Return created invoice or duplicate warning.
    - `runInvoiceFlowTick(supabase, orgId, agentConfigId)` — Process pending invoice commands (from approval queue approved items with action_type invoice_create). Never-throw batch pattern returning { processed, created, duplicatesBlocked, failed }.

    Generate next invoice number using pattern: `{ORG_PREFIX}-{YYYYMM}-{NNN}` where NNN is auto-incremented per org per month.

    Add vitest coverage for:
    - NL parsing (various phrasings: "Invoice X for Y", "Send invoice to X for Y work", "Bill X $500 for Y")
    - Duplicate detection (same contact+project+total = blocked, different total = allowed, cancelled invoice = not duplicate)
    - Invoice number generation (sequential within month)
    - Entity resolution failure handling (unknown contact returns error, not crash)
  </action>
  <verify>Run `cd personal-assistant && npx vitest run src/lib/agent/invoice-flow.test.ts`.</verify>
  <done>NL invoice commands resolve to structured invoices with duplicate protection. INVC-01 and INVC-05 satisfied.</done>
