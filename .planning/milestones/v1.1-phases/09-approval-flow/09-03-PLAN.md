  <files>

  <action>
    Create whatsapp.ts -- WhatsApp Cloud API v21.0 client:
    - sendMessage(to: string, text: string): Send a text message via POST to https://graph.facebook.com/v21.0/{WHATSAPP_PHONE_NUMBER_ID}/messages with Authorization: Bearer {WHATSAPP_ACCESS_TOKEN}. Body: { messaging_product: 'whatsapp', to, type: 'text', text: { body: text } }. Return message ID or null on failure. Never throw (fire-and-forget pattern).
    - sendApprovalRequest(to: string, approvalId: string, summary: string, agentName: string, confidence: number): Format message like: "[BitBit] {agentName} wants to: {summary}\nConfidence: {confidence}%\nReply Y to approve, N to reject\nRef: {approvalId short}". Call sendMessage. Return message ID.
    - sendDigest(to: string, approvals: Array<{id, summary, agentName}>): Format message listing all approvals: "[BitBit] Daily Digest - {count} pending approvals:\n\n1. {agentName}: {summary}\n2. ...\n\nReply with number + Y/N (e.g. '1Y' or '2N') or open dashboard to review all." Call sendMessage. Return message ID.
    - parseApprovalReply(text: string): Parse incoming reply. Handle: "Y"/"N" (simple for single pending), "1Y"/"2N" (digest reply format). Return { type: 'simple'|'indexed', index?: number, decision: 'approved'|'rejected' } or null if unparseable.

    Env vars: WHATSAPP_PHONE_NUMBER_ID, WHATSAPP_ACCESS_TOKEN, WHATSAPP_VERIFY_TOKEN, WHATSAPP_ANDY_PHONE. All read from process.env. If missing, sendMessage returns null with console.warn.

    Create webhook route.ts:
    - GET handler: WhatsApp webhook verification. Check query params hub.mode === 'subscribe' && hub.verify_token === WHATSAPP_VERIFY_TOKEN. Return hub.challenge as plain text. Return 403 otherwise.
    - POST handler: Receive incoming messages. Parse WhatsApp webhook payload (entry[0].changes[0].value.messages[0]). Extract sender phone and message text. If sender matches WHATSAPP_ANDY_PHONE, call parseApprovalReply. If valid reply:
      - For 'simple' type: find most recent pending approval for the org via getPendingApprovals (limit 1), call resolveApproval with resolved_via='whatsapp'.
      - For 'indexed' type: get pending approvals, resolve the one at the given index.
      - Reply with confirmation: "Approved: {summary}" or "Rejected: {summary}".
    - If not an approval reply or sender not Andy, log and return 200 (acknowledge but ignore).
    - Always return 200 to WhatsApp (they retry on non-200).

    Create whatsapp.test.ts:
    - Test parseApprovalReply: "Y" -> approved, "N" -> rejected, "y" -> approved (case-insensitive), "1Y" -> indexed approve, "2N" -> indexed reject, "hello" -> null, "" -> null.
    - Test sendApprovalRequest message formatting
    - Test sendDigest message formatting with multiple items

    Use the RELAY_SECRET / SCHEDULER_SECRET pattern for the webhook endpoint -- but WhatsApp webhook uses its own verify_token mechanism, so no Bearer auth needed on this endpoint.
  </action>

Only sends if routing_decision is 'escalate' OR (routing_decision is 'ask' AND priority is 'urgent' or 'normal').     - sendDailyDigest(supabase, orgId: string): Calls getDigestApprovals to get all pending digest_eligible items. If none, do nothing. If any, calls sendDigest with all items. Returns count sent. Never throws.

    Create digest route.ts at /api/agent/approvals/digest:

    - Calls sendDailyDigest
    - Also calls expireStaleApprovals to clean up expired items
    - Returns JSON { digestSent: count, expired: count }
    - This endpoint will be called by a daily cron job (e.g., 9 AM AEST)
