  <action>
    Create migration 020_approval_queue.sql with table:
    - id uuid PK default gen_random_uuid()
    - org_id uuid REFERENCES organizations NOT NULL
    - agent_config_id uuid REFERENCES agent_configs NOT NULL
    - agent_run_id uuid REFERENCES agent_runs (nullable, link to run that produced this)
    - action_type text NOT NULL (e.g. 'send_email', 'create_invoice', 'update_contact')
    - action_payload jsonb NOT NULL DEFAULT '{}'
    - action_summary text NOT NULL (human-readable description of what agent wants to do)
    - confidence_score float NOT NULL
    - routing_decision text NOT NULL CHECK (routing_decision IN ('ask', 'escalate'))
    - priority text NOT NULL DEFAULT 'normal' CHECK (priority IN ('urgent', 'normal', 'low'))
    - digest_eligible boolean NOT NULL DEFAULT false (true for low-priority items that can batch into daily digest)
    - status text NOT NULL DEFAULT 'pending' CHECK (status IN ('pending', 'approved', 'rejected', 'expired', 'auto_expired'))
    - context_snapshot jsonb DEFAULT '{}' (entity context at time of action for reviewer)
    - resolved_by uuid REFERENCES auth.users (nullable)
    - resolved_at timestamptz (nullable)
    - resolved_via text CHECK (resolved_via IN ('dashboard', 'whatsapp', 'auto_expire', NULL))
    - expires_at timestamptz (default now() + interval '24 hours')
    - created_at timestamptz DEFAULT now()

    Indexes: org_id + status (pending filter), org_id + digest_eligible + status (digest query), expires_at WHERE status = 'pending' (expiry cleanup).

    RLS: Enable RLS. Policy allowing authenticated users to SELECT and UPDATE where org_id matches their org (same pattern as 017_rls_new_tables.sql).

    Create approval-queue.ts with DI pattern (SupabaseClient as first param):
    - createApproval(supabase, params): Insert new approval record. Params include org_id, agent_config_id, agent_run_id, action_type, action_payload, action_summary, confidence_score, routing_decision, priority, context_snapshot. Auto-set digest_eligible = true when priority === 'low'. Returns the created record.
    - resolveApproval(supabase, approvalId, decision: 'approved'|'rejected', resolvedBy: string, resolvedVia: 'dashboard'|'whatsapp'): Update status, resolved_by, resolved_at, resolved_via. Return updated record. Throw if already resolved.
    - getPendingApprovals(supabase, orgId, opts?: { limit?, offset?, priorityFilter? }): Get pending approvals ordered by priority (urgent first) then created_at ASC. Join agent_configs to include agent name.
    - getDigestApprovals(supabase, orgId): Get pending approvals where digest_eligible = true, for batching into daily digest.
    - expireStaleApprovals(supabase, orgId): Update status to 'auto_expired' where expires_at < now() AND status = 'pending'. Return count expired.
    - queueAgentAction(supabase, params): High-level function that calls routeAgentAction from confidence-router.ts, and if decision is 'ask' or 'escalate', calls createApproval. If 'act', returns null (no approval needed). This is the main entry point agents will call.

    Create approval-queue.test.ts with vitest tests:
    - Test queueAgentAction returns null for confidence > 0.85
    - Test queueAgentAction creates approval for confidence 0.55-0.85 with routing_decision='ask'
    - Test queueAgentAction creates approval for confidence < 0.55 with routing_decision='escalate'
    - Test digest_eligible auto-set for low priority
    - Test resolveApproval rejects already-resolved items
    - Test getPendingApprovals ordering (urgent first)
