Create `018_channel_relay.sql` migration that adds to channel_connections:
- `poll_cursor TIMESTAMPTZ` -- last successfully polled timestamp (relay reads from here)
- `poll_interval_seconds INTEGER NOT NULL DEFAULT 300` -- configurable per-org poll interval (5 min default)
- `relay_enabled BOOLEAN NOT NULL DEFAULT false` -- opt-in per channel per org

Create `relay-daemon.ts` with these exports:

1. `pollChannel(supabase: SupabaseClient, orgId: string, channelType: ChannelType): Promise<PollResult>` -- Main entry point:
   - Read channel_connections row for (orgId, channelType) to get poll_cursor and config
   - If relay_enabled is false, return early with { skipped: true }
   - Call the appropriate adapter's pull(config, since=poll_cursor) to fetch new messages
   - Upsert each message into channel_messages using ON CONFLICT (org_id, channel, external_id) DO NOTHING (idempotent)
   - Update channel_connections.poll_cursor to the latest message receivedAt
   - Update channel_connections.last_sync to NOW()
   - Return PollResult { messagesFound, messagesInserted, skipped: false }

2. `processNewMessages(supabase: SupabaseClient, orgId: string): Promise<ChannelMessage[]>` -- Fetch unprocessed messages:
   - SELECT from channel_messages WHERE org_id = orgId AND processed = false ORDER BY received_at ASC LIMIT 50
   - Return the messages (classification happens in plan 08-02)

Use the Supabase DI pattern (SupabaseClient as first param). Import gmailAdapter from ./gmail. Use the existing ChannelMessage type.

Define PollResult interface: `{ messagesFound: number; messagesInserted: number; skipped: boolean; error?: string }`.

Never throw on adapter failures -- catch errors and return them in PollResult.error.

Create `relay-daemon.test.ts` with unit tests using mock supabase:
- pollChannel returns skipped when relay_enabled is false
- pollChannel upserts messages from adapter
- pollChannel updates poll_cursor after successful poll
- processNewMessages returns only unprocessed messages
- pollChannel handles adapter errors gracefully (returns error in PollResult)
