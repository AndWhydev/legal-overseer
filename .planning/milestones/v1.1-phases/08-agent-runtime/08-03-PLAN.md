Create `scheduler.ts`:

Export `AgentScheduleResult` interface:
```
{
  agentType: string
  orgId: string
  triggered: boolean
  reason: string           // "due", "not_due", "disabled", "already_running"
  lastRunAt?: string
  nextDueAt?: string
}
```

Export `shouldRunAgent(schedule: AgentSchedule, lastRunAt: Date | null, now: Date): boolean`:
- Pure function (no DB, no async)
- AgentSchedule: `{ type: 'continuous' | 'interval' | 'cron', interval_seconds?: number, cron_expression?: string }`
- For `type === 'interval'`: return true if lastRunAt is null OR (now - lastRunAt) >= interval_seconds * 1000
- For `type === 'cron'`: parse cron expression manually (simple 5-field cron: min hour dom month dow). Determine if the current minute matches the pattern. Use a minimal cron matcher -- do NOT import node-cron or external library. Support: `*`, specific numbers, and `*/N` step syntax. This is sufficient for v1.1 agents (every 5 min = `*/5 * * * *`, hourly = `0 * * * *`, daily = `0 9 * * *`).
- For `type === 'continuous'`: always return true (runs every tick)

Export `runScheduledAgents(supabase: SupabaseClient, orgId?: string): Promise<AgentScheduleResult[]>`:
- Query agent_configs WHERE enabled = true (and optionally org_id = orgId)
- For each enabled agent config:
  - Query agent_runs for this agent+org ORDER BY created_at DESC LIMIT 1 to get lastRunAt
  - Call shouldRunAgent with the config's schedule and lastRunAt
  - If should run: insert a placeholder agent_run with trigger_type='scheduled', output_summary='pending' (the actual agent execution will be wired in Phase 10-12 when agents exist). Log via run-logger's logAgentRun.
  - Build AgentScheduleResult with triggered=true/false and reason
- Return all results

The scheduler is a "tick" function -- it runs once per invocation, checks what's due, and triggers. It does NOT loop or sleep. The loop is handled by the cron caller (Vercel cron or external).

Create `scheduler.test.ts`:
- shouldRunAgent with interval schedule: not due (ran 30s ago, interval 300s) → false
- shouldRunAgent with interval schedule: due (ran 600s ago, interval 300s) → true
- shouldRunAgent with interval schedule: never ran → true
- shouldRunAgent with cron `*/5 * * * *`: at minute 5 → true, at minute 3 → false
- shouldRunAgent with cron `0 9 * * *`: at 09:00 → true, at 09:01 → false
- shouldRunAgent with continuous: always true
- runScheduledAgents skips disabled agents
- runScheduledAgents inserts agent_run for triggered agents
