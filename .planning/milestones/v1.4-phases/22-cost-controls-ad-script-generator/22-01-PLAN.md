<objective>
Add per-execution token budget caps and per-role daily budget limits that prevent growth role token cost spirals.

Purpose: Growth roles (ad scripts, SEO, content, tenders) are generative workloads that use 10-50x more tokens than classification tasks. Without per-execution and per-role budget limits, a single user session could generate hundreds of dollars in API costs. This plan implements the cost safety net that must exist before any growth role is exposed to users.

Output: Extended cost-guard.ts with role-aware budget enforcement, engine.ts wired to check and enforce budgets, usage-metering.ts with per-role daily query function.
</objective>
