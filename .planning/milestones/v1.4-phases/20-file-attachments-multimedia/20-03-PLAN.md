  <files>personal-assistant/src/components/chat/chat-attachment.tsx</files>
  <action>
Create `chat-attachment.tsx` exporting two components:

**ChatAttachment** -- Single attachment preview. Props:
```typescript
interface ChatAttachmentProps {
  type: string      // MIME type
  name: string      // filename
  url: string       // storage_path (NOT a signed URL -- we fetch that on demand)
  attachmentId?: string  // if available, use /api/attachments/:id for download URL
}
```

Rendering logic based on MIME type:

1. **Images** (type starts with 'image/'):
   - Render an `<img>` element with the image loaded from a signed download URL
   - Use a `useState` + `useEffect` to fetch the signed URL from `/api/attachments/:id` (GET) on mount, or if attachmentId is not available, show placeholder
   - Style: max-width 300px, max-height 200px, rounded corners (8px), subtle border (1px solid rgba(255,255,255,0.1))
   - On click: open full-size image in new tab
   - Show a loading skeleton (gray placeholder) while the signed URL is being fetched
   - Handle error: show broken image icon with filename

2. **PDFs** (type === 'application/pdf'):
   - Render a card with: FileText icon (from lucide-react, 24px), filename (truncated to 30 chars with ellipsis), file size if available, and a "Download" link
   - Style: inline-flex card with glassmorphic background (rgba(255,255,255,0.06)), padding 12px, rounded corners, hover brightness increase
   - On click: fetch signed download URL from `/api/attachments/:id`, open in new tab
   - Do NOT attempt to render PDF thumbnails (deferred per research -- use static icon)

3. **Other files** (DOCX, CSV, TXT, etc.):
   - Render same card layout as PDF but with a generic File icon (from lucide-react)
   - Filename + "Download" link
   - Same click behavior: fetch signed URL, open/download

All download link clicks should:
- Fetch `/api/attachments/:id` GET to get a fresh signed download URL (don't cache URLs -- they expire)
- Open the signed URL in a new tab or trigger download

Use **inline React styles** per project convention (not Tailwind classes for visual design). Tailwind is OK for layout utilities (flex, gap, items-center).

**ChatAttachmentList** -- Renders a list of ChatAttachment components. Props:
```typescript
interface ChatAttachmentListProps {
  attachments: Array<{ type: string; url: string; name: string }>
  // url here is the storage_path, used to derive attachmentId or fetch download URL
}
```

Renders attachments in a flex-wrap container with gap-2 spacing. Images in a row, documents below.
