### Task 1: Workflow Executor
Create `personal-assistant/src/lib/roles/workflow-executor.ts`:

```typescript
export interface WorkflowDefinition {
  type: string  // e.g. 'invoice_reminder_sequence'
  steps: WorkflowStepDef[]
  context: Record<string, unknown>
}

export interface WorkflowStepDef {
  id: string
  name: string
  execute: (ctx: WorkflowStepContext) => Promise<WorkflowStepResult>
  delaySeconds?: number  // wait before executing this step (for time-delayed workflows)
  condition?: (ctx: WorkflowStepContext) => boolean  // skip if false
}

export interface WorkflowStepContext {
  supabase: SupabaseClient
  orgId: string
  roleConfig: RoleConfig
  workflow: RoleWorkflow
  stepResults: Record<string, unknown>  // results from prior steps
}

export interface WorkflowStepResult {
  success: boolean
  result?: unknown
  error?: string
  nextStepDelay?: number  // override delay for next step
}

/**
 * Start a new workflow — creates DB record, executes first eligible step.
 */
export async function startWorkflow(
  supabase: SupabaseClient,
  roleConfig: RoleConfig,
  definition: WorkflowDefinition,
): Promise<RoleWorkflow>

/**
 * Resume a paused/active workflow — picks up from current_step.
 * Called during role tick when next_step_at <= now.
 */
export async function resumeWorkflow(
  supabase: SupabaseClient,
  workflow: RoleWorkflow,
  roleConfig: RoleConfig,
  stepDefs: WorkflowStepDef[],
): Promise<RoleWorkflow>

/**
 * Cancel a workflow — sets status to 'cancelled'.
 */
export async function cancelWorkflow(
  supabase: SupabaseClient,
  workflowId: string,
): Promise<void>
```

Key behaviors:
- Each step execution is atomic: start → execute → save result → advance current_step
- Time-delayed steps: set `next_step_at` and return. Next role tick picks up.
- Failed step: set workflow status to 'failed', log error to role_activity
- All steps complete: set status to 'completed'
- Each step logs to role_activity with type 'workflow_step'

### Task 3: Per-Role Cost Guards
Create `personal-assistant/src/lib/roles/role-cost-guard.ts`:

```typescript
export interface RoleCostCheck {
  allowed: boolean
  reason?: string
  spentToday: number
  dailyBudget: number
  remainingBudget: number
}

/**
 * Check if a role can proceed based on its daily budget.
 * Queries agent_runs for today's cost for this role_config_id.
 */
export async function canRoleProceed(
  supabase: SupabaseClient,
  roleConfigId: string,
  dailyBudgetCents: number,
): Promise<RoleCostCheck>

/**
 * Haiku pre-screen: cheap check before expensive evaluation.
 * Returns true if there's new data since last tick that warrants full evaluation.
 */
export async function shouldEvaluate(
  supabase: SupabaseClient,
  roleConfig: RoleConfig,
  roleState: RoleState,
): Promise<{ shouldRun: boolean; reason: string }>
```

The `shouldEvaluate` function checks:
- Finance: new invoices, payments, or overdue items since last tick
- Comms: new messages received since last tick
- Sales: new leads or proposal responses since last tick

This is the Haiku pre-screen that prevents unnecessary Sonnet/Opus calls.

### Task 5: Integration Test — End-to-End Role Tick
Create `personal-assistant/src/lib/roles/__tests__/role-engine.test.ts`:

Test the full flow:
1. Create a role config (finance, copilot mode)
2. Create role state
3. Execute role tick
4. Verify: lock acquired, state loaded, evaluation ran, actions dispatched through autonomy gate, activity logged, state saved, lock released

Test scenarios:
- Observer mode: actions become insights only
- Co-pilot mode: actions become approvals
- Autopilot mode: high-confidence actions execute
- Concurrent tick: second tick skips (advisory lock)
- Version conflict: detected and handled
- Cost guard: over-budget tick skips
- Workflow: start, execute step 1, pause at step 2 (delayed), resume on next tick
