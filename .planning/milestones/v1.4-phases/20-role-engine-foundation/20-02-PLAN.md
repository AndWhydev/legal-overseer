# Plan 20-02: Role Runtime — State, Ticks, Events, Concurrency

## Goal
Build the role runtime: loading state, executing role ticks, handling events, and ensuring concurrent activations don't corrupt state.

## Context
- Existing scheduler pattern: `runScheduledAgents()` in `scheduler.ts` — stateless tick, dispatch map, cost guard, circuit breaker
- Existing cron: `withCronGuard()` wrapper, service-role Supabase client
- Pattern: Agents are functions `(supabase, orgId, configId) => Promise<string>`
- New: Roles need persistent state loaded before tick, saved after, with concurrency control

## Tasks

### Task 1: Role Runtime Core
Create `personal-assistant/src/lib/roles/role-runtime.ts`:

```typescript
// Core runtime that manages role lifecycle
export interface RoleTickResult {
  roleType: RoleType
  orgId: string
  triggered: boolean
  actionsGenerated: number
  insightsGenerated: number
  costCents: number
  durationMs: number
  error?: string
}

export interface RoleContext {
  config: RoleConfig
  state: RoleState
  supabase: SupabaseClient
  orgId: string
  autonomyLevel: AutonomyLevel
}

// Load role state (create if not exists)
export async function loadRoleState(supabase, roleConfigId, orgId): Promise<RoleState>

// Save role state with optimistic concurrency (version check)
export async function saveRoleState(supabase, roleState): Promise<RoleState>

// Acquire advisory lock for role execution (prevents concurrent ticks)
export async function acquireRoleLock(supabase, roleConfigId): Promise<boolean>

// Release advisory lock
export async function releaseRoleLock(supabase, roleConfigId): Promise<void>

// Main tick executor
export async function executeRoleTick(supabase, roleConfig): Promise<RoleTickResult>
```

The `executeRoleTick` flow:
1. `acquireRoleLock(roleConfigId)` — if locked, skip (another tick is running)
2. `loadRoleState()` — get or create persistent state
3. Haiku pre-screen: "Has anything changed since last tick?" — if no, skip (cost saving)
4. Execute role-specific logic (delegated to role implementation)
5. `saveRoleState()` with version check
6. `releaseRoleLock()`
7. Log activity to `role_activity`
8. Return result

### Task 2: Role Registry

// Each role implements this interface
export interface RoleImplementation {
  type: RoleType
  name: string
  description: string

  // Called during tick — returns what the role wants to do
  evaluate(ctx: RoleContext): Promise<RoleEvaluation>

  // Called to check if anything changed (Haiku pre-screen)
  hasChanges(ctx: RoleContext): Promise<boolean>

  // Default config for new orgs
  defaultConfig(): Partial<RoleConfig>
}

export interface RoleEvaluation {
  actions: RoleAction[]      // things to do
  insights: RoleInsight[]    // things to surface
  stateUpdates: Record<string, unknown>  // state mutations
  workflowsToStart: WorkflowDefinition[]
}

export interface RoleAction {
  type: string
  summary: string
  payload: Record<string, unknown>
  confidence: number
  reversible: boolean
}

export interface RoleInsight {
  summary: string
  details: Record<string, unknown>
  priority: 'high' | 'medium' | 'low'
}

// Registry
const roleImplementations = new Map<RoleType, RoleImplementation>()

### Task 3: Role Scheduler Integration
Create `personal-assistant/src/lib/roles/role-scheduler.ts`:

```typescript
// Analogous to agent scheduler but for roles
export async function runScheduledRoles(supabase: SupabaseClient): Promise<RoleTickResult[]>
```

Flow:
1. Query `role_configs` for enabled roles
2. For each: check if tick is due (last_tick_at + tick_interval_seconds < now)
3. Execute role tick with concurrency guard
4. Log to `agent_runs` (reuse existing logging, add `trigger_type: 'role_tick'`)

### Task 4: Cron Endpoint
Create `personal-assistant/src/app/api/cron/role-tick/route.ts`:

```typescript
export const maxDuration = 300
export const dynamic = 'force-dynamic'

export async function GET(request: Request) {
  return withCronGuard(request, async (supabase) => {
    const results = await runScheduledRoles(supabase)
    return {
      message: `Role tick complete, processed ${results.length} roles`,
      details: { roleTickResults: results },
    }
  })
}
```

Add to `vercel.json` crons (every 5 minutes).

### Task 5: Role Initialization — Seed Default Configs
Create `personal-assistant/src/lib/roles/role-init.ts`:

When an org enables a role for the first time:
1. Insert `role_configs` row with defaults

3. Link existing agent_configs (e.g., finance role links to invoice-flow agent)

API route: `POST /api/roles/[roleType]/enable` and `POST /api/roles/[roleType]/disable`
