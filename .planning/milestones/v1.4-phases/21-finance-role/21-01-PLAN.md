# Plan 21-01: Finance Role Implementation — Wraps Invoice Agent

## Goal
Create the Finance role that wraps the existing invoice agent as a sub-component. All current invoice capabilities must work through the Finance role. This is the "wrap, don't rewrite" approach from pitfalls research.

## Context
- Existing: `src/lib/agent/invoice-flow.ts` (727 lines) — createInvoiceFromIntent, runInvoiceFlowTick, duplicate detection
- Existing: `src/lib/agent/invoice-sender.ts` — checkOverdueInvoices, processApprovedInvoiceSends
- Existing: `src/lib/roles/role-registry.ts` — RoleImplementation interface, registerRole
- Existing: `src/lib/roles/role-runtime.ts` — executeRoleTick, RoleContext
- Pattern: Roles implement RoleImplementation interface: evaluate(), hasChanges(), defaultConfig()

## Tasks

### Task 1: Finance Role Implementation
Create `personal-assistant/src/lib/roles/finance/finance-role.ts`:

```typescript
import type { RoleImplementation, RoleContext, RoleEvaluation } from '../role-registry'
import { registerRole } from '../role-registry'

// Finance role wraps the invoice agent and adds proactive behaviors
const financeRole: RoleImplementation = {
  type: 'finance',
  name: 'Finance',
  description: 'Owns all money operations: invoicing, collections, cash flow, financial reporting',

  async evaluate(ctx: RoleContext): Promise<RoleEvaluation> {
    const actions = []
    const insights = []
    const stateUpdates = {}
    const workflowsToStart = []

    // 1. Run existing invoice flow tick (wrap, don't rewrite)
    const invoiceTick = await runWrappedInvoiceTick(ctx)

    // 2. Convert invoice tick results to role actions/insights
    // - New invoices created → action (type: 'invoice_created')
    // - Duplicates blocked → insight (type: 'duplicate_blocked')
    // - Overdue invoices → action (type: 'send_reminder')
    // - Invoices sent → insight (type: 'invoice_sent')

    return { actions, insights, stateUpdates, workflowsToStart }
  },

  async hasChanges(ctx: RoleContext): Promise<boolean> {
    // Check for: new approved invoice actions, new payments, newly overdue invoices
    // This is the Haiku pre-screen — returns true if there's work to do
  },

  defaultConfig() {
    return {
      tick_interval_seconds: 3600, // Hourly (finance doesn't need 5-min ticks)
      daily_budget_cents: 300,     // $3/day
      autonomy_level: 'copilot',
      config: {
        overdue_reminder_days: [7, 14, 30],
        auto_invoice_enabled: false, // Opt-in for proactive invoicing
      },
    }
  },
}

registerRole(financeRole)
```
