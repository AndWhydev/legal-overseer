<objective>
Consolidate Stripe webhook handling into a single idempotent endpoint, install the Stripe SDK, replace ad-hoc price creation with pre-created Products/Prices, and fix the subscription lifecycle to correctly map org-to-subscription on all event types.

Purpose: The current codebase has two overlapping webhook routes (/api/billing/webhook and /api/webhooks/stripe), creates a new Stripe Price object per checkout (polluting the dashboard), and fails to link orgs to subscriptions on "created" events. These defects must be fixed before any real customer signs up.

Output: Single webhook route, Stripe SDK singleton, pre-created price checkout, subscription-handler with full lifecycle, migration adding stripe_customer_id to organizations.
</objective>
