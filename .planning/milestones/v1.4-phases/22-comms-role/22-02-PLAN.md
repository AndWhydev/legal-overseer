# Plan 22-02: Follow-Up Tracking + Relationship Monitoring

## Goal
Comms role tracks unanswered threads, monitors communication frequency per client, flags engagement drops, and escalates overdue responses.

## Tasks

### Task 1: Follow-Up Tracker
Create `personal-assistant/src/lib/roles/comms/follow-up-tracker.ts`:
- detectUnansweredThreads(supabase, orgId, thresholdHours): finds threads awaiting response
- Configurable SLA threshold per priority level
- Returns unanswered threads with urgency scoring and suggested response

### Task 2: Relationship Monitor
Create `personal-assistant/src/lib/roles/comms/relationship-monitor.ts`:
- monitorCommunicationFrequency(supabase, orgId): tracks message frequency per client
- detectEngagementDrops(): compares recent vs historical communication rates
- CommsHealthSignal: per-client engagement status (active/cooling/dormant)

### Task 3: Tone Adaptation
Create `personal-assistant/src/lib/roles/comms/tone-adapter.ts`:
- learnClientTone(supabase, orgId, contactId): analyzes historical messages for tone/style
- adaptDraft(draft, contactId): adjusts response tone to match client's communication style
- Stores learned patterns in Comms role state

### Task 4: Integrate into Comms evaluate()
Update comms-role.ts:
- Follow-up check → surface overdue threads as actions or insights
- Relationship monitoring → flag engagement drops
- Tone adaptation → apply to all drafted responses
