  <action>
    Build the chat screen with SSE streaming from the existing /api/agent/chat endpoint.

    1. Create `mobile/src/lib/sse.ts`:
       - Use react-native-sse EventSource (NOT fetch -- Android streaming is broken per RN #28835)
       - streamChat function: takes message, threadId, callbacks (onEvent, onDone, onError)
       - Gets session token from supabase.auth.getSession()
       - Creates POST EventSource to `${EXPO_PUBLIC_API_URL}/api/agent/chat`
       - Headers: Content-Type application/json, Authorization Bearer {access_token}
       - Body: JSON.stringify({ message, threadId })
       - Parses SSE events, dispatches to onEvent callback
       - Returns cleanup function that closes the EventSource
       - Follow Pattern 2 from research exactly

    2. Create `mobile/src/hooks/useChat.ts`:
       - State: messages array, isStreaming boolean, currentThreadId, streamingText
       - loadHistory: fetches from /api/conversations/list via apiClient, then thread messages
       - sendMessage(text): adds user message to array, calls streamChat, accumulates streamed tokens into streamingText, on done converts to assistant message
       - Uses useCallback for stable references
       - TanStack useQuery for conversation list, useMutation for sending messages
       - Handles thread creation (null threadId on first message, server returns threadId in response)

    3. Create `mobile/src/components/StreamingText.tsx`:
       - Takes streamingText string prop
       - Renders text with blinking cursor at end while streaming
       - Uses Animated.View for cursor blink

    4. Create `mobile/src/components/ChatBubble.tsx`:
       - Takes message { role: 'user' | 'assistant', content: string, timestamp: string }
       - User messages: right-aligned, primary color background
       - Assistant messages: left-aligned, secondary color background
       - Timestamp below bubble in muted text
       - If isStreaming prop true, render StreamingText instead of static Text

    5. Create `mobile/src/components/ChatInput.tsx`:
       - TextInput with send button
       - Multiline input that grows with content (maxHeight 120)
       - Send button (arrow icon) on right, disabled when empty or streaming
       - Microphone button toggle (to be wired in Task 2)
       - onSubmit calls provided onSend callback
       - KeyboardAvoidingView behavior for iOS/Android

    6. Update `mobile/app/(tabs)/chat.tsx`:
       - Thread list view showing recent conversations
       - Each conversation shows title, last message preview, timestamp
       - Tap navigates to /chat/[threadId]
       - "New Chat" button in header
       - FlatList with pull-to-refresh

    7. Create `mobile/app/chat/[threadId].tsx`:
       - Full chat view for a single thread
       - FlatList inverted for bottom-up message rendering
       - ChatBubble for each message, StreamingText for current stream
       - ChatInput at bottom with KeyboardAvoidingView
       - Auto-scroll to bottom on new messages
       - Back button to return to thread list

    Wire voice recording into the chat input, using the existing /api/ai/voice endpoint for Whisper transcription.

    1. Create `mobile/src/lib/voice.ts`:
       - recordAndTranscribe function:
         a. Request microphone permission via AudioModule.requestRecordingPermissionsAsync()
         b. Create recorder with HIGH_QUALITY preset from expo-audio
         c. Start recording, return stop function
         d. On stop: get recording URI, create FormData, upload to /api/ai/voice
         e. Return { transcript, response } from API response
       - Uses expo-audio (NOT expo-av which is deprecated in SDK 55)

    2. Create `mobile/src/hooks/useVoice.ts`:
       - State machine: idle -> requesting_permission -> recording -> transcribing -> done/error
       - startRecording: transitions idle -> recording, creates recorder
       - stopAndSend: transitions recording -> transcribing, uploads audio, gets transcript
       - On success: calls onTranscript callback with transcribed text
       - On error: transitions to error state with message
       - Cleanup: stop recording on unmount
       - Follow the research code example closely but use proper state machine

    3. Update `mobile/src/components/ChatInput.tsx`:
       - Add microphone button next to send button
       - When mic button pressed: hide text input, show recording indicator (pulsing red dot + duration timer)
       - When recording, show stop button
       - On stop: show "Transcribing..." state, then populate text input with transcript
       - After transcript appears in input, user can edit before sending or tap send immediately
       - Long-press mic for push-to-talk behavior (release = stop + transcribe)
       - Haptic feedback on recording start/stop via expo-haptics
