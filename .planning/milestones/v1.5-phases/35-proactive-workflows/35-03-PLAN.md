  <action>
    1. Create `src/app/api/workflows/route.ts` (GET + POST):
       - **GET**: List all workflow rules for the authenticated user's org
         - `createRouteHandlerClient(cookies)` for auth
         - Get org_id from session via `supabase.auth.getUser()` -> lookup org membership
         - Query `workflow_rules` table: `SELECT * FROM workflow_rules WHERE org_id = ? ORDER BY created_at DESC`
         - Return `{ rules: WorkflowRule[] }`
       - **POST**: Create a new workflow rule from natural language
         - Accept body: `{ description: string, name?: string }` -- description is the NL input
         - Call `parseWorkflowRule(description, { roles: [...registeredRoles], tools: [...allToolNames] })`
         - If `needsReview`: return the parsed rule with `status: 'draft'` and confidence score for user confirmation
         - If confidence >= 0.5: insert into `workflow_rules` table with parsed trigger, conditions, actions
         - Auto-generate name from first 50 chars of description if not provided
         - Return `{ rule: WorkflowRule, confidence: number, needsReview: boolean }`

    2. Create `src/app/api/workflows/[id]/route.ts` (GET + PATCH + DELETE):
       - **GET**: Fetch single workflow rule by ID (with org_id check via RLS)
       - **PATCH**: Update workflow rule fields
         - Accept body: `{ enabled?: boolean, name?: string, description?: string }`
         - If `description` changed: re-parse via `parseWorkflowRule` and update trigger/conditions/actions
         - If only `enabled` changed: simple toggle update
         - Return updated `{ rule: WorkflowRule }`
       - **DELETE**: Soft-delete by setting `enabled = false` and marking as deleted, or hard-delete
         - Cancel any active workflows linked via `workflow_rule_id`
         - Return `{ success: true }`

    3. Create `src/app/api/workflows/[id]/runs/route.ts` (GET):
       - Fetch workflow runs (from `role_workflows` table) linked to this rule
       - Query: `SELECT * FROM role_workflows WHERE workflow_rule_id = ? ORDER BY started_at DESC LIMIT 50`
       - Return `{ runs: RoleWorkflow[] }` with status, steps, started_at, completed_at

    Follow the existing API route patterns from `api/settings/standing-orders/route.ts` for auth, error handling, and response format. Use try/catch with structured error responses `{ error: string }` and appropriate HTTP status codes.
