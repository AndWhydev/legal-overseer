The existing `onboarding-pipeline.ts` runs a FULL Opus synthesis ($2-5 per user) which is too heavy and slow for the 60-second first-run requirement. Create a lightweight discovery pipeline that:

1. Scans last 30 days (not 6 months)
2. Extracts top contacts, active threads, and user identity WITHOUT full Opus synthesis
3. Completes in under 60 seconds
4. Triggers the full Opus synthesis in the background AFTER the user proceeds

**Create `first-run-discovery.ts`:**

```typescript
export interface FirstRunDiscoveryResult {
  userIdentity: {
    name: string;
    email: string;
    company: string;
  };
  topContacts: Array<{
    name: string;
    email: string;
    messageCount: number;
    lastContact: string;
    relationship: 'frequent' | 'recent' | 'important';
  }>;
  activeThreads: Array<{
    subject: string;
    participants: string[];
    lastActivity: string;
    needsReply: boolean;
  }>;
  stats: {
    totalMessages: number;
    channelBreakdown: Record<string, number>;
    scanDurationMs: number;
  };
  insights: {
    emailsNeedingReply: number;
    overdueFollowUps: number;
    staleContacts: number;
    upcomingDeadlines: string[];
  };
}
```

Implementation approach:
1. Call `crawlAllChannels()` with `monthsBack: 1` and `maxPerChannel: 50` for speed
2. Extract user identity from email headers (From address on sent emails)
3. Build contact frequency map from all messages — top 10 by message count
4. Identify active threads by grouping messages by subject line (strip Re:/Fwd: prefixes) — top 10 most recent
5. For each active thread, check if the most recent message is from someone else (needsReply = true)
6. Count emails needing reply: received messages with no corresponding sent message to that sender in last 7 days
7. Identify overdue follow-ups: threads where user was last sender and no reply in 7+ days
8. DO NOT call Opus synthesis — this is pure data extraction
9. Upsert top contacts into the contacts table (name, email, org_id) so the dashboard has data immediately
10. Store discovery result in `profiles.preferences.first_run_discovery` for the welcome conversation

Target: < 60 seconds for accounts with < 500 emails in the last 30 days.

**Create `welcome-conversation.ts`:**

```typescript
export interface WelcomeMessageInput {
  userIdentity: FirstRunDiscoveryResult['userIdentity'];
  topContacts: FirstRunDiscoveryResult['topContacts'];
  activeThreads: FirstRunDiscoveryResult['activeThreads'];
  insights: FirstRunDiscoveryResult['insights'];
  connectedChannels: string[];
}

export function generateWelcomeMessage(input: WelcomeMessageInput): string
```

The welcome message should be a natural, specific, data-grounded message. NOT a generic "Welcome to BitBit!" but something like:

"I've scanned your last 30 days across Gmail. Here's what I found:

- **3 emails need replies** -- including one from Dave Johnson about the Henderson project proposal
- **Sarah Mitchell** has been your most active contact (14 messages this month)
- **The 'Website Redesign' thread** has gone quiet for 5 days -- want me to draft a follow-up?

I'll keep learning in the background. Ask me anything about your emails, contacts, or schedule."

Implementation rules:
1. If discovery found contacts: mention the top 1-2 by name with context
2. If discovery found threads needing reply: mention the most important one
3. If discovery found overdue follow-ups: mention the count and most notable
4. If discovery found nothing useful: fall back to "I'm connected to {channels} and ready to start learning"
5. Keep it under 150 words
6. Use markdown for formatting (bold for names, bullet points for items)
7. End with an actionable suggestion ("Ask me about..." or "Want me to draft a follow-up?")
8. Do NOT use an LLM call for this -- it's pure template logic with data interpolation. Speed matters.
