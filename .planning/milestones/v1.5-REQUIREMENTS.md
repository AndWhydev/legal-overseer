# v1.5 Beta Launch & First Revenue — Requirements

## Milestone Goal
Close every gap between the current product (feature-complete dogfood) and getting real paying users. Ship onboarding, verify production channels, build the marketing funnel, launch beta, and add premium features that justify pricing.

## Phase 30: Onboarding E2E & First-Run Experience

### Requirements

**ONBD-01: Onboarding Wizard Verification**
Verify all 12 FR requirements from T010 spec pass end-to-end. New user can complete signup → channel connection → first interaction in under 5 minutes.

**ONBD-02: First-Run Channel Discovery**
When a user connects their first email channel, BitBit automatically scans the last 30 days of email within 60 seconds and builds: user identity (name, role), key contacts (top 10 by frequency), active threads, and a "here's what I found" summary.

**ONBD-03: Empty State Guidance**
Every dashboard page (tasks, contacts, leads, invoices, inbox, analytics, roles) shows contextual guidance when empty — not blank screens. Each empty state explains what the page does and offers a one-click action to populate it.

**ONBD-04: Connection Wizard Flow**
Guided channel connection experience (not the raw settings page). Step-by-step: "Connect your email" → "Connect your calendar" → "Connect your messaging" → "You're set up." Each step is skippable. Progress is persisted.

**ONBD-05: Welcome Conversation**
After setup completes, BitBit initiates a welcome conversation that demonstrates capabilities using the user's real data: "We found 3 emails that need replies, 1 overdue invoice from Dave, and Steve's been quiet for 2 weeks."

## Phase 31: Channel Smoke Tests & Production Hardening

### Requirements

**CHAN-SMOKE-01: Gmail Live Pull**
Automated test: connect test Gmail account, send a test email to it, verify it appears in channel_messages within the poll interval (5 minutes).

**CHAN-SMOKE-02: Outlook Graph API**
Automated test: connect test Outlook account via Graph API, verify message pull and send work against production Microsoft tenant.

**CHAN-SMOKE-03: WhatsApp Bridge Stability**
Verify Baileys bridge at bitbit-wa-bridge.fly.dev maintains stable connection for 24+ hours. Send/receive test messages. Verify voice note transcription.

**CHAN-SMOKE-04: Telnyx SMS Round-Trip**
Automated test: send SMS via Telnyx, verify delivery webhook fires and message appears in channel_messages.

**CHAN-SMOKE-05: Concurrent Load**
10 simultaneous agent executions complete without connection pool exhaustion, timeout, or data corruption. Measure p50/p95/p99 latency.

**CHAN-SMOKE-06: Cron Resilience**
All 23 cron routes handle: DB connection failure (retry + DLQ), LLM timeout (circuit breaker), rate limit (backoff), and partial batch failure (continue with remaining) gracefully.

**CHAN-SMOKE-07: Monitoring Dashboard**
Production health dashboard showing: cron success rate, agent execution latency, channel sync status, error rate by route, token spend by org. Accessible from admin settings.

## Phase 32: Marketing Site & Checkout Flow

### Requirements

**MKTG-01: Product Landing Page**
Replace waitlist-only page at bitbit.chat with full product landing page: hero with value proposition, feature sections (roles, intelligence, channels), social proof (AWU case study), pricing CTA, footer with legal links.

**MKTG-02: Industry Pages**
Three industry-specific landing pages: Marketing Agencies, Trades & Services, Professional Services. Each highlights relevant roles, use cases, and pricing tier.

**MKTG-03: AWU Case Study**
Structured case study page with: problem, solution, results (real metrics from Andy's usage — messages triaged, invoices automated, leads captured, time saved). Quote from Andy.

**MKTG-04: Pricing Page with Checkout**
Interactive pricing page showing all tiers (Free, Starter, Growth, Scale). Feature comparison matrix. "Get Started" buttons create Stripe Checkout sessions and redirect. Current plan highlighted for logged-in users.

**MKTG-05: SEO Foundation**
Structured data (Organization, Product, FAQ), meta tags, Open Graph, sitemap.xml, robots.txt. Target keywords: "AI business assistant", "automated invoicing", "lead management AI".

## Phase 33: Beta Program Infrastructure

### Requirements

**BETA-01: Invite Flow**
Admin can select waitlist entries and send beta invites. Invite email contains unique setup link. Link creates account with beta plan (30-day trial of Growth tier).

**BETA-02: Guided Beta Onboarding**
Beta users get enhanced onboarding with: setup checklist, "BitBit is learning your world" progress indicator, daily tips for first week, direct feedback channel.

**BETA-03: Feedback Collection**
In-app feedback widget accessible from every page. Structured feedback: category (bug, feature request, confusion, praise), free text, optional screenshot. Stored in DB with user/org context.

**BETA-04: Usage Monitoring**
Admin dashboard showing per-org metrics: active days, messages processed, agent runs, token spend, channels connected, error count. Alerts when a beta user goes inactive for 3+ days.

**BETA-05: Beta User Limit**
System supports 10 concurrent beta orgs without performance degradation. Rate limits per org prevent one user from consuming excessive resources.

## Phase 34: Builder Role (Premium Differentiator)

### Requirements

**BUILD-01: Website Generation via Chat**
User says "build a landing page for Dave's Plumbing" and receives a deployable HTML page with: hero section, services, contact form, responsive design, SEO basics.

**BUILD-02: Template Library**
10+ industry-specific website templates: trades (plumber, electrician, builder), professional services (accountant, lawyer, dentist), agency (marketing, design, development). Each template is customizable via chat.

**BUILD-03: WordPress/Elementor Integration**
For agency users: generate Elementor-compatible page templates that can be imported into client WordPress sites. Export as .json or deploy via WordPress REST API.

**BUILD-04: Staging Preview**
Generated pages render in a preview panel before deployment. User can iterate ("make the hero bigger", "change the color to blue") via chat.

**BUILD-05: Plan Gating**
Builder role gated to Scale tier ($500+/month). Lower tiers see upgrade prompt with preview of what Builder can do.

## Phase 35: Proactive Workflows & Standing Orders

### Requirements

**FLOW-01: Workflow Definition**
Users can define automation rules in natural language: "When a new lead comes in, always draft a proposal within 1 hour." Rules stored as standing orders with trigger conditions and action chains.

**FLOW-02: Multi-Step Sequences**
Time-delayed action sequences: "Follow up with leads: day 1 intro email, day 3 value email, day 7 meeting request." Each step executes on schedule with skip conditions (if they replied, stop the sequence).

**FLOW-03: Cross-Role Orchestration**
Workflows can chain actions across roles: Sales detects high-quality lead → Comms drafts personalized reply → Finance prepares quote → all within one workflow.

**FLOW-04: Workflow Dashboard**
UI showing active workflows, execution history, success/failure rates, and manual override controls. Users can pause, edit, or delete workflows.

**FLOW-05: Standing Order Enhancement**
Upgrade existing standing orders from simple pattern matching to full workflow engine. Backward compatible with current standing order format.

## Phase 36: Mobile-First Experience

### Requirements

**MOBL-01: React Native App**
Cross-platform mobile app (iOS + Android) via Expo. Core screens: chat, inbox, tasks, approvals. Shared authentication with web app.

**MOBL-02: Push Notifications**
Push notifications for: approval requests, escalations, high-priority messages, completed actions. Actionable notifications (approve/reject from notification).

**MOBL-03: Voice Input**
Voice-first interaction: hold-to-talk, speech-to-text, BitBit processes and responds. Optimized for hands-busy scenarios (tradies on job sites).

**MOBL-04: Offline Queue**
When no network: queue outgoing messages and approvals locally. Sync when connection returns. Show clear offline indicator.

**MOBL-05: Quick Actions**
Home screen widget with 3-4 quick actions: "What's urgent?", "Approve all", "Check inbox", "Voice message". Launches directly into action without full app load.

## Success Criteria (Milestone Level)

1. A new user can sign up, connect Gmail, and have BitBit working within 5 minutes
2. All production channels verified working with real credentials
3. Stripe checkout converts waitlist signups to paying customers
4. 5+ beta users actively using BitBit within 2 weeks of launch
5. Builder role generates deployable websites from chat commands
6. At least one automation workflow runs unattended for 7 days
7. Mobile app installable and functional for chat + approvals
