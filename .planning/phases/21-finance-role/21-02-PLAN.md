# Plan 21-02: Proactive Invoicing + Collections

## Goal
Finance role proactively identifies billable work and generates invoices without being asked. Detects overdue invoices and runs escalating reminder workflows.

## Context
- Finance role registered in 21-01, wraps invoice agent
- Existing: `checkOverdueInvoices()` in invoice-sender.ts (detects overdue, returns list)
- Existing: workflow executor in role-runtime (startWorkflow, resumeWorkflow)
- Pattern: Time-delayed workflow steps for reminder sequences

## Tasks

### Task 1: Proactive Invoice Detection
Create `personal-assistant/src/lib/roles/finance/proactive-invoicing.ts`:

Scans for billable work that hasn't been invoiced:
- Projects with completed tasks but no recent invoice
- Time-tracked hours that haven't been billed
- Recurring services past their billing date

```typescript
export async function detectBillableWork(
  supabase: SupabaseClient,
  orgId: string,
  state: Record<string, unknown>,
): Promise<BillableItem[]>

export interface BillableItem {
  contactId: string
  contactName: string
  projectId?: string
  projectName?: string
  description: string
  estimatedAmount: number
  confidence: number
  lastInvoicedAt: string | null
}
```

Returns items as role insights (Observer) or draft invoice actions (Co-pilot/Autopilot).

### Task 2: Collection Reminder Workflow
Create `personal-assistant/src/lib/roles/finance/collection-workflow.ts`:

Defines the escalating reminder workflow:

```typescript
// Step 1: Gentle reminder (Day 7 overdue)
// Step 2: Firm reminder (Day 14 overdue)
// Step 3: Final notice (Day 30 overdue)
// Step 4: Escalation alert to user (Day 45 overdue)

export function createCollectionWorkflow(invoice: OverdueInvoice): WorkflowDefinition
export function getCollectionStepDefs(): WorkflowStepDef[]
```

Each step:
- Checks if invoice was paid since last step (condition: skip if paid)
- Drafts reminder email appropriate to severity level
- Routes through autonomy gate (Observer: insight, Co-pilot: draft for approval, Autopilot: send)
- Sets next_step_at for time delay

### Task 3: Integrate Proactive + Collections into Finance Evaluate
Update `finance-role.ts` evaluate():

1. Run wrapped invoice tick (existing behavior)
2. Detect billable work → generate insights or draft invoice actions
3. Check for newly overdue invoices → start collection workflows
4. Resume active collection workflows (via workflow executor)

### Task 4: Finance-Specific State Schema
Define what Finance stores in role_states.state JSONB:

```typescript
interface FinanceState {
  last_billable_scan_at: string | null
  last_overdue_check_at: string | null
  active_collection_workflows: string[] // workflow IDs
  known_payment_patterns: Record<string, { avgDays: number; count: number }>
  billable_items_surfaced: string[] // item hashes to avoid re-surfacing
}
```

## Verification
- [ ] Proactive invoicing detects unbilled completed projects
- [ ] Collection workflow starts when invoice becomes 7 days overdue
- [ ] Reminder steps are skipped if invoice is paid between steps
- [ ] Time delays between collection steps work (next_step_at)
- [ ] Observer mode: overdue invoices surface as insights, no reminders sent
- [ ] Co-pilot mode: reminder drafts queued for approval
- [ ] Autopilot mode: high-confidence reminders sent automatically
