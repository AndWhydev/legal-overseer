2. `getEffectiveThresholds(agentThresholds?: ConfidenceThresholds, orgThresholds?: ConfidenceThresholds): ConfidenceThresholds` — resolves threshold cascade: agent-level > org-level > defaults. For each field (act, ask), use the most specific non-undefined value.

3. `routeByConfidence(confidence: number, thresholds?: ConfidenceThresholds): ConfidenceRoutingResult` where:
```ts
export interface ConfidenceRoutingResult {
  decision: ConfidenceDecision  // 'act' | 'ask' | 'escalate'
  confidence: number
  thresholds: ConfidenceThresholds
  reasoning: string
}
```

Logic:
- `confidence >= thresholds.act` -> `'act'` with reasoning "Confidence {N} >= act threshold {T}"
- `confidence >= thresholds.ask` -> `'ask'` with reasoning "Confidence {N} between ask ({T1}) and act ({T2}) thresholds"
- `confidence < thresholds.ask` -> `'escalate'` with reasoning "Confidence {N} < ask threshold {T}"

4. `routeAgentAction(confidence: number, agentConfig?: { confidence_thresholds?: ConfidenceThresholds }, orgSettings?: { confidence_thresholds?: ConfidenceThresholds }): ConfidenceRoutingResult` — convenience function that resolves effective thresholds from agent config and org settings, then routes. This is what callers will typically use.

5. Validate thresholds: if act <= ask, log warning and use defaults. If values outside 0-1, clamp.

Import `ConfidenceThresholds` and `ConfidenceDecision` from `@bitbit/core` (re-exported in types.ts, already there).
