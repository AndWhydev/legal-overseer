<objective>
Extract CRUD operations into a shared tool system that any agent can import directly, independent of the Anthropic tool-calling interface.

Purpose: Currently, CRUD operations live inside `tools.ts` as Anthropic tool handlers (coupled to the chat interface). Agents running outside the chat loop (e.g., scheduled lead-swarm, invoice-flow) need the same DB operations without the Anthropic tool wrapper. Extract the core CRUD logic into `shared-tools.ts` and refactor `tools.ts` to delegate to it.

Output: `shared-tools.ts` with typed, org-scoped CRUD functions; `tools.ts` refactored to be a thin Anthropic wrapper over shared tools.
</objective>

Create `personal-assistant/src/lib/agent/shared-tools.ts`:

Extract the CRUD logic from `tools.ts` handlers into standalone typed functions. Each function takes explicit typed parameters (not `Record<string, unknown>`) and returns typed results.

Functions to extract:

1. **Tasks:**
   - `createTask(orgId: string, params: { title: string; description?: string; priority?: string; column?: string; contact_id?: string }): Promise<TaskResult>` — includes column resolution, position calculation, timeline write, relationship link (from current tools.ts)
   - `updateTask(orgId: string, taskId: string, params: { title?: string; description?: string; status?: string; column?: string; priority?: string }): Promise<TaskResult>` — includes timeline write for updates and completions
   - `searchTasks(orgId: string, params: { query?: string; status?: string; priority?: string }): Promise<SearchResult<Task>>` — with typed Task result

2. **Contacts:**
   - `getContact(orgId: string, slug: string): Promise<Contact | null>` — returns typed Contact from @bitbit/core
   - `searchContacts(orgId: string, query: string): Promise<ContactMatch[]>` — wraps resolveEntityRanked, returns matches with confidence

3. **Invoices:**
   - `createInvoice(orgId: string, params: { invoice_number: string; client_contact_id: string; items: InvoiceLineItem[]; due_date: string; currency?: string }): Promise<InvoiceResult>` — inserts into invoices table, calculates subtotal/tax/total
   - `updateInvoice(orgId: string, invoiceId: string, params: { status?: InvoiceStatus; paid_date?: string; payment_method?: string }): Promise<InvoiceResult>`
   - `searchInvoices(orgId: string, params: { status?: InvoiceStatus; client_contact_id?: string }): Promise<SearchResult<Invoice>>` — queries invoices table with filters

4. **Messages:**
   - `searchMessages(orgId: string, params: { channel?: string; contact_id?: string; query?: string; since?: string }): Promise<SearchResult<ChannelMessage>>` — queries channel_messages

5. **Activity:**
   - `logActivity(orgId: string, params: { action_type: string; action: string; reasoning?: string; result?: string }): Promise<{ id: string }>` — inserts into activity_feed

Shared types:
```ts
export interface CrudResult<T> { success: boolean; data?: T; error?: string }
export type TaskResult = CrudResult<{ id: string; title: string; [key: string]: unknown }>
export type InvoiceResult = CrudResult<Invoice>
export type SearchResult<T> = CrudResult<{ results: T[]; total: number }>
export interface ContactMatch { contact: Contact; matchConfidence: number; matchStep: string }
```

Then refactor `personal-assistant/src/lib/agent/tools.ts`:
- Import shared tool functions from `./shared-tools`
- Each handler in `tools.ts` becomes a thin wrapper: parse Record<string, unknown> input -> call typed shared function -> return ToolResult
- Keep `getAgentTools()` and `executeAgentTool()` signatures unchanged (backward compatible)
- The tool definitions (Anthropic schema) stay in tools.ts
- The CRUD logic moves to shared-tools.ts

Import context modules (resolveEntityRanked, writeTaskEvent, linkTaskToContact) in shared-tools.ts directly. The fire-and-forget pattern from Phase 3 applies to timeline writes.
