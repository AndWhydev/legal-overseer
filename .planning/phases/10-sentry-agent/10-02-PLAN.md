  <action>
    Build alert escalation and authenticated APIs using existing Supabase auth + org scoping patterns.

    1) Create sentry-escalation.ts with:
    - processSentryEscalations(supabase, orgId): find alerts where status in ('pending','escalated'), acknowledged_at is null, and next_escalation_at <= now.
    - For each due alert: create urgent approval_queue entry summarizing the incident and remediation suggestion; increment escalation_count; set escalated_at=now; set status='escalated'; set next_escalation_at += watch.escalation_minutes.
    - acknowledgeSentryAlert(supabase, alertId, userId): set status='acknowledged', acknowledged_by, acknowledged_at, clear next_escalation_at.
    - Never throw from batch processor: collect per-alert errors and return counts (processed/escalated/failed).

    2) Create /api/agent/sentry/watches route:
    - GET: list watches for org ordered by created_at desc.
    - POST: create watch with payload { watchType, description, conditions, intervalSeconds, escalationMinutes, escalationChannel }.
    - PATCH: update watch status ('active'|'paused') and editable fields.
    - DELETE: hard delete watch by id (org-scoped).
    - Validate required fields and reject invalid watch type / interval ranges with 400.

    3) Create /api/agent/sentry/alerts route:
    - GET: list recent alerts with status filter (default pending + escalated).
    - PATCH: acknowledge alert via { alertId, action: 'acknowledge' } and call acknowledgeSentryAlert.
    - POST: process escalation tick for org by calling processSentryEscalations (for scheduler/cron hooks).

    4) Reuse existing API auth helper pattern from approvals route (createClient, auth.getUser, profile org lookup).

    Add tests covering:
    - alert escalates when next_escalation_at is due and unacknowledged
    - acknowledged alert is excluded from escalation
    - escalation increments count and schedules next escalation window
    - watch and alert API invalid payload handling
  </action>
  <verify>Run `cd personal-assistant && npx vitest run src/lib/agent/sentry-escalation.test.ts`.</verify>
  <done>Unacknowledged alerts escalate on schedule into urgent notification records, alerts can be acknowledged, and watch/alert APIs provide authenticated CRUD + escalation endpoints.</done>
