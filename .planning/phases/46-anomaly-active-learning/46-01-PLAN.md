**anomaly_baselines table:**
```sql
CREATE TABLE IF NOT EXISTS anomaly_baselines (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id UUID NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
  entity_id UUID NOT NULL REFERENCES entity_nodes(id) ON DELETE CASCADE,
  metric_name TEXT NOT NULL CHECK (metric_name IN (
    'payment_timing', 'payment_amount', 'message_frequency', 'response_latency'
  )),
  mean FLOAT NOT NULL DEFAULT 0,
  stddev FLOAT NOT NULL DEFAULT 0,
  sample_count INT NOT NULL DEFAULT 0,
  last_computed TIMESTAMPTZ DEFAULT now(),
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE (org_id, entity_id, metric_name)
);
```

Indexes on anomaly_baselines:
- `idx_anomaly_baselines_entity` on `(org_id, entity_id)`
- `idx_anomaly_baselines_metric` on `(org_id, metric_name)` (for cross-entity aggregation queries)

RLS on anomaly_baselines: ENABLE ROW LEVEL SECURITY, then 4 policies (select/insert/update/delete) using `org_id = get_user_org_id()` following the exact pattern from `20260411000001` (policy names: `anomaly_baselines_select`, `anomaly_baselines_insert`, `anomaly_baselines_update`, `anomaly_baselines_delete`).

updated_at trigger: `CREATE TRIGGER trg_anomaly_baselines_updated_at BEFORE UPDATE ON anomaly_baselines FOR EACH ROW EXECUTE FUNCTION update_updated_at();`

**brain_alerts table:**
```sql
CREATE TABLE IF NOT EXISTS brain_alerts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id UUID NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
  entity_id UUID NOT NULL REFERENCES entity_nodes(id) ON DELETE CASCADE,
  alert_type TEXT NOT NULL CHECK (alert_type IN ('anomaly', 'pattern_break', 'learning_prompt')),
  metric_name TEXT,
  z_score FLOAT,
  baseline_text TEXT NOT NULL,
  explanation TEXT NOT NULL,
  severity TEXT NOT NULL DEFAULT 'medium' CHECK (severity IN ('low', 'medium', 'high', 'critical')),
  channel TEXT,
  dismissed_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

Indexes on brain_alerts:
- `idx_brain_alerts_entity_time` on `(org_id, entity_id, created_at DESC)` (for budget sliding window query)
- `idx_brain_alerts_type` on `(org_id, alert_type, created_at DESC)`

RLS on brain_alerts: same pattern, 4 policies (brain_alerts_select, brain_alerts_insert, brain_alerts_update, brain_alerts_delete).
