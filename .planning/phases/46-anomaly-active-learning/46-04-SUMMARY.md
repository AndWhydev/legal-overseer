## What Was Built

### Task 1: Anomaly hooks in brain-consolidation.ts (commit 62d251d0)

Modified `personal-assistant/src/lib/brain/brain-consolidation.ts`:

**Imports added:**
```typescript
import { detectAndAlertAnomalies, detectCrossEntityPatternBreaks } from './anomaly-detector'
import { resolveEntityByAlias } from '@/lib/knowledge-graph/graph-queries'
```

**BrainConsolidationReport interface extension (3 new fields):**
- `anomaliesDetected: number` -- total z-score |z|>=3 observations across all entities
- `alertsSent: number` -- number of alerts actually dispatched to user channels (post-budget, post-insert, post-dispatch)
- `crossEntityBreaks: number` -- pattern_break alerts generated by cross-entity aggregation

**Per-entity anomaly detection (inside dossier loop):**
After `processDomainJobDirect` succeeds and `updatedDomains.add(group.domain)` runs:
1. Resolve `group.entity_name` (string) to `entity_nodes` row via `resolveEntityByAlias`.
2. Re-fetch the full WAL entries for this entity group (`select('*').in('id', group.entry_ids)`) because the anomaly detector consumes full `KnowledgeLogEntry` objects, not just IDs.
3. Call `detectAndAlertAnomalies(supabase, orgId, entityNode.id, walEntries)`.
4. Accumulate `anomaliesDetected` and `alertsSent` into loop-level counters.
5. Wrap the entire block in its own inner `try/catch` with `logger.warn` -- anomaly failures do not abort dossier processing.

**Cross-entity aggregation (step 6b, between domain synthesis and WAL consolidation mark):**
```typescript
let crossEntityBreaks = 0
try {
  const crossResult = await detectCrossEntityPatternBreaks(supabase, orgId)
  crossEntityBreaks = crossResult.breaksDetected
  alertsSent += crossResult.alertsSent
} catch (err) {
  logger.warn('[brain-consolidation] Cross-entity aggregation failed', ...)
}
```

**Early-return path (empty entries):** Populates the 3 new report fields with 0 so the shape is stable.

**Final report object:** Includes `anomaliesDetected`, `alertsSent`, `crossEntityBreaks` alongside existing fields.
