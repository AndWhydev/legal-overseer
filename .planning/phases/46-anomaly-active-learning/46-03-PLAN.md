**Step 2 (GREEN): Modify routeByConfidence in confidence-router.ts.**

The clarify band sits in the upper portion of the ask band. The clarify threshold is calculated as:
```
clarifyThreshold = effective.ask + (effective.act - effective.ask) * 0.5
```

With default thresholds (act=0.85, ask=0.55), clarifyThreshold = 0.70. So:
- 0.70 to 0.85 -> `clarify` (upper ask band)
- 0.55 to 0.70 -> `ask` (lower ask band)
- below 0.55 -> `escalate`

Updated `routeByConfidence` function body:
```typescript
export function routeByConfidence(
  confidence: number,
  thresholds?: ConfidenceThresholds,
): ConfidenceRoutingResult {
  const effective = thresholds ?? DEFAULT_THRESHOLDS

  if (confidence >= effective.act) {
    return {
      decision: 'act',
      confidence,
      thresholds: effective,
      reasoning: `Confidence ${confidence} >= act threshold ${effective.act}`,
    }
  }

  if (confidence >= effective.ask) {
    // Clarify band: upper half of ask range
    const clarifyThreshold = effective.ask + (effective.act - effective.ask) * 0.5
    if (confidence >= clarifyThreshold) {
      return {
        decision: 'clarify',
        confidence,
        thresholds: effective,
        reasoning: `Confidence ${confidence} in clarify band (${clarifyThreshold.toFixed(2)}-${effective.act}) -- targeted question needed`,
      }
    }
    return {
      decision: 'ask',
      confidence,
      thresholds: effective,
      reasoning: `Confidence ${confidence} between ask (${effective.ask}) and clarify (${clarifyThreshold.toFixed(2)}) thresholds`,
    }
  }

  return {
    decision: 'escalate',
    confidence,
    thresholds: effective,
    reasoning: `Confidence ${confidence} < ask threshold ${effective.ask}`,
  }
}
```

No changes to `routeAgentAction` -- it calls `routeByConfidence` internally, so the clarify band propagates automatically. The `ConfidenceDecision` type already includes `'clarify'` (set in Plan 01).

     - System prompt: "You are a personal assistant asking a clarifying question. Generate a single conversational sentence that asks about the specific ambiguity. Reference the entity name and context naturally. Do not use meta-language like 'I need clarification' -- just ask the question directly."
     - Prompt: "Entity: {entityName}\nContext: {context}\nAmbiguity: {ambiguity}\nDossier: {dossierSummary ?? 'No additional context'}"
   - Returns the generated text trimmed, or null on error
   - Logs warning on error: `[active-learner] Question generation failed`

2. `async createClarificationWALEntry(supabase: SupabaseClient, orgId: string, entityIds: string[], userReply: string, originalContext: { question: string; entityName: string; ambiguity: string }): Promise<boolean>`
   - Inserts into knowledge_log:
     ```
     {
       org_id: orgId,
       entity_ids: entityIds,
       signal_type: 'clarification',
       content: `Clarification for ${originalContext.entityName}: Q: ${originalContext.question} A: ${userReply}`,
       confidence: 0.95,
       source_memory_id: null,
       source_thread_id: null,
     }
     ```
   - Returns true on success, false on error
   - Logs warning on error: `[active-learner] Failed to create clarification WAL entry`

3. `async fetchLearningPromptItems(supabase: SupabaseClient, orgId: string): Promise<string[]>`
   - Query entity_dossiers for all entities in org
   - For each entity, check `schema_json` for domain confidence scores (stored as `{ domain_confidence: { financial: 0.3, operational: 0.8, ... } }`)
   - For domains with confidence < 0.5:
     - Check brain_alerts for recent `learning_prompt` type alerts for this entity in last 7 days
     - If no recent learning prompt, generate prompt text: "I have limited understanding of {entityName}'s {domain} domain -- a quick update would help me serve you better"
   - Return array of prompt strings, max 5
   - Returns empty array on any error
