# Plan 22-01: Comms Role Implementation — Wraps Channel Triage

## Goal
Create the Comms role that wraps channel triage as a sub-component. All current triage capabilities work through the Comms role. Same "wrap, don't rewrite" pattern as Finance.

## Context
- Existing: `src/lib/agent/channel-triage.ts` (995 lines) — runTriage, classifyMessage, routeMessage
- Existing: `src/lib/agent/client-comms.ts` — runClientCommsTick (drafts, sends, queues)
- Finance role pattern: `src/lib/roles/finance/` (established in Phase 21)
- Role engine: RoleImplementation, registerRole, autonomy gate, action dispatcher

## Tasks

### Task 1: Comms Role Implementation
Create `personal-assistant/src/lib/roles/comms/comms-role.ts`:
- Implements RoleImplementation: evaluate(), hasChanges(), defaultConfig()
- evaluate(): runs wrapped triage tick + client comms tick
- hasChanges(): checks for new unprocessed messages since last tick
- defaultConfig(): 5-min tick (comms needs to be responsive), $5/day, copilot

### Task 2: Triage Wrapper
Create `personal-assistant/src/lib/roles/comms/triage-wrapper.ts`:
- Wraps runTriage() and runClientCommsTick()
- Translates TriageResult into RoleAction[] (route_message, draft_response) and RoleInsight[] (spam_filtered, informational)

### Task 3: Response Drafter
Create `personal-assistant/src/lib/roles/comms/response-drafter.ts`:
- Drafts contextual replies using conversation history + entity context
- Uses existing context assembler for entity-aware prompts
- Output: draft response queued through autonomy gate

### Task 4: Comms Role Index + Registration
Create `personal-assistant/src/lib/roles/comms/index.ts`
Update `personal-assistant/src/lib/roles/index.ts` — add comms import
