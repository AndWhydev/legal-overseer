# Plan 39-02: Heartbeat Monitor + Step Tracker + Retry Engine

<objective>
Build the heartbeat monitoring subsystem (cron-based orphan detection), per-step progress tracking with execution_steps writes, and the retry engine that calculates backoff delays and manages the retry cycle.
</objective>

<read_first>

- personal-assistant/src/lib/agent/tasks/task-service.ts (Supabase query patterns)
</read_first>

<action>
Create `personal-assistant/src/lib/agent/tasks/step-tracker.ts`:

Functions:
- `createSteps(supabase, taskId: string, steps: Array<{ step_name: string; input?: Record<string, unknown> }>): Promise<ExecutionStep[]>` -- batch insert steps with step_number 1..N
- `startStep(supabase, taskId: string, stepNumber: number): Promise<ExecutionStep | null>` -- transitions step pending->working, sets started_at, also updates parent task's current_step and progress_message
- `completeStep(supabase, taskId: string, stepNumber: number, output: Record<string, unknown>): Promise<ExecutionStep | null>` -- working->completed, sets output, completed_at, duration_ms. Updates parent task progress_pct = (completedSteps / totalSteps * 100)
- `failStep(supabase, taskId: string, stepNumber: number, errorMessage: string): Promise<ExecutionStep | null>` -- working->failed, sets error_message, completed_at
- `skipStep(supabase, taskId: string, stepNumber: number): Promise<ExecutionStep | null>` -- pending->skipped
- `getStepsForTask(supabase, taskId: string): Promise<ExecutionStep[]>` -- ordered by step_number

Each step mutation MUST also update the parent execution_task:
```typescript
// After completing a step:
const completedCount = steps.filter(s => s.status === 'completed').length
const totalCount = steps.length
const progressPct = Math.round((completedCount / totalCount) * 100)
const nextStep = steps.find(s => s.status === 'pending')

await supabase.from('execution_tasks').update({
  current_step: stepNumber,
  progress_pct: progressPct,
  progress_message: nextStep
    ? `Step ${stepNumber}/${totalCount} done. Next: ${nextStep.step_name}`
    : `All ${totalCount} steps complete`,
  heartbeat_at: new Date().toISOString(), // heartbeat on step progress
}).eq('id', taskId)
```
</action>

### Task 2: Retry Engine

<read_first>

</read_first>

<action>
Create `personal-assistant/src/lib/agent/tasks/retry-engine.ts`:

Functions:
- `shouldRetry(task: ExecutionTask): boolean` -- returns true if retry_count < max_retries and status is 'failed'
- `getRetryDelay(task: ExecutionTask): number` -- calls calculateRetryDelay with task's retry_policy fields and current retry_count
- `enqueueRetry(supabase, taskId: string): Promise<ExecutionTask | null>` -- atomically transitions failed->pending, increments retry_count, clears worker_id/claimed_at/heartbeat_at. Returns null if state already changed.
- `sendToDeadLetter(supabase, task: ExecutionTask): Promise<void>` -- writes terminal failure to DLQ via writeToDeadLetterQueue with task metadata (taskId, taskName, taskPayload, retryCount, lastError). Also updates task status column to remain 'failed' (no further transitions).

The retry engine does NOT handle scheduling the retry delay itself -- the caller (heartbeat monitor or task-service failTask) is responsible for waiting before calling enqueueRetry.
</action>

### Task 3: Heartbeat Monitor

<read_first>
- personal-assistant/src/lib/agent/tasks/task-service.ts (Supabase patterns)
- personal-assistant/src/lib/agent/tasks/retry-engine.ts (shouldRetry, enqueueRetry, sendToDeadLetter)
</read_first>

<action>
Create `personal-assistant/src/lib/agent/tasks/heartbeat-monitor.ts`:

Function:
- `detectAndRecoverOrphans(supabase, opts?: { staleThresholdMs?: number }): Promise<{ recovered: number; deadLettered: number }>` -- 
  1. Query tasks WHERE status IN ('claimed', 'working') AND heartbeat_at < now() - staleThresholdMs (default 90000ms = 90s)
  2. For each orphaned task:
     a. Set error_message = 'Worker heartbeat lost — task orphaned'
     b. Transition to 'failed' via atomic update (WHERE status IN ('claimed', 'working'))
     c. If shouldRetry(task): enqueueRetry(supabase, task.id) → increment recovered count
     d. If not retryable: sendToDeadLetter(supabase, task) → increment deadLettered count
  3. Return counts

Log each recovery and dead-letter via logger.info/warn.
</action>

### Task 4: Cron Endpoint for Heartbeat Monitor

<read_first>
- personal-assistant/src/app/api/cron/relay/route.ts (existing cron route pattern — guards, admin client)
</read_first>

<action>
Create `personal-assistant/src/app/api/cron/task-heartbeat/route.ts`:

```typescript
import { NextResponse } from 'next/server'
import { createAdminClient } from '@/lib/supabase/admin'
import { detectAndRecoverOrphans } from '@/lib/agent/tasks/heartbeat-monitor'
import { logger } from '@/lib/core/logger'

export const runtime = 'nodejs'
export const maxDuration = 30

export async function GET(request: Request) {
  // Verify cron secret
  const authHeader = request.headers.get('authorization')
  if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
    return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
  }

  try {
    const supabase = createAdminClient()
    const result = await detectAndRecoverOrphans(supabase)
    
    logger.info('[cron/task-heartbeat] Orphan scan complete', result)
    
    return NextResponse.json({
      ok: true,
      recovered: result.recovered,
      deadLettered: result.deadLettered,
    })
  } catch (error) {
    logger.error('[cron/task-heartbeat] Failed', error)
    return NextResponse.json({ error: 'Internal error' }, { status: 500 })
  }
}
```

Add to vercel.json crons array:
```json
{ "path": "/api/cron/task-heartbeat", "schedule": "* * * * *" }
```
(Runs every minute.)
</action>
