<title>Integration tests for tier feedback loop</title>
<read_first>
- personal-assistant/src/lib/agent/engine/tool-resolver.ts (context injection, outcome recording)
- personal-assistant/src/lib/agent/engine/reliability-tracker.ts (data layer)
- personal-assistant/src/lib/agent/engine/tool-executor.ts (where recording happens)
- personal-assistant/src/lib/agent/engine/__tests__/taor-loop.test.ts (existing integration test patterns)
</read_first>
<action>
Create `personal-assistant/src/lib/agent/engine/__tests__/tier-feedback-loop.test.ts` with integration tests:

1. **CHAIN-01: Multi-tier fallback flow**
   - Simulate: API tool fails ظْ `recordToolOutcome` called with `success: false` ظْ reliability data updated ظْ `buildTierContextBlock` includes the failure data ظْ model context would show low API reliability for that service
   - Verify the data flows correctly from recording through aggregation to context

2. **CHAIN-02: Service-to-tier knowledge in context**
   - Seed reliability data for multiple services (xero=api 96%, linkedin=browser 40%, stripe=api 100%)
   - Call `buildTierContextBlock()`
   - Verify returned context string contains all three services with correct tiers and success rates
   - Verify context contains the tier description text (structured API, browser, workspace, human)

3. **CHAIN-03: Reliability tracking and automatic tier escalation signal**
   - Record 10 executions for "xero" on "api" tier: 8 success, 2 failure
   - Query `getReliabilitySummary()` ظْ verify `success_rate` is 0.80
   - Record 5 more failures ظْ verify `success_rate` drops
   - Verify the model context accurately reflects the degraded reliability
   - This doesn't test "automatic escalation" directly (the model decides), but verifies the data signal is correct

4. **CHAIN-04: Human handoff in the tier chain**
   - Simulate: browser tier fails for linkedin ظْ `recordToolOutcome` records failure ظْ `request_human_handoff` called ظْ approval record created with `action_type: 'human_handoff'` ظْ `recordToolOutcome` records human tier usage
   - Verify reliability data shows both browser failure and human success for linkedin

5. **Cold start scenario**
   - Call `buildTierContextBlock()` with no reliability data
   - Verify it still returns tier descriptions (model uses base knowledge)
   - Verify it does NOT include an empty reliability table (clean output)

6. **Error resilience**
   - Simulate Supabase connection failure during `recordToolOutcome`
   - Verify tool execution is not blocked (fire-and-forget recording)
   - Simulate Supabase failure during `buildTierContextBlock`
   - Verify it returns empty string (graceful degradation), does not throw
</action>
