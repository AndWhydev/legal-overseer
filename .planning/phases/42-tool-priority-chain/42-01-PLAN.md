# Plan 42-01: Reliability Tracker

<objective>
Build the execution reliability tracking system that records success/failure per service per tier and surfaces aggregated reliability data for model context injection. This is the "learning signal" that makes ToolResolver improve over time (CONTEXT.md D-05, D-06).
</objective>

<read_first>

<action>
Create a new Supabase migration file with:

1. `execution_reliability` table:
```sql
CREATE TABLE execution_reliability (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  org_id UUID NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
  service_name TEXT NOT NULL,          -- e.g., "xero", "linkedin", "stripe", "gmail"
  tier TEXT NOT NULL CHECK (tier IN ('api', 'browser', 'workspace', 'human')),
  success BOOLEAN NOT NULL,
  error_category TEXT,                  -- e.g., "auth_expired", "rate_limited", "dom_changed", "timeout"
  error_message TEXT,                   -- truncated error for debugging
  latency_ms INTEGER,
  cost_estimate_cents INTEGER DEFAULT 0, -- rough cost tracking per execution
  task_id UUID,                          -- references execution_tasks if async
  tool_name TEXT,                        -- the specific tool used (e.g., "send_email", "spawn_browser_agent")
  metadata JSONB DEFAULT '{}',          -- extra context (URL, action type, etc.)
  created_at TIMESTAMPTZ DEFAULT NOW() NOT NULL
);

CREATE INDEX idx_exec_reliability_org_service ON execution_reliability(org_id, service_name, created_at DESC);
CREATE INDEX idx_exec_reliability_org_tier ON execution_reliability(org_id, tier, created_at DESC);
```

2. RLS policies:
```sql
ALTER TABLE execution_reliability ENABLE ROW LEVEL SECURITY;
CREATE POLICY "org_members_read_reliability" ON execution_reliability
  FOR SELECT USING (org_id IN (SELECT org_id FROM org_members WHERE user_id = auth.uid()));
CREATE POLICY "service_role_all_reliability" ON execution_reliability
  FOR ALL USING (auth.role() = 'service_role');
```

3. Aggregation view for last 7 days:
```sql
CREATE OR REPLACE VIEW execution_reliability_summary AS
SELECT
  org_id,
  service_name,
  tier,
  COUNT(*) AS sample_count,
  COUNT(*) FILTER (WHERE success) AS success_count,
  ROUND(COUNT(*) FILTER (WHERE success)::numeric / NULLIF(COUNT(*), 0), 2) AS success_rate,
  ROUND(AVG(latency_ms))::integer AS avg_latency_ms,
  ROUND(AVG(cost_estimate_cents))::integer AS avg_cost_cents,
  MODE() WITHIN GROUP (ORDER BY error_category) FILTER (WHERE NOT success) AS most_common_error
FROM execution_reliability
WHERE created_at > NOW() - INTERVAL '7 days'
GROUP BY org_id, service_name, tier;
```
</action>

<action>
Create `personal-assistant/src/lib/agent/engine/reliability-tracker.ts` with:

1. **`recordExecution()`** function:
```typescript
export interface ExecutionRecord {
  orgId: string
  serviceName: string
  tier: 'api' | 'browser' | 'workspace' | 'human'
  success: boolean
  errorCategory?: string
  errorMessage?: string
  latencyMs?: number
  costEstimateCents?: number
  taskId?: string
  toolName?: string
  metadata?: Record<string, unknown>
}

export async function recordExecution(
  supabase: SupabaseClient,
  record: ExecutionRecord
): Promise<void>
```
Insert into `execution_reliability` table. Fire-and-forget (catch errors, log, don't throw).

2. **`getReliabilitySummary()`** function:
```typescript
export interface ReliabilitySummary {
  serviceName: string
  tier: string
  sampleCount: number
  successRate: number
  avgLatencyMs: number
  avgCostCents: number
  mostCommonError: string | null
}

export async function getReliabilitySummary(
  supabase: SupabaseClient,
  orgId: string
): Promise<ReliabilitySummary[]>
```
Query `execution_reliability_summary` view filtered by org_id. Return empty array on error.

3. **`formatReliabilityContext()`** function:
```typescript
export function formatReliabilityContext(summaries: ReliabilitySummary[]): string
```
Formats reliability data as a markdown table for system prompt injection:
```
## Execution Reliability (last 7 days)
| Service | Tier | Success Rate | Samples | Avg Latency | Common Error |
|---------|------|-------------|---------|-------------|--------------|
| xero | api | 96% | 48 | 450ms | - |
| linkedin | browser | 40% | 5 | 12000ms | dom_changed |
```
Returns empty string if no data.

4. **`inferServiceName()`** helper:
```typescript
export function inferServiceName(toolName: string, input: Record<string, unknown>): string
```
Maps tool names to service names:
- `send_email` / `send_gmail` → "gmail"
- `send_outlook` → "outlook"
- Composio tools → extract service from tool name prefix
- `spawn_browser_agent` → extract from URL domain in input
- `spawn_ephemeral_workspace` → "workspace"
- Default: use tool name as service name
</action>
