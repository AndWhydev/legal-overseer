  <name>Task 1: WordPress REST API client</name>
  <files>src/lib/builder/wordpress-client.ts</files>
  <action>
Create `src/lib/builder/wordpress-client.ts` -- a typed WordPress REST API v2 client:

1. `WordPressClient` class:
   - Constructor takes `{ siteUrl: string, username: string, applicationPassword: string }`. WordPress Application Passwords are the standard auth method (available since WP 5.6). Store base URL as `${siteUrl}/wp-json/wp/v2`.
   - All methods use `fetch()` with `Authorization: Basic ${btoa(username + ':' + applicationPassword)}` header.
   - Include `User-Agent: BitBit-Builder/1.0` header.
   - All methods handle WP REST API error responses (status codes, WP error objects with `code` and `message` fields).

2. Methods:
   a. `async testConnection(): Promise<{ success: boolean; siteName: string; wpVersion: string }>`:
      - GET `/` (site root endpoint) to verify credentials work
      - Return site name and WP version from response

   b. `async createPage(data: { title: string; content: string; status?: 'draft' | 'publish'; slug?: string }): Promise<WPPage>`:
      - POST `/pages` with title, content (HTML), status (default 'draft'), slug
      - Return the created page object

   c. `async updatePage(pageId: number, data: Partial<{ title: string; content: string; status: string }>): Promise<WPPage>`:
      - PUT `/pages/${pageId}`
      - Return updated page

   d. `async getPages(params?: { per_page?: number; search?: string; status?: string }): Promise<WPPage[]>`:
      - GET `/pages` with query params
      - Return page list

   e. `async uploadMedia(file: Buffer, filename: string, mimeType: string): Promise<WPMedia>`:
      - POST `/media` with FormData containing the file
      - Return the media object with source_url

   f. `async getPlugins(): Promise<WPPlugin[]>`:
      - GET `/plugins` (requires admin auth)
      - Used to detect if Elementor is installed

   g. `async checkElementor(): Promise<{ installed: boolean; version: string | null }>`:
      - Call getPlugins() and look for 'elementor/elementor.php' in the list
      - Return whether Elementor is detected

3. Type definitions (in the same file):
   - `WPPage`: `{ id: number, title: { rendered: string }, content: { rendered: string }, slug: string, status: string, link: string }`
   - `WPMedia`: `{ id: number, source_url: string, title: { rendered: string } }`
   - `WPPlugin`: `{ plugin: string, name: string, status: string, version: string }`
   - `WordPressConfig`: `{ siteUrl: string, username: string, applicationPassword: string }`

4. Factory function:
   - `createWordPressClient(config: WordPressConfig): WordPressClient`

Note: Do NOT store credentials in plaintext. The `api_key_ref` in DeploymentTarget is meant to reference an encrypted credential stored in org_integrations or a secrets manager. For now, the client accepts credentials directly -- credential management will be wired in the deployment tool (Plan 04).

1. Create `src/lib/builder/elementor-export.ts` -- HTML to Elementor JSON converter:

   Elementor stores page content as a JSON structure of sections > columns > widgets. The converter parses common HTML patterns into this structure.

   a. Define Elementor JSON types:
      - `ElementorWidget`: `{ id: string, elType: 'widget', widgetType: string, settings: Record<string, unknown> }`
      - `ElementorColumn`: `{ id: string, elType: 'column', settings: Record<string, unknown>, elements: ElementorWidget[] }`
      - `ElementorSection`: `{ id: string, elType: 'section', settings: Record<string, unknown>, elements: ElementorColumn[] }`
      - `ElementorDocument`: `ElementorSection[]`

   b. `htmlToElementorJson(html: string): ElementorDocument`:
      - This is a best-effort converter, not a full HTML parser. Uses regex/string parsing for common patterns:
        - `<section>` or `<div class="section">` blocks -> ElementorSection
        - `<h1>`-`<h6>` -> heading widget with `{ title, header_size, align }`
        - `<p>` text blocks -> text-editor widget with `{ editor: htmlContent }`
        - `<img>` -> image widget with `{ image: { url } }`
        - `<a>` buttons (detected by class containing 'btn' or 'button') -> button widget
        - `<ul>/<ol>` -> text-editor widget containing the list HTML
        - `<form>` -> placeholder HTML widget (Elementor forms need the Pro plugin)
      - Each element gets a unique 7-char hex ID (Elementor standard)
      - Sections without clear column structure default to a single full-width column
      - Inline styles are extracted and mapped to Elementor settings where possible (background_color, text_color, padding, margin)
      - CSS variables in `<style>` blocks are resolved by extracting `:root` variable declarations first

   c. `elementorJsonToMeta(doc: ElementorDocument): string`:
      - Serializes the ElementorDocument to the `_elementor_data` meta value format (JSON string)
      - This is what gets stored in WordPress postmeta for Elementor to render

   d. Export all types and functions.

2. Create `src/lib/builder/deploy.ts` -- deployment pipeline orchestrator:

   a. `async deployToWordPress(projectId: string, orgId: string, supabase: SupabaseClient): Promise<DeployResult>`:
      - Load the website_project from DB (verify it belongs to orgId)
      - Verify project has html_content and deploy_target with type 'wordpress'
      - Load WordPress credentials from deploy_target config. For now, expect `deploy_target.credentials` to contain `{ username, applicationPassword }` (future: resolve from org_integrations encrypted store via api_key_ref).
      - Create a WordPressClient
      - Test the connection -- if it fails, return error with helpful message
      - Check if Elementor is installed on the target site
      - If Elementor detected AND `deploy_target.elementor_compatible`:
        - Convert HTML to Elementor JSON via `htmlToElementorJson()`
        - Create a WP page with a minimal content wrapper
        - Set `_elementor_data` and `_elementor_edit_mode: 'builder'` postmeta via a raw REST call: `POST /wp-json/wp/v2/pages/{id}` with `meta: { _elementor_data: jsonString, _elementor_edit_mode: 'builder' }`
      - If no Elementor (standard WP):
        - Create a WP page with the full HTML as content
        - Set page template to 'elementor_canvas' if available, else blank/full-width
      - Update website_projects: status='deployed', deployed_at=now(), deployed_url=page.link
      - Return `{ success: true, pageId, pageUrl, elementorUsed: boolean }`
      - On any error: return `{ success: false, error: string }`, do not update project status

   b. `DeployResult` interface:
      - `{ success: boolean; pageId?: number; pageUrl?: string; elementorUsed?: boolean; error?: string }`

   c. `async checkDeploymentStatus(projectId: string, orgId: string, supabase: SupabaseClient): Promise<{ status: DeploymentStatus; url: string | null; deployedAt: string | null }>`:
      - Simple DB query on website_projects to return current deployment state

   d. Import logger from '@/lib/core/logger' for all error/info logging.
