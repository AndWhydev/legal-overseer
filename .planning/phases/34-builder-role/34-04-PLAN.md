1. Create `src/app/api/builder/preview/[projectId]/route.ts`:
   - GET handler that serves the generated HTML as a standalone webpage
   - Load the website_project from Supabase by projectId
   - Verify the project exists and has html_content
   - Auth: Check if the requesting user belongs to the project's org (via Supabase auth). If the project status is 'preview' or 'deployed', also allow unauthenticated access (public preview link) so the URL can be shared with clients.
   - Return the HTML as `text/html` content type with proper headers:
     - `Content-Type: text/html; charset=utf-8`
     - `X-Frame-Options: SAMEORIGIN` (allow embedding in our own iframe but not external sites)
     - `Cache-Control: no-cache` (preview should always show latest)
   - If project not found: return 404 with a simple HTML error page
   - If no html_content: return 404 with "Site is still being generated"
   - Update the website_project's preview_url to the canonical preview URL: `${process.env.NEXT_PUBLIC_APP_URL}/api/builder/preview/${projectId}`

2. Add 2 new tools to `src/lib/agent/tools/builder-tools.ts`:

   a. `deploy_website` tool:
      - Description: "Deploy a generated website to a connected WordPress site. Requires the project to have a deploy_target configured. The site will be pushed as a WordPress page (with Elementor formatting if Elementor is detected on the target site). Returns the live URL."
      - Input schema: `{ project_id: string (required), publish?: boolean (default false -- creates as draft. Set true to publish immediately) }`
      - Handler:
        - Call `deployToWordPress(project_id, orgId, supabase)`
        - If deploy_target is not configured, return helpful error: "This project doesn't have a WordPress site connected. Ask the user to configure the WordPress connection first with their site URL and application password."
        - Return result with deployed URL or error message

   b. `preview_website` tool:
      - Description: "Get the preview URL for a generated website. The preview is a live, sharable link that shows the current state of the site."
      - Input schema: `{ project_id: string (required) }`
      - Handler:
        - Construct preview URL: `${process.env.NEXT_PUBLIC_APP_URL}/api/builder/preview/${project_id}`
        - Also load the project to return status and metadata
        - Return `{ preview_url, status, name, created_at }`

3. Update `src/lib/agent/tools.ts`:
   - Add 'deploy_website' and 'preview_website' to the builder tool group's tools array
   - Add JIT_INSTRUCTIONS:
     - `deploy_website`: "Website deployed. Confirm the live URL and whether it was published or saved as a draft. If Elementor was used, mention that the client can edit it in the Elementor editor. If deployment failed, explain the error clearly and suggest next steps."
     - `preview_website`: "Share the preview URL with the user. Mention they can share this link with clients for review. Format as a clickable link."
