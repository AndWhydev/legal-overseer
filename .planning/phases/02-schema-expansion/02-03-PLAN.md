**013_templates_voices.sql** — Communication templates and voice profiles.

Two tables in one migration (they're closely related and both small).

Map from `VoiceProfile` and `Template` TypeScript interfaces:

```sql
-- Voice profiles first (templates reference them)
CREATE TABLE voice_profiles (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id uuid REFERENCES organizations ON DELETE CASCADE NOT NULL,
  name text NOT NULL,  -- 'Andy', 'Tor-AWU', 'Formal Client'
  description text NOT NULL DEFAULT '',
  tone text NOT NULL DEFAULT 'professional',  -- 'friendly', 'professional', 'formal', 'casual'
  formality text NOT NULL DEFAULT 'professional' CHECK (formality IN ('casual', 'professional', 'formal')),
  greeting_patterns text[] DEFAULT '{}',  -- ['Hi {name}', 'Hey {name}']
  sign_off_patterns text[] DEFAULT '{}',  -- ['Cheers', 'Best regards']
  emoji_usage text NOT NULL DEFAULT 'minimal' CHECK (emoji_usage IN ('none', 'minimal', 'moderate', 'heavy')),
  example_messages jsonb DEFAULT '{}',  -- { "follow_up": "Hey mate, just checking in...", "invoice": "Hi {name}, please find attached..." }
  do_patterns text[] DEFAULT '{}',  -- ['Use Australian English', 'Keep it brief']
  dont_patterns text[] DEFAULT '{}',  -- ['No corporate jargon', 'Never use "Dear Sir/Madam"']
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now(),
  UNIQUE(org_id, name)
);

CREATE TABLE templates (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id uuid REFERENCES organizations ON DELETE CASCADE NOT NULL,
  name text NOT NULL,
  category text NOT NULL CHECK (category IN ('email', 'proposal', 'invoice', 'onboarding', 'follow-up', 'notification')),
  subject_template text,  -- 'Invoice {{invoice_number}} from {{company_name}}'
  body_template text NOT NULL,
  voice_profile_id uuid REFERENCES voice_profiles ON DELETE SET NULL,
  channel text,  -- 'email', 'whatsapp', null for any
  variables jsonb DEFAULT '{}',  -- { "client_name": "Client's full name", "amount": "Invoice total" }
  usage_count integer NOT NULL DEFAULT 0,
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now(),
  UNIQUE(org_id, name)
);
```

Indexes for voice_profiles:
- `idx_voice_profiles_org` on (org_id)

Indexes for templates:
- `idx_templates_org_cat` on (org_id, category)
- `idx_templates_voice` on (voice_profile_id) WHERE voice_profile_id IS NOT NULL

Add updated_at triggers for both tables.

**016_contacts_enhancements.sql** — Add columns to existing contacts table.

The contacts table (from 001_core_schema.sql) needs additional columns for agent intelligence. Use ALTER TABLE ADD COLUMN (not CREATE TABLE). The TypeScript Contact interface in types.ts already has these fields defined.

```sql
-- Add agent-intelligence columns to contacts
ALTER TABLE contacts ADD COLUMN IF NOT EXISTS lead_score text CHECK (lead_score IN ('hot', 'warm', 'cold'));
ALTER TABLE contacts ADD COLUMN IF NOT EXISTS lifetime_value numeric(12,2);
ALTER TABLE contacts ADD COLUMN IF NOT EXISTS last_interaction_at timestamptz;
ALTER TABLE contacts ADD COLUMN IF NOT EXISTS preferred_channel text;
ALTER TABLE contacts ADD COLUMN IF NOT EXISTS voice_profile_id uuid REFERENCES voice_profiles ON DELETE SET NULL;
ALTER TABLE contacts ADD COLUMN IF NOT EXISTS tags text[] DEFAULT '{}';
```

Note: Use IF NOT EXISTS for idempotency since this modifies an existing table.

Indexes:

- `idx_contacts_tags` GIN on (tags)
- `idx_contacts_voice` on (voice_profile_id) WHERE voice_profile_id IS NOT NULL
- `idx_contacts_last_interaction` on (org_id, last_interaction_at DESC) WHERE last_interaction_at IS NOT NULL
