```

  recommendedActions: string[] // e.g., ["reply", "create_task", "forward_to_lead_swarm"]

  category: 'lead' | 'client' | 'vendor' | 'personal' | 'spam' | 'notification' | 'newsletter'
```

Export `classifyMessage(supabase: SupabaseClient, message: ChannelMessage, orgId: string): Promise<ClassificationResult>`:
- Build a classification prompt that includes:
  - Message sender, subject, body (truncated to 2000 chars)
  - Instruction to return JSON with significance (1-10), timeSensitivity, recommendedActions, reasoning, category
  - Scoring guidelines: 10=business-critical (contract, legal, payment), 7-9=important client/lead, 4-6=routine business, 1-3=newsletter/spam/noise

  - Parse JSON from response text content block
- Store classification result on the channel_messages row:
  - `supabase.from('channel_messages').update({ significance, time_sensitivity, recommended_actions, classification_model: 'haiku', classified_at: new Date().toISOString() }).eq('id', message.id)`
- Return the ClassificationResult
- Wrap entire function in try/catch -- on failure, return a default low-significance result rather than throwing

Create `action-router.ts`:

Export `RoutingDecision` type: `'immediate' | 'queue' | 'batch' | 'skip'`

Export `MessageRoute` interface:
```
{
  decision: RoutingDecision
  reason: string
  targetAgent?: string    // e.g., 'lead-swarm', 'invoice-flow', 'sentry'
  priority: number        // 1-10 (from significance)
  batchWindow?: number    // minutes to wait before processing batch
}
```

Export `routeMessage(classification: ClassificationResult): MessageRoute`:
- Pure function (no DB, no async) -- deterministic routing from classification
- Routing rules:
  - `significance >= 8 AND timeSensitivity === 'immediate'` → immediate (process now)
  - `significance >= 6 AND timeSensitivity IN ('immediate', 'today')` → queue (process soon, in order)
  - `significance >= 4 AND timeSensitivity IN ('today', 'this_week')` → batch (group and process, batchWindow: 30)
  - `significance >= 4 AND timeSensitivity IN ('whenever', 'none')` → batch (batchWindow: 120)
  - `significance < 4 OR category === 'spam' OR category === 'newsletter'` → skip
- Target agent mapping from category:
  - 'lead' → 'lead-swarm'
  - 'client' with recommendedActions containing 'invoice' → 'invoice-flow'
  - category has error/alert keywords in recommendedActions → 'sentry'
  - Default: null (no specific agent, general processing)

Export `routeMessages(supabase: SupabaseClient, messages: ChannelMessage[], orgId: string): Promise<RoutedMessage[]>`:
- For each message: classifyMessage, then routeMessage, combine into RoutedMessage
- RoutedMessage = ChannelMessage & { classification: ClassificationResult, route: MessageRoute }
- Mark processed messages: update channel_messages SET processed = true WHERE id IN (...)
- Return array of RoutedMessage sorted by priority DESC
