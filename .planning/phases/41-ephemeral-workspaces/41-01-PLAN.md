# Plan 41-01: E2B Provider + Workspace Tool Registration

<objective>
Create the E2B sandbox provider module and register `spawn_ephemeral_workspace` as a TAOR tool. After this plan, BitBit can provision a Firecracker microVM via E2B's API and the tool is wired into the agent's tool dispatch pipeline.
</objective>

<must_haves>
- E2B provider module that wraps `@e2b/code-interpreter` SDK
- `spawn_ephemeral_workspace` tool definition registered in the tool system
- Tool handler that provisions an E2B sandbox and returns a workspace_id
- Types for workspace sessions, workspace config, and provider interface
- Unit tests with mocked E2B SDK
</must_haves>

<action>
Create `personal-assistant/src/lib/workspaces/e2b-provider.ts`:

```typescript
import { Sandbox } from '@e2b/code-interpreter'
import { logger } from '@/lib/core/logger'
import type {
  WorkspaceProvider,
  WorkspaceConfig,
  WorkspaceExecResult,
  WorkspaceLanguage,
} from './types'

const DEFAULT_TIMEOUT_MS = 5 * 60 * 1000 // 5 minutes

export class E2BProvider implements WorkspaceProvider {
  private sandboxes = new Map<string, Sandbox>()

  async create(config: WorkspaceConfig): Promise<{ sandboxId: string }> {
    const apiKey = process.env.E2B_API_KEY
    if (!apiKey) throw new Error('E2B_API_KEY environment variable required')

    logger.info('[workspace] Creating E2B sandbox', {
      template: config.template || 'default',
      timeoutMs: config.timeoutMs || DEFAULT_TIMEOUT_MS,
    })

    const sandbox = await Sandbox.create({
      timeoutMs: config.timeoutMs || DEFAULT_TIMEOUT_MS,
      metadata: config.metadata,
      apiKey,
    })

    const sandboxId = sandbox.sandboxId
    this.sandboxes.set(sandboxId, sandbox)

    logger.info('[workspace] E2B sandbox created', { sandboxId })
    return { sandboxId }
  }

  async exec(
    sandboxId: string,
    code: string,
    language: WorkspaceLanguage,
  ): Promise<WorkspaceExecResult> {
    const sandbox = this.sandboxes.get(sandboxId)
    if (!sandbox) throw new Error(`Sandbox ${sandboxId} not found or already destroyed`)

    logger.info('[workspace] Executing code', { sandboxId, language, codeLength: code.length })

    if (language === 'shell') {
      const result = await sandbox.commands.run(code)
      return {
        stdout: result.stdout,
        stderr: result.stderr,
        exitCode: result.exitCode,
      }
    }

    // Python or JavaScript via code interpreter
    const execution = await sandbox.runCode(code, { language })

    return {
      stdout: execution.logs.stdout.join('\n'),
      stderr: execution.logs.stderr.join('\n'),
      exitCode: execution.error ? 1 : 0,
      error: execution.error?.message,
      artifacts: execution.results?.map((r, i) => ({
        type: 'code' as const,
        name: `result-${i}`,
        content: typeof r === 'string' ? r : JSON.stringify(r),
      })),
    }
  }

  async uploadFile(sandboxId: string, path: string, content: Buffer | string): Promise<void> {
    const sandbox = this.sandboxes.get(sandboxId)
    if (!sandbox) throw new Error(`Sandbox ${sandboxId} not found`)
    await sandbox.files.write(path, content)
  }

  async downloadFile(sandboxId: string, path: string): Promise<Buffer> {
    const sandbox = this.sandboxes.get(sandboxId)
    if (!sandbox) throw new Error(`Sandbox ${sandboxId} not found`)
    const content = await sandbox.files.read(path)
    return Buffer.from(content)
  }

  async listFiles(sandboxId: string, path: string): Promise<string[]> {
    const sandbox = this.sandboxes.get(sandboxId)
    if (!sandbox) throw new Error(`Sandbox ${sandboxId} not found`)
    const entries = await sandbox.files.list(path)
    return entries.map((e) => e.name)
  }

  async destroy(sandboxId: string): Promise<void> {
    const sandbox = this.sandboxes.get(sandboxId)
    if (!sandbox) {
      logger.warn('[workspace] Sandbox not found for destroy', { sandboxId })
      return
    }
    await sandbox.kill()
    this.sandboxes.delete(sandboxId)
    logger.info('[workspace] E2B sandbox destroyed', { sandboxId })
  }
}
```

Note: The E2B SDK API may differ slightly from training data. The executor should check `@e2b/code-interpreter` installed version and adjust:
- `Sandbox.create()` may use `template` parameter for custom images
- `sandbox.runCode()` may use `onStdout`/`onStderr` callbacks for streaming
- `sandbox.commands.run()` for shell execution
- `sandbox.files.write()`/`sandbox.files.read()` for filesystem ops
- `sandbox.kill()` for cleanup (may also be `sandbox.close()`)

Read the installed package's type definitions to confirm exact API surface.
