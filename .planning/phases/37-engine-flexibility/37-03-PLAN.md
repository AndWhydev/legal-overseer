# Plan 37-03: LTV-Aware Dynamic Cost Guard Budget Scaling

<objective>
Extend the cost guard's `checkRoleBudget` and `canProceed` functions to accept an optional LTV multiplier that scales daily token budgets and cost limits for high-value entities. Default multiplier is 1.0 (no change). Cap at 10x to prevent runaway spend.
</objective>

## Tasks

<task id="37-03-01">
<title>Add ltvMultiplier parameter to cost guard functions</title>
<read_first>
- personal-assistant/src/lib/agent/cost-guard.ts (full file — canProceed, checkRoleBudget, getExecutionTokenCap signatures)
</read_first>
<action>
In `personal-assistant/src/lib/agent/cost-guard.ts`:

1. Add a constant for max LTV multiplier after `DEFAULT_DAILY_LIMIT`:
```typescript
/** Maximum LTV multiplier to prevent runaway cost scaling */
const MAX_LTV_MULTIPLIER = 10.0
```

2. Add a helper to clamp the multiplier:
```typescript
/** Clamp LTV multiplier to [0.1, MAX_LTV_MULTIPLIER], defaulting to 1.0 */
function effectiveLtvMultiplier(multiplier?: number): number {
  if (multiplier == null) return 1.0
  return Math.max(0.1, Math.min(MAX_LTV_MULTIPLIER, multiplier))
}
```

3. Extend `canProceed` to accept an optional `ltvMultiplier`:
```typescript
export async function canProceed(
  supabase: SupabaseClient,
  orgId: string,
  ltvMultiplier?: number,
): Promise<CostGuardResult> {
```
Inside the function, after reading `dailyLimit` from org_settings, apply the multiplier:
```typescript
  dailyLimit = dailyLimit * effectiveLtvMultiplier(ltvMultiplier)
```

4. Extend `checkRoleBudget` to accept an optional `ltvMultiplier`:
```typescript
export async function checkRoleBudget(
  supabase: SupabaseClient,
  orgId: string,
  role: string,
  ltvMultiplier?: number,
): Promise<RoleBudgetResult> {
```
After looking up `config` from `ROLE_BUDGET_CONFIG`, apply the multiplier to the budget values:
```typescript
  const ltv = effectiveLtvMultiplier(ltvMultiplier)
  const scaledDailyBudget = Math.round(config.dailyTokenBudget * ltv)
  const scaledMaxPerExecution = Math.round(config.maxTokensPerExecution * ltv)
```
Use `scaledDailyBudget` in place of `dailyTokenBudget` for the budget check and `scaledMaxPerExecution` is not directly used in this function but should be stored for reference. The comparison `dailyUsed >= dailyTokenBudget` becomes `dailyUsed >= scaledDailyBudget`, and the warning threshold uses `scaledDailyBudget * warningThresholdPct`.

5. Extend `getExecutionTokenCap` to accept an optional `ltvMultiplier`:
```typescript
export function getExecutionTokenCap(role: string, ltvMultiplier?: number): number | undefined {
  const config = ROLE_BUDGET_CONFIG[role]
  if (!config) return undefined
  return Math.round(config.maxTokensPerExecution * effectiveLtvMultiplier(ltvMultiplier))
}
```

All parameters are optional — existing callers without ltvMultiplier get identical behavior (multiplier = 1.0).
</action>
