  <action>
    Create `scripts/verify-oauth-credentials.ts` -- a standalone script (runnable with `npx tsx scripts/verify-oauth-credentials.ts`) that validates all OAuth credentials are properly configured.

    The script should check each provider in sequence and report pass/fail:

    **Google (Gmail/Calendar):**
    - Check `GOOGLE_CLIENT_ID` exists and matches pattern `*.apps.googleusercontent.com`
    - Check `GOOGLE_CLIENT_SECRET` exists and starts with `GOCSPX-`
    - Attempt token info endpoint: `GET https://oauth2.googleapis.com/tokeninfo` (just verify endpoint reachable)
    - Check `NEXT_PUBLIC_APP_URL` is set (needed for redirect URI)

    **Microsoft (Outlook):**
    - Check `OUTLOOK_CLIENT_ID` exists and is UUID format
    - Check `OUTLOOK_CLIENT_SECRET` exists
    - Check `OUTLOOK_TENANT_ID` exists (optional -- can be "common")
    - Attempt OpenID config fetch: `GET https://login.microsoftonline.com/common/v2.0/.well-known/openid-configuration`

    **Asana:**
    - Check `ASANA_CLIENT_ID` exists
    - Check `ASANA_CLIENT_SECRET` exists

    **Calendly:**
    - Check `CALENDLY_CLIENT_ID` exists
    - Check `CALENDLY_CLIENT_SECRET` exists

    **WhatsApp Bridge:**
    - Check `WHATSAPP_BRIDGE_URL` env var (URL of deployed bridge)
    - If set, fetch `{WHATSAPP_BRIDGE_URL}/health` and report bridge status

    **General:**
    - Check `NEXT_PUBLIC_APP_URL` is HTTPS (not localhost)
    - Check `RELAY_SECRET` is set
    - Check `CRON_SECRET` is set

    Output format: colorized table with provider name, check name, status (PASS/FAIL/WARN), detail.
    Exit code 0 if all critical checks pass, 1 if any FAIL.

    Load env from `personal-assistant/.env.local` if it exists (using dotenv), otherwise use process.env.

<task type="auto">
  <name>Task 2: Create channel smoke test script</name>
  <files>scripts/channel-smoke-test.ts</files>
  <action>
    Create `scripts/channel-smoke-test.ts` -- a standalone script that exercises the live channel endpoints to verify credentials are working in the deployed environment.

    The script takes the deployed app URL as argument: `npx tsx scripts/channel-smoke-test.ts https://app.bitbit.chat`

    **Tests (run sequentially, continue on failure):**

    1. **Health check**: `GET {url}/api/monitoring/health` -- verify app is running
    2. **Channel status**: `GET {url}/api/channels/status` (with auth cookie or service key) -- verify all channels report status
    3. **Gmail OAuth start**: `GET {url}/api/auth/oauth/start?provider=gmail` -- verify redirect URL is generated (don't follow redirect, just check 302 location contains accounts.google.com)
    4. **Outlook OAuth start**: `GET {url}/api/auth/oauth/start?provider=outlook` -- verify redirect URL contains login.microsoftonline.com
    5. **Relay daemon trigger**: `POST {url}/api/channels/relay` with `Authorization: Bearer {RELAY_SECRET}` -- verify relay runs (check for X-Relay-Duration-Ms header)
    6. **Token refresh cron**: `POST {url}/api/cron/token-refresh` with `Authorization: Bearer {CRON_SECRET}` -- verify token refresh runs
    7. **WhatsApp bridge health** (if WHATSAPP_BRIDGE_URL set): `GET {bridge_url}/health` -- verify bridge is connected

    **Output format:**
    - Colorized pass/fail for each test
    - Summary at end: X/Y passed
    - JSON report written to `scripts/smoke-test-results.json`

    **Auth handling:**
    - Use `CRON_SECRET` for cron endpoints
    - Use `RELAY_SECRET` for relay endpoint
    - For channel status, use Supabase service role key to make direct API call
    - All secrets loaded from env or `personal-assistant/.env.local`

    Handle network errors gracefully -- timeout after 10s per request, report as FAIL with error detail.
  </action>
