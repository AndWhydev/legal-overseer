  <how-to-verify>
    Complete these steps in order:

    **Step 1: Google Cloud (Gmail)**
    1. Follow runbook Section 1 to create OAuth credentials in Google Cloud Console
    2. Set `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET` in Vercel env vars
    3. Add redirect URI `https://{APP_DOMAIN}/callback/gmail` in Google Cloud Console

    **Step 2: Microsoft Azure (Outlook)**
    1. Follow runbook Section 2 to register app in Azure Portal
    2. Set `OUTLOOK_CLIENT_ID`, `OUTLOOK_CLIENT_SECRET`, `OUTLOOK_TENANT_ID` in Vercel env vars
    3. Add redirect URI `https://{APP_DOMAIN}/callback/outlook` in Azure Portal
    4. Grant admin consent for Mail.Read, Mail.Send, offline_access

    **Step 3: Asana**
    1. Follow runbook Section 3 to create developer app
    2. Set `ASANA_CLIENT_ID` and `ASANA_CLIENT_SECRET` in Vercel env vars
    3. Add redirect URI `https://{APP_DOMAIN}/callback/asana`

    **Step 4: Calendly**
    1. Follow runbook Section 4 to create OAuth application
    2. Set `CALENDLY_CLIENT_ID` and `CALENDLY_CLIENT_SECRET` in Vercel env vars
    3. Add redirect URI `https://{APP_DOMAIN}/callback/calendly`

    **Step 5: Deploy WhatsApp Bridge**
    1. Follow runbook Section 5 to deploy bridge to Fly.io
    2. Set WHATSAPP_BRIDGE_URL in Vercel env vars pointing to bridge URL

    **Step 6: Redeploy Vercel**
    1. Trigger a Vercel redeploy to pick up new env vars
    2. Verify deployment succeeds

    **Step 7: Run credential verification**
    ```bash
    npx tsx scripts/verify-oauth-credentials.ts
    ```
    All checks should show PASS.
