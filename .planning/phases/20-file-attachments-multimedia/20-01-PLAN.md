1. **Attachments table:**
```sql
CREATE TABLE IF NOT EXISTS attachments (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id UUID NOT NULL REFERENCES organizations(id),
  user_id UUID NOT NULL REFERENCES auth.users(id),
  thread_id UUID REFERENCES conversation_threads(id),
  message_id UUID REFERENCES conversation_messages(id),
  filename TEXT NOT NULL,
  mime_type TEXT NOT NULL,
  size BIGINT NOT NULL,
  storage_path TEXT NOT NULL,
  status TEXT NOT NULL DEFAULT 'uploading'
    CHECK (status IN ('uploading', 'ready', 'processing', 'failed', 'deleted')),
  extracted_text TEXT,
  metadata JSONB DEFAULT '{}',
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

2. **RLS policies** using existing `get_user_org_id()` pattern:
- `attachments_org_read` FOR SELECT USING (org_id = get_user_org_id())
- `attachments_org_insert` FOR INSERT WITH CHECK (org_id = get_user_org_id())
- `attachments_org_update` FOR UPDATE USING (org_id = get_user_org_id())

3. **Indexes:**
- `idx_attachments_org` on (org_id) WHERE status != 'deleted'
- `idx_attachments_thread` on (thread_id) WHERE status = 'ready'
- `idx_attachments_message` on (message_id) WHERE message_id IS NOT NULL

```sql
CREATE OR REPLACE FUNCTION get_org_storage_bytes(p_org_id UUID)
RETURNS BIGINT AS $$
  SELECT COALESCE(SUM(size), 0)
  FROM attachments
  WHERE org_id = p_org_id AND status != 'deleted'
$$ LANGUAGE sql STABLE SECURITY DEFINER;
```

**constants.ts** -- Define all attachment validation constants:
- `ALLOWED_MIME_TYPES`: Set of allowed MIME types (image/jpeg, image/png, image/gif, image/webp, application/pdf, application/vnd.openxmlformats-officedocument.wordprocessingml.document, text/plain, text/csv)
- `MAX_FILE_SIZE`: 10 * 1024 * 1024 (10MB)
- `MAX_FILES_PER_MESSAGE`: 5
- `BLOCKED_EXTENSIONS`: Set of blocked extensions (.exe, .sh, .bat, .cmd, .com, .msi, .dll, .scr, .ps1)
- `UPLOAD_URL_EXPIRY`: 7200 (2 hours, for signed upload URLs)
- `DOWNLOAD_URL_EXPIRY`: 3600 (1 hour, for signed download URLs)
- `STORAGE_BUCKET`: 'chat-attachments'
- Export a `validateFile(filename: string, mimeType: string, size: number)` function that returns `{ valid: boolean, error?: string }` checking all three constraints (size, MIME type, extension)

**attachment-service.ts** -- Upload orchestration service with three exported functions:
1. `createUploadUrl(supabase, { orgId, userId, threadId, filename, mimeType, size })`:
   - Call `validateFile()` from constants, throw on invalid
   - Generate storage path: `${orgId}/${threadId || 'unthreaded'}/${crypto.randomUUID()}/${filename}`
   - Call `supabase.storage.from(STORAGE_BUCKET).createSignedUploadUrl(storagePath)`
   - Insert row into `attachments` table with status='uploading'
   - Return `{ attachmentId, signedUrl, token, path }`

2. `confirmUpload(supabase, attachmentId: string, orgId: string)`:
   - Update `attachments` row SET status='ready', updated_at=now() WHERE id=attachmentId AND org_id=orgId
   - Return the updated attachment record

3. `getDownloadUrl(supabase, attachmentId: string, orgId: string)`:
   - Select attachment record WHERE id=attachmentId AND org_id=orgId AND status='ready'
   - Generate signed download URL: `supabase.storage.from(STORAGE_BUCKET).createSignedUrl(storagePath, DOWNLOAD_URL_EXPIRY)`
   - Return `{ signedUrl, attachment }` (attachment record for metadata)

**upload/route.ts** -- POST endpoint:
- Auth: `createClient()` -> `auth.getUser()` -> profile lookup for org_id
- Parse JSON body: `{ filename, mimeType, size, threadId }`
- Call `createUploadUrl()` from service
- Return 200 with `{ attachmentId, signedUrl, token, path }`
- Return 400 for validation errors, 401 for unauth, 500 for storage errors

**[id]/route.ts** -- PATCH and GET endpoints:
- PATCH: Auth check -> call `confirmUpload(supabase, id, orgId)` -> return 200 with attachment record
- GET: Auth check -> call `getDownloadUrl(supabase, id, orgId)` -> return 200 with `{ signedUrl, filename, mimeType, size }`
