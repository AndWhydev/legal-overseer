# Plan 20-03: Autonomy Gate & Approval Integration

## Goal
Build the autonomy gate — the layer between role decisions and action execution that routes based on the role's autonomy level (Observer/Co-pilot/Autopilot).

## Context
- Existing: `routeAgentAction()` in `confidence-router.ts` — cascade of thresholds → act/ask/escalate
- Existing: `queueAgentAction()` in `approval-queue.ts` — creates approval if not `act`
- Existing: `createApproval()` with notifications (dashboard + WhatsApp)
- New: Autonomy level adds a layer BEFORE confidence routing

## Design

The autonomy gate sits between role evaluation and execution:

```
Role.evaluate() → RoleAction[]
        ↓
  [Autonomy Gate]
        ↓
  OBSERVER:  → log as insight to role_activity (no action taken)
  CO-PILOT:  → always queue to approval_queue (regardless of confidence)
  AUTOPILOT: → check confidence via routeAgentAction():
               act → execute immediately
               ask → queue to approval_queue
               escalate → log escalation to role_activity
```

Key insight: **Autopilot doesn't skip confidence routing** — it just removes the "always ask" gate. The existing confidence routing still protects against low-confidence autonomous actions.

## Tasks

### Task 1: Autonomy Gate Function
Create `personal-assistant/src/lib/roles/autonomy-gate.ts`:

```typescript
export type GateDecision = 'execute' | 'queue_approval' | 'log_insight' | 'escalate'

export interface GateResult {
  decision: GateDecision
  reasoning: string
  autonomyLevel: AutonomyLevel
  confidenceRouting?: ConfidenceRoutingResult  // only for autopilot
}

/**
 * Route a role action through the autonomy gate.
 *
 * Observer: always log_insight
 * Co-pilot: always queue_approval
 * Autopilot: delegate to confidence routing (act → execute, ask → queue, escalate → escalate)
 */
export function routeThroughAutonomyGate(
  action: RoleAction,
  autonomyLevel: AutonomyLevel,
  agentConfig?: { confidence_thresholds?: Partial<ConfidenceThresholds> },
  orgSettings?: { confidence_thresholds?: Partial<ConfidenceThresholds> },
  agentType?: string,
  calibratedThresholds?: CalibratedThresholds | null,
): GateResult
```

### Task 2: Action Dispatcher
Create `personal-assistant/src/lib/roles/action-dispatcher.ts`:

Dispatches role actions based on gate decision:

```typescript
export interface DispatchResult {
  actionId: string
  gateResult: GateResult
  approvalId?: string  // if queued
  executionResult?: unknown  // if executed
  activityId: string  // always logged
}

/**
 * Dispatch a role action: gate it, then execute/queue/log based on decision.
 */
export async function dispatchRoleAction(
  supabase: SupabaseClient,
  roleConfig: RoleConfig,
  action: RoleAction,
): Promise<DispatchResult>
```

Flow:
1. Call `routeThroughAutonomyGate(action, roleConfig.autonomy_level, ...)`
2. Based on decision:
   - `execute`: Call existing action execution (reuse `executeApprovedAction` pattern)
   - `queue_approval`: Call `createApproval()` with `autonomy_mode` in context_snapshot
   - `log_insight`: Insert to `role_activity` with type 'insight'
   - `escalate`: Insert to `role_activity` with type 'escalation', notify user
3. Always log to `role_activity` with reasoning chain
4. Return result

## Verification
- [ ] Observer mode: role evaluates but NEVER creates approval or executes action
- [ ] Co-pilot mode: ALL actions go to approval queue regardless of confidence
- [ ] Autopilot mode: high-confidence actions execute, low-confidence go to approval
- [ ] Changing autonomy level via API takes effect on next tick
- [ ] Every action (regardless of mode) logs to role_activity with reasoning
- [ ] Approval records include role_config_id and autonomy_mode
- [ ] Output formatting differs per autonomy level
