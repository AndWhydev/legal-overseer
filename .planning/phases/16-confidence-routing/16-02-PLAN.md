**Enhance confidence-harness.ts:**

Add `analyzeModelTierBehavior(scenarios: ConfidenceScenario[]): ModelTierAnalysis` that:
- Simulates how different model tiers affect confidence scoring by applying tier-specific adjustments:
  - Haiku: scores tend +-0.08 variance (less precise), apply jitter of -0.05 to +0.05 systematically
  - Sonnet: scores tend +-0.04 variance (baseline, no adjustment)
  - Opus: scores tend +-0.02 variance (most precise), apply jitter of -0.02 to +0.02
- For each tier, runs the full scenario set through routeByConfidence with per-agent thresholds
- Returns `ModelTierAnalysis`:
  - `byTier`: Record<ModelTier, { accuracy: number; falsePositives: number; falsePositiveRate: number; decisionChanges: number }>
  - `stabilityScore`: number (% of scenarios where all 3 tiers agree on decision)
  - `riskyCases`: scenarios where Haiku says 'act' but Sonnet/Opus say 'ask' or 'escalate'

Add `measureFalsePositives(report: ConfidenceHarnessReport): FalsePositiveAnalysis` that extracts:
- `totalAutoActions`: number (scenarios where routing decided 'act')
- `incorrectAutoActions`: number (decided 'act' but expected 'ask' or 'escalate')
- `falsePositiveRate`: number
- `details`: array of the specific false positive scenarios with full context
- `recommendation`: string -- "PASS: FP rate X% below 5% threshold" or "FAIL: FP rate X% exceeds 5%"
