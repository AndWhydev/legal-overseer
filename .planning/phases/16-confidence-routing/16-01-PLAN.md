Create `confidence-harness.ts` that:

1. Exports `ConfidenceHarnessReport` interface:
   - `totalScenarios`: number
   - `correct`: number
   - `accuracy`: number (0-1)
   - `byAgent`: Record<string, { total: number; correct: number; accuracy: number }>
   - `byCategory`: Record<string, { total: number; correct: number; accuracy: number }>
   - `falsePositives`: number (predicted 'act' when expected 'ask' or 'escalate')
   - `falsePositiveRate`: number
   - `misses`: Array<{ scenarioId: string; expected: string; actual: string; confidence: number }>
   - `runAt`: string

2. Exports `runConfidenceHarness(scenarios?: ConfidenceScenario[]): ConfidenceHarnessReport`:
   - Defaults to AWU_SCENARIOS if not provided
   - For each scenario, calls `routeAgentAction(scenario.confidenceScore, undefined, undefined)` with the agent-type-aware thresholds via `getAgentThresholds(scenario.agentType)`
   - Actually: use `routeByConfidence(scenario.confidenceScore, getAgentThresholds(scenario.agentType))`
   - Compares `result.decision` to `scenario.expectedDecision`
   - Tallies correct/incorrect, groups by agent type and category
   - Counts false positives: cases where routing says 'act' but expected was 'ask' or 'escalate'
   - Records all misses with full detail
   - Returns the report

3. Exports `formatHarnessReport(report: ConfidenceHarnessReport): string` that produces a human-readable summary table.
