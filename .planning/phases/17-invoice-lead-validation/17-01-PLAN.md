   - If `candidates.length >= 2` AND the top two matches have confidence within 0.1 of each other AND both are below 0.7 — return `{ resolved: null, error: 'ambiguous_contact' }`
   - Otherwise proceed with the best match as before

2. Modify `detectDuplicateInvoice` to use fuzzy matching:

   a. **Normalize project reference** — add a `normalizeProjectReference(value: string): string` function that:
      - Lowercases
      - Removes common suffixes: 'work', 'project', 'job', 'updates', 'changes'
      - Collapses whitespace
      - Strips trailing punctuation
      - Example: "White House RE work" → "white house re", "White House Real Estate" → "white house real estate"

   b. **Fuzzy project match** — instead of `.eq('project_reference', normalizedProject)`, query recent invoices for the same contact (last 30 days) and compare normalized project references. Use a simple containment check: if normalized A contains normalized B or vice versa, treat as match.

   d. **Time window** — only consider invoices created within the last 30 days (add `.gte('created_at', thirtyDaysAgo)` to the query).

   e. The query changes from exact match to: fetch recent invoices for this contact in this org (last 30 days, not cancelled), then filter in JS for fuzzy project + amount match.
