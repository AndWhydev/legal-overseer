       - `interface WorkflowToolBridge { resolveTool(toolGroup: ToolGroup, toolName: string): boolean; executeTool(toolGroup: ToolGroup, toolName: string, params: Record<string, unknown>): Promise<WorkflowStepResult> }`
       - `createWorkflowToolBridge(supabase: SupabaseClient, orgId: string): WorkflowToolBridge` factory function
       - `resolveTool`: checks `TOOL_GROUPS[toolGroup]?.tools.includes(toolName)` -- returns true if tool exists in registry
       - `executeTool`:
         a. Call `canProceed(supabase, orgId)` -- if not ok, return `{ success: false, error: reason }`
         b. Call `executeAgentTool(toolName, params, orgId, supabase)` -- reuses existing tool execution with plan gating, autonomy, etc.

       - `executeWorkflowStep(bridge: WorkflowToolBridge, action: WorkflowAction, ctx: WorkflowStepContext): Promise<WorkflowStepResult>` -- top-level function that:
         a. Validates tool exists via bridge.resolveTool
         b. Evaluates action.condition (if present) against ctx.stepResults -- simple NL condition eval not needed for v1, just check if referenced step_id result was successful
         c. Calls bridge.executeTool
         d. Handles on_failure: 'skip' returns success, 'abort' throws, 'retry' retries once
       - `ruleToWorkflowDefinition(rule: WorkflowRule, bridge: WorkflowToolBridge, orgId: string): WorkflowDefinition` -- converts a WorkflowRule's actions array into a WorkflowDefinition compatible with the existing startWorkflow(). Each action becomes a WorkflowStepDef where execute() calls executeWorkflowStep().
