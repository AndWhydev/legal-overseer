    - evaluateEventTriggers finds all enabled rules matching event type, evaluates conditions, and returns matched rule IDs
    - evaluateEventTriggers skips events with triggered_by_workflow=true (loop prevention)
    - evaluateScheduledTriggers finds rules with schedule triggers whose interval has elapsed since last_triggered_at
    - Both functions update last_triggered_at and increment trigger_count on match
  </behavior>
  <action>
    1. Create `workflow-rule-parser.ts`:
       - `parseWorkflowRule(naturalLanguage: string, orgContext: { roles: string[], tools: string[] }): Promise<{ rule: Partial<WorkflowRule>, confidence: number, needsReview: boolean }>`
       - Use Anthropic SDK with `claude-haiku-4-20250414` model for fast, cheap parsing
       - System prompt includes: available tool groups (from TOOL_GROUPS keys), available tools per group (from TOOL_GROUPS[g].tools), supported event types (from SUPPORTED_EVENTS), example input/output pairs
       - Parse LLM JSON response with `WorkflowRuleSchema.safeParse()` -- if validation fails, return `{ confidence: 0, needsReview: true }` with the raw parsed data in the rule
       - Include `confidence` field in LLM output schema (0-1) -- rules with confidence < 0.5 get `needsReview: true`
       - Export `RULE_PARSER_SYSTEM_PROMPT` for testing

    2. Create `workflow-rule-engine.ts`:
       - `getActiveWorkflowRules(supabase: SupabaseClient, orgId: string, triggerType?: string): Promise<WorkflowRule[]>` -- fetches enabled rules, optionally filtered by trigger_type
       - `evaluateEventTriggers(supabase: SupabaseClient, orgId: string, event: WorkflowEvent): Promise<string[]>` -- returns matched rule IDs:
         a. Skip if `event.triggered_by_workflow === true` (loop prevention)
         b. Fetch enabled rules with trigger_type='event' for this org
         c. Filter to rules where trigger_config.event matches event.event
         d. Evaluate each rule's conditions against event.data (field/operator/value matching)
         e. For matched rules: update last_triggered_at, increment trigger_count
         f. Return matched rule IDs (actual workflow creation happens in Plan 02 when tool bridge exists)
       - `evaluateScheduledTriggers(supabase: SupabaseClient, orgId: string): Promise<string[]>` -- returns matched rule IDs:
         a. Fetch enabled rules with trigger_type='schedule'
         b. For interval-based: check if (now - last_triggered_at) >= interval_seconds
         c. For cron-based: simple check if current time matches cron pattern (hour/minute matching, no full cron parser needed for v1 -- support daily at HH:MM format)
         d. Update last_triggered_at and trigger_count for matched rules
         e. Return matched rule IDs
       - `evaluateConditions(conditions: WorkflowCondition[], data: Record<string, unknown>): boolean` -- pure function, AND-logic across all conditions. Operators: eq (===), neq (!==), contains (String.includes), gt (>), lt (<).
       - Helper: `matchesCronPattern(pattern: string, now: Date): boolean` -- supports "HH:MM" daily pattern and "* */N * * *" interval patterns. Not a full cron parser.
