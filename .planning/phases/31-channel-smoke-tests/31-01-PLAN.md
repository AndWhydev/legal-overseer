**SmokeTestRunner (smoke-test-runner.ts):**

```typescript
export interface ChannelSmokeResult {
  channel: ChannelType
  status: 'pass' | 'fail' | 'skip'
  latencyMs: number
  message: string
  error?: string
  testedAt: string
}

export interface SmokeTestReport {
  overall: 'pass' | 'fail' | 'partial'
  channels: ChannelSmokeResult[]
  duration_ms: number
  testedAt: string
}
```

- `runAllSmokeTests(supabase: SupabaseClient, orgId: string): Promise<SmokeTestReport>` -- runs all 4 channel checks in parallel using `Promise.allSettled`:
  1. Gmail: calls `checkAdapterHealth(gmailAdapter)` + attempts `pull()` with org credentials from `getOrgCredential(supabase, orgId, 'gmail')`
  2. Outlook: calls `checkAdapterHealth(outlookAdapter)` + attempts `pull()` with org credentials from `getOrgCredential(supabase, orgId, 'outlook')`
  3. WhatsApp: fetches bridge health endpoint + calls `checkWhatsAppSession(supabase, orgId)`
  4. SMS: validates Telnyx API key via env var check + sends test ping if `SMOKE_TEST_PHONE` set

- Each channel test is wrapped in try/catch with 15-second timeout. Never throws -- always returns a result.
- `overall` is 'pass' if all non-skipped channels pass, 'fail' if any fail, 'partial' if some skip.
- Runner reuses existing `checkAdapterHealth` from `health.ts` where possible.
- Export `runChannelSmoke(channel: ChannelType, supabase, orgId)` for individual channel testing.

This runner is called by the monitoring dashboard (plan 31-03) and can be triggered from admin API.
