<action>
Stagehand's hybrid paradigm (DOM/accessibility-tree first with vision fallback) provides self-healing navigation natively (CUA-09). Per the CONTEXT.md decision D-05: "Self-healing navigation should be handled by the provider/model, not custom code."

Add documentation comments to `personal-assistant/src/lib/browser/stagehand-client.ts` in the `createSession` function:

```typescript
/**
 * Create a new ephemeral Browserbase session via Stagehand.
 *
 * Self-healing navigation (CUA-09):
 * Stagehand uses a hybrid paradigm — DOM/accessibility-tree parsing first,
 * with vision-based fallback when DOM is ambiguous. When a site's DOM layout
 * shifts (e.g., a button moves or its class changes), Stagehand:
 * 1. Tries the cached selector from previous successful actions (auto-caching)
 * 2. Falls back to DOM/accessibility-tree search for semantic equivalent
 * 3. Falls back to vision model to visually locate the element
 *
 * This three-tier approach handles DOM shifts without custom self-healing code.
 * Stagehand's enableCaching option stores successful selector paths and replays
 * them on subsequent runs (~2x faster). If replay fails due to layout change,
 * the AI re-engages and finds a new strategy, then updates the cache.
 *
 * Ephemeral sessions (CUA-05):
 * Each session is isolated -- no session data persists between tasks or orgs.
 * Browserbase handles container isolation at the infrastructure level.
 */
```

Also add a comment to the `enableCaching: true` configuration line:

```typescript
enableCaching: true, // CUA-09: Auto-caching for self-healing — replays successful selectors, re-engages AI if DOM shifts
```
