1. Create `deployments/fly/package.json`:
   - name: "@bitbit/fly-worker"
   - scripts: `{ "build": "tsc", "start": "node dist/worker.js", "dev": "tsx src/worker.ts" }`
   - dependencies: `tsx` (dev), `typescript` (dev)
   - No heavy frameworks — plain Node.js HTTP server to minimize image size and cold start

2. Create `deployments/fly/tsconfig.json`:
   - target: ES2022, module: Node16, moduleResolution: Node16
   - outDir: "dist", rootDir: "src"
   - strict: true

3. Create `deployments/fly/src/health.ts`:
   - Export `healthCheck()` returning JSON with status, uptime, timestamp, environment
   - Track boot time via module-level constant
   - Include memory usage from `process.memoryUsage()`

4. Create `deployments/fly/src/worker.ts`:
   - Plain Node.js HTTP server using `node:http` (no Express, no framework overhead)
   - Listen on `PORT` env var (default 3000, matching fly.toml internal_port)
   - Routes:
     - `GET /api/monitoring/health` — calls healthCheck(), returns JSON (matches fly.toml checks path)
     - `POST /api/agent/run` — accepts `{ task_id, agent_type, payload }`, validates required fields
     - All other routes return 404
   - Agent run handler:
     - Validates task_id and agent_type are present
     - Logs task receipt
     - For now: returns `{ accepted: true, task_id }` with 202 status (actual agent execution will be wired when specific agents need browser automation)
     - Updates task status to "processing" in Supabase via REST API (using SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY env vars)
     - Placeholder comment for where Puppeteer/Playwright execution will go
   - Graceful shutdown: listen for SIGTERM, close server, exit cleanly (Fly.io sends SIGTERM per fly.toml kill_signal)

5. Update `deployments/fly/Dockerfile`:
   - Fix: currently exposes port 3001 but fly.toml expects 3000. Align both to 3000.
   - Two-stage build: builder (npm ci, tsc) and runner (copy dist + node_modules)
   - Runner uses `node:22-slim`, creates non-root user
   - CMD: `["node", "dist/worker.js"]`

6. Update `deployments/fly/fly.toml`:
   - Keep existing config (Sydney region, shared-cpu-1x, 1024mb, health checks)
   - Verify health check path matches worker route: `/api/monitoring/health`
   - Ensure internal_port is 3000
