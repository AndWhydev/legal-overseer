1. Create `.github/workflows/deploy.yml`:
   - Trigger: push to main (after CI passes)
   - Three parallel jobs: deploy-vercel, deploy-fly, deploy-cloudflare

   **deploy-vercel job:**
   - Note in comment: Vercel auto-deploys from git integration — this job just verifies the deployment
   - After Vercel deploys (wait ~60s or use Vercel CLI to check), hit the health endpoint
   - `curl -sf https://$VERCEL_URL/api/health` and verify JSON response has `status: "ok"`
   - If no Vercel URL secret configured, skip with warning (Vercel integration handles deploy)

   **deploy-fly job:**
   - uses: superfly/flyctl-actions/setup-flyctl@master
   - Run: `cd deployments/fly && npm ci && npm run build && flyctl deploy --config fly.toml --remote-only`
   - Requires `FLY_API_TOKEN` secret
   - After deploy: `flyctl status --app bitbit-workers`
   - Health check: `curl -sf https://bitbit-workers.fly.dev/api/monitoring/health`

   **deploy-cloudflare job:**
   - Install wrangler: `npm install -g wrangler`
   - Run: `cd deployments/cloudflare && wrangler deploy`
   - Requires `CLOUDFLARE_API_TOKEN` secret
   - After deploy: `curl -sf https://bitbit-edge-cron.<account>.workers.dev/health`

   All jobs should:
   - Continue-on-error: false (deployment failures should be visible)
   - Have reasonable timeout (10 minutes each)
   - Use environment: production

2. Update `personal-assistant/src/lib/cron/cron-guard.ts` to import and use `getServiceClient` from the service-client.ts created in Plan 02, instead of creating its own Supabase client inline. This ensures all service-role access goes through the singleton with pool config.
