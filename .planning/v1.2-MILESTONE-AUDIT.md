### Critical Integration Gaps (from checker)

**INT-01: WhatsApp invoice command bypasses entire invoice pipeline** (Critical)
- `agent-dispatch.ts` line 86: `executeAgentTool('create_task', ...)` — comment confirms: "Using task as proxy until invoice creation tool is wired"
- The full pipeline (resolveInvoiceEntities, detectDuplicateInvoice, generateInvoicePdf, sendInvoiceEmail) only works via scheduler cron, NOT via WhatsApp
- This breaks the core demo scenario: "Andy says 'Invoice Sezer for the White House RE work'"
- Affects: INVC-06, INVC-07, INVC-08, INVC-09, INVC-10, WHATS-02

**INT-02: Lead auto-approve ack has no outbound delivery trigger** (High)
- `autoApproveLeadAcknowledgment` creates approval record with `status: approved` and calls `attemptAckDelivery`
- But no cron or dispatcher picks up queued acks for actual email/WhatsApp delivery
- LEAD-01 "sub-2-minute response" SLA cannot be met without outbound path
- Affects: LEAD-01

**INT-03: Fly.io invoice handler is extraction-only stub** (Medium)
- `deployments/fly/src/agent-executor.ts` lines 173-192: returns `{ status: "extracted" }` only
- No PDF generation, no email send, no lifecycle tracking via Fly.io dispatch
- Invoice pipeline only complete via Vercel cron/scheduler path
- Affects: INVC-08, INVC-09, INVC-10 (via Fly.io path only)

### Additional Findings

- **VPS worker table mismatch**: `deployments/vps/src/worker.ts` reads `agent_runs` while Cloudflare/Fly.io uses `agent_task_queue`. Safe if only `deployments/fly` is deployed (it is).
- **`classifyWithRetry` naming misleading**: Function sets `classification: 'pending'` but doesn't call classifier. Classification happens asynchronously via scheduler cron. Design intent, not a bug.
- **RELAY_SECRET not in env-validation.ts**: Relay POST endpoint silently rejects if RELAY_SECRET unset. Not a production break since channel-sync cron uses pollChannel directly.
