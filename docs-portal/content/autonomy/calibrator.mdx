# Confidence Calibrator

The Confidence Calibrator (`src/lib/intelligence/confidence-calibrator.ts`) runs nightly to adjust confidence routing thresholds based on historical outcome data. It replaces static thresholds with data-driven ones as the system accumulates enough samples.

## How It Works

<Mermaid chart={`flowchart LR
    A[Tool Outcome] --> B[Record in\naction_outcomes]
    B --> C[Nightly\nAnalysis]
    C -->|20+ samples| D[Calculate\nBand Rates]
    D --> E[New\nThresholds]
    E --> F[Confidence\nRouter]
    F --> A
    C -->|insufficient| G[Keep Static\nThresholds]

The calibrator analyzes the `action_outcomes` table, which records every agent action along with whether it was approved and whether it was correct:

```typescript
interface ActionOutcome {
  org_id: string
  agent_type: string
  action_type: string
  confidence_score: number
  was_approved: boolean
  was_correct?: boolean | null
  threshold_source?: string  // 'static' | 'calibrated' | 'seed'
}
```

## Band Analysis

The calibrator divides the confidence range into 5 bands and calculates approval rates for each:

| Band | Range | What It Tells Us |
|------|-------|-----------------|
| 0.50-0.60 | Low confidence | Should these be escalated? |
| 0.60-0.70 | Below default ask | Are we being too cautious? |
| 0.70-0.80 | Around default ask | Boundary behavior |
| 0.80-0.90 | Around default act | Where auto-execution starts |
| 0.90-1.00 | High confidence | Confirmation that high-conf actions succeed |

## Threshold Derivation

The calibrator finds the optimal thresholds:

- **Act threshold**: The lowest confidence band where the approval rate exceeds **95%** (`ACT_APPROVAL_RATE`)
- **Ask threshold**: The lowest band where the approval rate exceeds **70%** (`ASK_APPROVAL_RATE`)

## Safety Rails

<Warning>
Hard safety floors prevent the calibrator from ever making the system too permissive:

- **Act threshold minimum**: 0.70 (never auto-act below this)
- **Ask threshold minimum**: 0.45 (never auto-ask below this)
- **Minimum samples per band**: 20 (no adjustment with insufficient data)
- **Minimum total samples**: 50 (calibrated thresholds not used until this threshold)

These rails ensure that even with skewed training data, the system cannot become dangerously autonomous.
</Warning>

## Graduation Mechanism

The calibrator also handles tool graduation, promoting tools from L2 (Propose) to L3 (Notify):

**Criteria:**
- 20+ approved outcomes in the last 30 days
- 95%+ approval rate
- Only applies to high-impact L2 tools: `send_email`, `send_gmail`, `send_outlook`, `send_sms`

**Process:**
1. Nightly calibration checks per-tool approval rates
2. Qualifying tools get promoted via `organisations.settings.autonomy_overrides`
3. `getAutonomyLevel()` checks org overrides, so promotion takes effect immediately

**Safety cap:** Tools can only graduate to L3 (Act + Notify), never L4 (Silent). External communications always produce a notification.

<Note>
Seeded data (used during development) is excluded from graduation calculations via the `threshold_source` field. Only organic outcomes from real tool executions count toward graduation.
</Note>

## Recording Outcomes

Every agent action records its outcome, regardless of success or failure:

```typescript
await recordActionOutcome(
  supabase, orgId, agentType, actionType,
  confidence, wasApproved, wasCorrect, thresholdSource
)
```

This function never throws. Recording outcomes should not break agent execution.
