# Proactive Intelligence

The Proactive Intelligence layer (`src/lib/proactive/index.ts`) orchestrates an automated pipeline that gathers signals, classifies them, and takes action (or queues suggestions) without waiting for the user to ask.

It runs every 15 minutes via a cron route.

## Pipeline

<Steps>
  <Step title="Load Org Configuration">
    Each organization has configurable proactive settings:
    - `enabled`: master switch (default: true)
    - `maxActionsPerHour`: rate limit (default: 10)
    - `minConfidenceForAutoAction`: threshold for autonomous execution (default: 0.75)
    - `minConfidenceForSuggestion`: threshold for queuing a suggestion (default: 0.45)
  </Step>
  <Step title="Gather Signals">
    The aggregator collects signals from multiple intelligence sources:
    - Channel polling (new emails, messages)
    - Calendar events approaching
    - Task deadlines
    - Entity graph changes
    - Project lifecycle evaluations
  </Step>
  <Step title="Aggregate and Deduplicate">
    Raw signals are aggregated to prevent duplicate processing. The aggregator combines signals about the same entity or topic that arrived from different sources.
  </Step>
  <Step title="Classify Signals">
    The classifier evaluates each aggregated signal and assigns:
    - **Action type**: notify, create_task, send_reminder, escalate
    - **Delivery channel**: in-app, email, SMS
    - **Confidence score**: how certain the system is about the action
    - **Priority**: critical, high, medium, low
  </Step>
  <Step title="Execute or Queue">
    Based on confidence and org configuration:
    - **High confidence** (>= `minConfidenceForAutoAction`): Execute immediately
    - **Medium confidence** (>= `minConfidenceForSuggestion`): Queue as a suggestion in the attention queue
    - **Low confidence**: Log and discard
  </Step>
</Steps>

## Action Types

| Type | Description | Example |
|------|-------------|---------|
| `notify` | Send a notification to the user | "Steve replied to your email" |
| `create_task` | Auto-create a task on the kanban board | "Follow up with Maya (7 days overdue)" |
| `send_reminder` | Trigger a reminder | "Meeting with Ranal in 30 minutes" |
| `escalate` | Flag for immediate human attention | "Invoice overdue by 14 days" |

## Rate Limiting

The `maxActionsPerHour` setting prevents the system from overwhelming the user with notifications. Actions beyond the limit are queued for the next cycle.

<Tip>
Proactive Intelligence is what makes BitBit feel anticipatory rather than reactive. It notices patterns like "this project hasn't been updated in 2 weeks" or "Steve's invoice is overdue" and surfaces them before the user has to remember.
</Tip>

## Project Lifecycle Integration

Each proactive run also calls `evaluateProjectLifecycles()` to check for projects that need attention. See [Project Lifecycle](/autonomy/project-lifecycle) for the specific automation rules.
