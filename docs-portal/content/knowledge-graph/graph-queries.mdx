# Graph Queries API

The graph queries module (`src/lib/knowledge-graph/graph-queries.ts`) provides the core functions for reading and writing to the entity knowledge graph. All functions take a Supabase client and org_id for multi-tenant isolation.

## Read Functions

### getEntityByAlias

Find an entity by name or alias (case-insensitive).

```typescript
async function getEntityByAlias(
  supabase: SupabaseClient,
  orgId: string,
  alias: string
): Promise<EntityNode | null>
```

Searches both the `name` column (via `ilike`) and the `aliases` array (via `cs` contains). Returns the first active match or null.

### getNeighborhood

Get the 1-hop neighborhood of an entity: its edges and connected nodes.

```typescript
async function getNeighborhood(
  supabase: SupabaseClient,
  orgId: string,
  entityId: string,
  opts?: GraphSearchOptions
): Promise<GraphNeighborhood | null>
```

Returns the root node, all edges (incoming and outgoing), and all connected neighbor nodes. Supports filtering by `validAt` timestamp to query the graph at a point in time.

### getEntityEvents

Fetch event tuples for a specific entity within an optional time range.

```typescript
async function getEntityEvents(
  supabase: SupabaseClient,
  orgId: string,
  entityId: string,
  timeRange?: TimeRange
): Promise<EventTuple[]>
```

Returns events ordered by `occurred_at DESC`. Defaults to the last 30 days if no time range is specified.

### vectorSearchEntities

Search entities by embedding similarity using pgvector.

```typescript
async function vectorSearchEntities(
  supabase: SupabaseClient,
  orgId: string,
  queryEmbedding: number[],
  topK?: number
): Promise<EntityNode[]>
```

Uses the `match_entity_nodes` RPC function for cosine similarity (Supabase JS does not support the `<=>` operator directly). Returns the top K matches ordered by distance.

## Write Functions

### findOrCreateEntity

Resolve an entity by name, creating it if it does not exist.

```typescript
async function findOrCreateEntity(
  supabase: SupabaseClient,
  orgId: string,
  name: string,
  entityType: string,
  aliases?: string[]
): Promise<EntityNode>
```

### createEdge

Create a typed relationship between two entities.

```typescript
async function createEdge(
  supabase: SupabaseClient,
  orgId: string,
  sourceId: string,
  targetId: string,
  relationType: string,
  properties?: Record<string, unknown>
): Promise<EntityEdge>
```

### createEventTuple

Record a discrete event (Subject-Verb-Object).

```typescript
async function createEventTuple(
  supabase: SupabaseClient,
  orgId: string,
  subjectId: string,
  verb: string,
  objectText: string,
  objectId?: string
): Promise<EventTuple>
```

<Tip>
The **co-located query** advantage of pgvector means graph traversal and vector similarity can happen in a single SQL statement (~20ms), compared to ~150ms with separate vector DB + graph DB round-trips.
</Tip>
