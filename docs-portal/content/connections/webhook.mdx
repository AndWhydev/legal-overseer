# Webhook Transport

Receive events from external services. Configure the service to POST to your webhook URL, and BitBit handles the rest.

## Endpoint

```
POST /api/connections/{connection_id}/webhook
```

## Signature verification

When you create a webhook connection, BitBit generates a `webhook_secret`. Configure the sending service to sign payloads with this secret using HMAC-SHA256.

BitBit checks for the signature in these headers (in order):
- `x-webhook-signature`
- `x-hub-signature-256`
- `stripe-signature`

If no `webhook_secret` is set on the connection, signature verification is skipped.

## Payload handling

The webhook endpoint accepts any JSON payload. BitBit extracts:
- `id` or `event_id` as the dedup key
- `type` as the event type / subject
- The full payload is stored as the message body

## Setup

1. Create a webhook connection in the dashboard or via API
2. Copy the webhook URL and secret
3. Configure the external service to POST to the URL with HMAC signing
4. BitBit processes incoming events through the intelligence pipeline
