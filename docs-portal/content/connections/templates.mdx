# Connection Templates

Templates give you a quick-start configuration for common connection types. Select a template when creating a custom connection. BitBit pre-configures the expected data shape and transport.

## Available templates

| Template | Transport | Description |
|----------|-----------|-------------|
| **Email Provider** | Bridge | Email services. Expects sender, subject, body fields |
| **CRM** | Bridge | Customer relationship tools. Expects contact and deal data |
| **Task Tracker** | Bridge | Project management tools. Expects task and status data |
| **Webhook Receiver** | Webhook | Event-driven services. Accepts any JSON payload |
| **Custom / Raw** | Bridge | Fully custom. You define the data shape |

## Envelope format by template

All templates use the same base envelope format. The template determines which fields are expected and how data is classified.

### Email Provider

```json
{
  "id": "msg-id",
  "sender": "John Smith",
  "sender_email": "john@example.com",
  "subject": "Meeting tomorrow",
  "body": "Let's meet at 10am to discuss...",
  "received_at": "2026-04-05T10:00:00Z"
}
```

### CRM / Task Tracker / Custom

Same base format. Use `metadata` for domain-specific fields:

```json
{
  "id": "deal-123",
  "sender": "Salesforce",
  "subject": "Deal stage changed",
  "body": "Deal 'Enterprise Contract' moved to Negotiation",
  "metadata": {
    "deal_id": "123",
    "stage": "negotiation",
    "value": 50000
  }
}
```

## List templates via API

```bash
curl https://app.bitbit.chat/api/connections/templates
```
