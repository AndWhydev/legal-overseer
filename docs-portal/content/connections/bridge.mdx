# Bridge Transport

Push data to BitBit from any script. Any language, any platform, any data source. POST JSON to the ingest endpoint.

## When to use bridge

- Email providers that need OAuth (Outlook, custom IMAP)
- Local data sources (macOS Messages, desktop apps)
- Legacy systems with no webhook support
- Custom integrations where you control the data flow

## Authentication

Each bridge connection gets a unique bearer token when created. Include it in the `Authorization` header:

```
Authorization: Bearer {bridge_token}
```

The token is generated automatically when you create a bridge connection. View it in the connection detail drawer in the dashboard.

## Endpoint

```
POST /api/connections/{connection_id}/ingest
```

## Request format

```json
{
  "messages": [
    {
      "id": "unique-external-id",
      "sender": "Display Name",
      "sender_email": "email@example.com",
      "subject": "Optional subject line",
      "body": "Plain text message body",
      "body_html": "<p>Optional HTML body</p>",
      "received_at": "2026-04-05T10:30:00Z",
      "metadata": {
        "custom_field": "any value"
      }
    }
  ]
}
```

### Required fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique identifier for deduplication |
| `body` | string | Plain text message content |

### Optional fields

| Field | Type | Description |
|-------|------|-------------|
| `sender` | string | Sender display name |
| `sender_email` | string | Sender email address |
| `subject` | string | Subject line |
| `body_html` | string | HTML version of the body |
| `received_at` | string | ISO 8601 timestamp (defaults to now) |
| `metadata` | object | Arbitrary key-value data |

## Response

```json
{
  "ok": true,
  "inserted": 5,
  "duplicates": 2
}
```

## Deduplication

BitBit deduplicates messages by `id` and content hash. A duplicate push is silently skipped. Run bridge scripts on a schedule without worrying about duplicates.

## Batch limits

- Maximum 100 messages per request
- No rate limiting on the endpoint (subject to change)

## Example: Python bridge script

```python
import json
import urllib.request

def push_to_bitbit(connection_id, token, messages):
    url = f"https://app.bitbit.chat/api/connections/{connection_id}/ingest"
    payload = json.dumps({"messages": messages}).encode()
    
    req = urllib.request.Request(url, data=payload, headers={
        "Content-Type": "application/json",
        "Authorization": f"Bearer {token}",
    })
    
    with urllib.request.urlopen(req) as resp:
        return json.loads(resp.read())
```
