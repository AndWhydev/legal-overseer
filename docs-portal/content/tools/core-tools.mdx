# Core Tools

Core tools are the foundational operations BitBit uses for task management, memory, contacts, and knowledge. They belong to the `core` and `memory` tool groups and are loaded eagerly (always available).

## Memory Tools

### search_memory

Search the knowledge base using hybrid retrieval (sparse + dense + graph).

<ApiRefTable
  title="search_memory"
  props={[
    { name: "query", type: "string", required: true, description: "What to search for" },
    { name: "category", type: "string", required: false, description: "Filter by memory type" },
  ]}
/>

The query is routed through the [Adaptive Query Router](/intelligence/confidence-routing) which selects the right-sized retrieval pipeline (simple, moderate, or complex) based on query complexity.

### add_memory

Store a new knowledge entry in the Memory Palace.

<ApiRefTable
  title="add_memory"
  props={[
    { name: "content", type: "string", required: true, description: "The memory text to store" },
    { name: "category", type: "string", required: true, description: "Type: preference, pattern, domain, contact, workflow" },
    { name: "confidence", type: "number", required: false, description: "Initial confidence score (0-1)" },
  ]}
/>

After writing, the entity extraction pipeline runs asynchronously to populate the knowledge graph.

### create_procedure

Create a new procedural memory (learned workflow).

<ApiRefTable
  title="create_procedure"
  props={[
    { name: "name", type: "string", required: true, description: 'Procedure name (e.g., "AWU email protocol")' },
    { name: "trigger_pattern", type: "string", required: true, description: "Regex pattern to match against messages" },
    { name: "steps", type: "string[]", required: true, description: "Ordered action sequence" },
  ]}
/>
