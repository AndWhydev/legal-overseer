# Core Tools

Core tools are the foundational operations BitBit uses for task management, memory, contacts, and knowledge. They belong to the `core` and `memory` tool groups and are loaded eagerly (always available).

## Memory Tools

### search_memory

Search the knowledge base using hybrid retrieval (sparse + dense + graph).

| Parameter | Type | Description |
|-----------|------|-------------|
| `query` | string | What to search for |
| `category` | string | Filter by memory type (optional) |

The query is routed through the [Adaptive Query Router](/intelligence/confidence-routing) which selects the right-sized retrieval pipeline (simple, moderate, or complex) based on query complexity.

### add_memory

Store a new knowledge entry in the Memory Palace.

| Parameter | Type | Description |
|-----------|------|-------------|
| `content` | string | The memory text to store |
| `category` | string | Type: preference, pattern, domain, contact, workflow |
| `confidence` | number | Initial confidence score (0-1) |

After writing, the entity extraction pipeline runs asynchronously to populate the knowledge graph.

### create_procedure

Create a new procedural memory (learned workflow).

| Parameter | Type | Description |
|-----------|------|-------------|
| `name` | string | Procedure name (e.g., "AWU email protocol") |
| `trigger_pattern` | string | Regex pattern to match against messages |
| `steps` | string[] | Ordered action sequence |

## Task Tools

### create_task

Create a kanban task with priority, column assignment, and metadata.

| Parameter | Type | Description |
|-----------|------|-------------|
| `title` | string | Task title |
| `description` | string | Detailed description (optional) |
| `priority` | string | critical, high, medium, low |
| `column_id` | string | Target kanban column (optional) |

### update_task

Modify an existing task's status, priority, column, or description.

| Parameter | Type | Description |
|-----------|------|-------------|
| `task_id` | string | Task to update |
| `status` | string | pending, in_progress, completed, archived |
| `priority` | string | New priority (optional) |
| `column_id` | string | Move to column (optional) |

### search_tasks

Full-text search across all tasks in the organization.

| Parameter | Type | Description |
|-----------|------|-------------|
| `query` | string | Search terms |
| `status` | string | Filter by status (optional) |

## Contact Tools

### search_contacts

Entity-resolved contact search across name, email, phone, and aliases.

| Parameter | Type | Description |
|-----------|------|-------------|
| `query` | string | Name, email, phone, or alias to search |

### get_contact

Retrieve a full contact profile including communication patterns and relationship data.

| Parameter | Type | Description |
|-----------|------|-------------|
| `contact_id` | string | Contact ID to retrieve |

<Tip>
All CRUD tools automatically trigger Supabase Realtime broadcasts. When the agent creates a task, the kanban board updates instantly in the browser without polling.
</Tip>
