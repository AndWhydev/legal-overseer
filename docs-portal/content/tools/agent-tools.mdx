# Agent Tools

Agent tools handle multi-agent orchestration, the approval queue, deferred tool resolution, and project/standing order management.

## Orchestration

### spawn_agent

Launch a sub-agent for a complex task that requires its own TAOR loop.

<ApiRefTable
  title="spawn_agent"
  props={[
    { name: "agent_type", type: "string", required: true, description: "The agent type to spawn (e.g., invoice-flow, sentry)" },
    { name: "task", type: "string", required: true, description: "Description of what the sub-agent should accomplish" },
    { name: "context", type: "object", required: false, description: "Additional context to pass to the sub-agent" },
  ]}
/>

The orchestrator supports dependency graphs between tasks. Tasks without dependencies execute in order; circular dependencies are detected and reported as failures.

<Note>
Each spawned agent runs a full TAOR loop with its own model routing, context assembly, and tool execution. The spawn depth is tracked to prevent infinite recursion.
</Note>

### approve_action

Resolve a pending action in the approval queue.

<ApiRefTable
  title="approve_action"
  props={[
    { name: "action_id", type: "string", required: true, description: "The pending action to approve or reject" },
    { name: "approved", type: "boolean", required: true, description: "Whether to approve the action" },
    { name: "feedback", type: "string", required: false, description: "Optional feedback for the agent" },
  ]}
/>

When approved, the action is executed via `executeApprovedAction()`. The outcome is recorded for future calibration.

### resolve_tool

Dynamically resolve a deferred tool schema by name.

<ApiRefTable
  title="resolve_tool"
  props={[
    { name: "tool_name", type: "string", required: true, description: "Name of the deferred tool to load" },
  ]}
/>

This powers the Tool RAG system. When the agent needs a tool that was not eagerly loaded, it can resolve the schema on-demand.

## Standing Orders

Standing orders are recurring instructions that persist across conversations.

### create_standing_order

Create a new standing order — a persistent instruction the agent follows.

<ApiRefTable
  title="create_standing_order"
  props={[
    { name: "instruction", type: "string", required: true, description: "The standing order text" },
    { name: "scope", type: "string", required: false, description: "When it applies (always, specific context)" },
    { name: "priority", type: "string", required: false, description: "How important it is" },
  ]}
/>

<Tip>
The tool system is extensible. Tools are organized into groups (core, memory, channel, web, comms, agentic, ads, seo, tenders, content, builder) and can be eagerly loaded or deferred based on relevance to the current conversation.
</Tip>
