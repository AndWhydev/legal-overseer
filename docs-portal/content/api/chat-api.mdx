# Chat API

<StatusIndicator status="operational" label="API Operational" />

The Chat API (`src/app/api/agent/chat/route.ts`) is the primary endpoint for interacting with BitBit. It accepts user messages and streams back agent responses as Server-Sent Events.

## Endpoint

<CodeBlock code="POST /api/agent/chat" language="bash" title="Chat endpoint" />

## Request

<CodeBlock code={`{
  "message": "What tasks do I have this week?",
  "threadId": "optional-thread-uuid",
  "attachmentIds": ["optional-attachment-uuid"]
}`} language="json" title="Request body" />

<ApiRefTable
  title="Request Parameters"
  props={[
    { name: "message", type: "string", required: true, description: "User message (max 10,000 chars)" },
    { name: "threadId", type: "string", required: false, description: "Conversation thread ID (max 128 chars)" },
    { name: "attachmentIds", type: "string[]", required: false, description: "File attachment IDs (max 10)" },
  ]}
/>

## Authentication

The endpoint supports two authentication methods.

1. **Bearer token** for mobile clients: `Authorization: Bearer <token>`
2. **Cookie-based** for web clients: Supabase session cookie

In development, set `DEV_BYPASS=true` to skip authentication.

## Response (SSE Stream)

The response is a `text/event-stream` with the following events.

<CodeBlock code={`data: {"type":"stage","data":{"stage":"model_routing","status":"start"}}

data: {"type":"tool_call","data":{"name":"search_tasks","input":{...}}}

data: {"type":"tool_result","data":{"name":"search_tasks","output":"..."}}

data: {"type":"message","data":{"text":"Here are your tasks..."}}

data: {"type":"done","data":{"inputTokens":1234,"outputTokens":567}}`} language="json" title="SSE event stream" />

## Security

The endpoint includes several security layers.

- **Input length validation** ensures message, threadId, and attachmentIds are all size-limited.
- **Injection detection** scans messages for prompt injection attempts via `detectInjection()`.
- **Injection neutralization** neutralizes detected injections before processing.
- **Rate limiting** enforces per-user endpoint limits via `checkUserEndpointLimit()`.
- **Timing jitter** via `addTimingJitter()` prevents timing-based side-channel attacks.

<Warning>
The dev bypass mode grants full access without authentication. It should never be enabled in production environments.
</Warning>

## Conversation Pipeline

The endpoint delegates to `UnifiedConversationPipeline` which manages thread state, message history, and the TAOR loop lifecycle. Attachment content is built into multimodal content blocks via `buildAttachmentContentBlocks()`.
