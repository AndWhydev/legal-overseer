# Dashboard APIs

The dashboard API routes (`src/app/api/dashboard/`) provide the data behind BitBit's dashboard views. All endpoints require authentication and are scoped to the user's organization.

## Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/dashboard/stats` | GET | Aggregate statistics (tasks, contacts, messages) |
| `/api/dashboard/weekly-summary` | GET | Weekly activity summary with trends |
| `/api/dashboard/morning-briefing` | GET | AI-generated morning briefing from sleep consolidation |
| `/api/dashboard/activity-feed` | GET | Recent agent and user actions |
| `/api/dashboard/attention-queue` | GET | Items requiring user attention (pending approvals, suggestions) |
| `/api/dashboard/financial-snapshot` | GET | Revenue, invoices, and financial metrics |
| `/api/dashboard/project-timeline` | GET | Active projects with phase progress |
| `/api/dashboard/relationship-health` | GET | Contact engagement scores and communication gaps |

## Stats

Returns aggregate counts and trends.

```json
{
  "tasks": { "total": 42, "completed": 28, "overdue": 3 },
  "contacts": { "total": 15, "active": 8 },
  "messages": { "today": 12, "thisWeek": 67 },
  "agent": { "actionsToday": 5, "approvalRate": 0.94 }
}
```

## Morning Briefing

Returns the briefing generated by [Sleep Consolidation](/memory/sleep-consolidation) Stage 5.

```json
{
  "generatedAt": "2026-04-04T16:00:00Z",
  "upcomingDeadlines": [...],
  "blockedEntities": [...],
  "newDiscoveries": 2,
  "pendingApprovals": 1,
  "lifecycleActions": [...]
}
```

## Attention Queue

Surfaces items that need human decision-making.

- Pending approval queue items (L2 tool actions awaiting confirmation)
- Proactive intelligence suggestions
- Stale project flags from lifecycle evaluation
- Overdue follow-ups

<Tip>
The attention queue is the primary way BitBit surfaces its proactive intelligence to the user. Rather than interrupting with notifications, items accumulate in the queue for the user to review at their convenience.
</Tip>

## Activity Feed

Paginated audit trail of all actions.

```json
{
  "items": [
    {
      "action_type": "task",
      "action": "Created task: Follow up with Steve",
      "reasoning": "Email from Steve mentioned pending invoice",
      "user_confirmed": true,
      "created_at": "2026-04-04T10:30:00Z"
    }
  ],
  "hasMore": true,
  "cursor": "..."
}
```

## Financial Snapshot

Revenue and billing metrics pulled from project data and invoice records.

- Total revenue (this month / this quarter)
- Outstanding invoices
- Overdue amounts
- Revenue by client
