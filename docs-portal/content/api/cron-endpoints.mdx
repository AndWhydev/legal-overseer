## Nightly Pipeline

The nightly jobs run in sequence between 3am and 6am UTC.

<Steps>
  <Step title="Sleep Consolidation (3am UTC)">
    5-stage pipeline: summarize, resolve, discover, prune, brief.
    <CronSchedule expression="0 3 * * *" title="Sleep Consolidation" showNextRuns={3} />
  </Step>
  <Step title="Memory Consolidation (4am UTC)">
    Decay, promote, and archive memories based on access patterns.
    <CronSchedule expression="0 4 * * *" title="Memory Consolidation" />
  </Step>
  <Step title="Confidence Calibration (5am UTC)">
    Adjust confidence routing thresholds from outcome data.
    <CronSchedule expression="0 5 * * *" title="Confidence Calibration" />
  </Step>
  <Step title="Morning Briefing (6am local)">
    Compile actionable intelligence for the day.
    <CronSchedule expression="0 6 * * *" title="Morning Briefing" />
  </Step>
</Steps>

<Note>
Cron jobs are implemented as standard Next.js API route handlers. They are triggered by an external scheduler (Vercel Cron or a cron daemon) that calls each endpoint on schedule. Each job is idempotent and safe to re-run.
</Note>

<Warning>
The sentry cron runs every 5 minutes and monitors configured watches (e.g., website uptime, Google Search Console changes). It should be lightweight to avoid excessive API costs.
</Warning>
