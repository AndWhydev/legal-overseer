# Cron Endpoints

BitBit runs **27+ cron jobs** via Next.js API routes (`src/app/api/cron/`). These handle everything from memory consolidation to proactive intelligence to channel synchronization.

## Full Cron Schedule

| Endpoint | Schedule | Purpose |
|----------|----------|---------|
| `/api/cron/channel-sync` | Every 15 min | Poll all connected channels for new messages |
| `/api/cron/proactive-intelligence` | Every 15 min | Run the full proactive analysis pipeline |
| `/api/cron/proactive-alerts` | Every 15 min | Evaluate and dispatch proactive notifications |
| `/api/cron/triage` | Every 15 min | Classify and route incoming messages |
| `/api/cron/intelligence` | Every 30 min | Run intelligence analytics and pattern detection |
| `/api/cron/process-embeddings` | Every 30 min | Process pending embedding queue |
| `/api/cron/sentiment-drift` | Every 1 hour | Track sentiment changes in contact communications |

| `/api/cron/revenue-intelligence` | Every 6 hours | Analyze revenue patterns and forecasts |
| `/api/cron/relationship-health` | Every 6 hours | Score contact engagement and flag gaps |
| `/api/cron/entity-profile-refresh` | Every 6 hours | Update entity_node properties from latest data |

<Note>
Cron jobs are implemented as standard Next.js API route handlers. They are triggered by an external scheduler (Vercel Cron or a cron daemon) that calls each endpoint on schedule. Each job is idempotent and safe to re-run.
</Note>

## Nightly Pipeline

The nightly jobs run in sequence between 3am and 6am UTC:

1. **Sleep Consolidation** (3am): summarize, resolve, discover, prune, brief
2. **Memory Consolidation** (4am): decay, promote, archive
3. **Confidence Calibration** (5am): adjust thresholds from outcome data
4. **Morning Briefing** (6am): compile actionable intel for the day

## High-Frequency Jobs

The 15-minute cycle handles real-time responsiveness:

- **Channel Sync**: discovers new messages across all connected channels
- **Proactive Intelligence**: evaluates signals and takes action
- **Triage**: classifies and routes incoming messages to the right handler

<Warning>
The sentry cron runs every 5 minutes and monitors configured watches (e.g., website uptime, Google Search Console changes). It should be lightweight to avoid excessive API costs.
</Warning>
