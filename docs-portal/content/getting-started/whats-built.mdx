# What's Built

BitBit shipped five milestones in five weeks. What started as a chat interface with one agent became a fully autonomous operating system with six specialist roles, a knowledge graph, a memory system that thinks while you sleep, and native connections to every channel your business runs on.

This page covers everything that's live and working today.

## The Foundation

### Dashboard

The main interface. Built on Next.js 16 with React 19, deployed to Vercel. Every user gets their own workspace with their own data, their own agents, and their own connections. Multi tenant from day one, with row level security across every table.

The dashboard includes:

- **Chat interface** with syntax highlighted code blocks, voice input, slash commands, follow up suggestions, message editing with conversation forking, and an artifact panel for code and HTML preview
- **Unified activity feed** showing every action every agent takes
- **Role status cards** with autonomy toggles (Observer, Co pilot, Autopilot)
- **Intelligence widgets** showing Revenue Radar, Client Health Score, Cash Flow Prophet, and Capacity Oracle metrics
- **Approval queue** where anything above your confidence threshold lands for review
- **Notification badges** with unread counts across every section

### Design System

Monochrome glassmorphic design language. No decorative colors. Status colors only for semantic meaning. Glass hierarchy where top level surfaces get backdrop blur and inset shadows while children stay flat. Full light and dark theme support via CSS variables. 150 files touched in the overhaul. 10 micro animation keyframes with stagger and lift utilities.

### Database

Supabase PostgreSQL with 120 migrations. Row level security on every table scoped by org_id. pgvector for semantic search over entity relationships. Four PL/pgSQL helper functions for the conversation memory system. Dual tier tenancy where personal orgs auto create and shared team orgs are supported.

## The Agent Engine

### TAOR Loop

The core decision cycle. Think, Act, Observe, Reflect. Every agent request flows through this loop with unbounded iterations. The model decides when it's done, not an arbitrary cap.

The engine replaced a legacy 1,100 line monolith with clean separated modules for pre flight checks, tool execution, and sub agent decomposition. Sub agents run isolated parallel queries for complex multi domain requests.

### Confidence Routing

Three tiers that control how much autonomy each action gets:

- **Above 0.85**: Execute autonomously, report after
- **0.55 to 0.85**: Queue for approval via dashboard or WhatsApp
- **Below 0.55**: Escalate directly to the user

Calibrated against 50 real world scenarios from the first production deployment. Per agent threshold tuning. Adversarial test suite for false positive measurement.

### Tool Orchestration

A fast planner (Haiku) reads user intent and selects only the relevant tool groups. Six groups, 26 tools total. The execution model (Sonnet) receives 5 to 12 filtered tools instead of all 26. This keeps responses fast and costs low, with 90 to 95% KV cache hit rates.

Tools include contact CRUD, task management, invoice operations, email send/find/read, web search, calendar, and memory recall.

### Action Dispatcher

Seven transport handlers for executing agent decisions: email, SMS, WhatsApp, task creation, invoice creation, invoice send, and reminders. Every action gets an idempotency guard to prevent duplicates, with retry and exponential backoff on failure.

Conversational approval works naturally. When BitBit asks "Should I send this?", you can reply "yep, send it" and the approve_action tool fires.

### Bridge Infrastructure

For WhatsApp, Android Messages, and iMessage, BitBit runs per user bridge instances that connect to your actual accounts. Not API wrappers. Your real conversations, your real identity.

**WhatsApp and Android Messages** use mautrix bridges running on Fly.io Machines with a shared Conduit Matrix homeserver. Link by scanning a QR code. Cost: $1.90 per month per user.

**iMessage** uses BlueBubbles running on a LightNode Mac VPS with macOS Sequoia. Link by signing into your Apple ID through an embedded VNC session locked down to look like a native widget. A warm pool of pre provisioned instances means provisioning takes under 5 seconds. Cost: $7.70 per month per user.

### Provider Registry

Runtime extensible plugin system. Every connection implements the same ProviderPlugin interface with pull, send, webhookParse, and healthCheck methods. Adding a new channel means writing one file. The Envelope format normalizes every message from every provider before it enters the pipeline.

### Webhook Infrastructure

Signature verification on all incoming webhooks. Token based authentication for providers that do not support HMAC signing. Connection sync logging. Deduplication via content hashing.

## The Intelligence Layer

### Knowledge Graph

Entity graph built on pgvector with in process Kuzu WASM for relationship queries. Entities include people, companies, projects, invoices, and any concept BitBit extracts from your data.

Fact extraction fires per conversation turn via Haiku. Entity mention scanning with word boundary matching. The context baseplate injects entity profiles into system prompts so BitBit always knows who Dave is, what he owes you, and which project he mentioned last Tuesday.

### Memory Palace

Six layer Cognitive Memory OS:

1. **Working memory**: Last 10 conversation turns, verbatim
2. **Short term memory**: Turns 11 to 30, compressed via Haiku
3. **Long term memory**: Turns 31 and beyond, key facts only (commitments, decisions, deadlines, amounts)
4. **Procedural memory**: Learned patterns and preferences
5. **Semantic memory**: Embedded knowledge chunks via dual embeddings (Voyage 3.5 primary, Google multimodal fallback)
6. **Episodic memory**: Historical interaction patterns

### Sleep Consolidation

Background process that runs during low activity periods. Compresses conversation memory, extracts durable facts, strengthens important connections in the knowledge graph, and archives stale threads. Like your brain consolidating memories while you sleep, except BitBit does it with a cron job.

### Context Assembly

Six tier token budgeting within an 8K token limit. Every agent request assembles context from entity profiles, active threads, recent memory, procedural knowledge, and proactive signals. The pipeline prioritizes what matters most for the current request.

### Proactive Intelligence

BitBit does its thinking when new information arrives, not when you ask. Four always on intelligence modules:

- **Revenue Radar**: Identifies upsell opportunities by analyzing customer spend patterns
- **Client Health Score**: 0 to 100 rating per relationship using real engagement signals
- **Cash Flow Prophet**: Forward financial projections with confidence intervals
- **Capacity Oracle**: Workload modeling that warns before you overcommit

All computed from real data with minimum thresholds so predictions are never shown when the data is insufficient.

## Autonomous Roles

Six specialist roles, each owning a domain. Not task executing agents. Domain owning autonomous operators.

### Primary Roles

**Finance**: Invoice management, collections, cash flow tracking, payment timing learning. Knows which clients pay late, which invoices are overdue, and when to nudge.

**Comms**: Message triage across all channels. Classifies, prioritizes, routes. Drafts responses that match the tone of each conversation. Manages follow up automation and relationship health.

**Sales**: Lead qualification, pipeline management, proposal generation from meeting transcripts, client onboarding automation including Asana project creation and kickoff call booking. Nurture sequences that actually run.

### Growth Roles

**Ad Script Generator**: Platform adapted ad copy for Meta, Google, and LinkedIn. Daily budget caps and cost controls. Token spend tracking.

**SEO Monitor and Tender Hunter**: Dual purpose role. SEO audits, content recommendations, and schema validation on one side. Tender discovery, eligibility checking (ABN, insurance, case studies), and fast response generation on the other. The tender hunting capability is the premium play at $1,000 to $2,000 per month.

**Content Creator**: Blog posts, social content, content calendar planning. Uses Claude Sonnet for cost effective generation with scheduling to connected platforms.

### Builder Role

Website generation through natural conversation. Template library with variable syntax. WordPress and Elementor integration. Staging preview with scaled iframe rendering. Deploy from chat. Budget capped at $2 per day in co pilot mode.

### Autonomy Control

Every role runs at one of three levels:

- **Observer**: Watch only. Generates insights but takes no action.
- **Co pilot**: Drafts and proposes. User approves before execution.
- **Autopilot**: Acts and reports. Confidence gated with automatic escalation on uncertainty.

Tools auto graduate from Co pilot to Autopilot based on approval history. If you approve the same type of action 10 times in a row, BitBit learns it can handle that category on its own.

## Conversation Memory

### Unified Pipeline

Web chat, WhatsApp, SMS, email, and Slack all route through the same conversation pipeline. One active thread per user per org. Cross channel identity resolution ties phone numbers, email addresses, and Slack IDs to authenticated users.

### Persistent Threads

Every conversation is a persistent thread. Switch from WhatsApp to the dashboard and pick up where you left off. The three tier compression (verbatim, compressed, key facts) means even very long conversations stay within token budgets.

Thread archival runs every 15 minutes for threads inactive longer than 24 hours.
