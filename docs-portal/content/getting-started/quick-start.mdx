# Quick Start

This guide walks you through setting up BitBit for local development.

## Prerequisites

- **Node.js 20+** and **pnpm** (or npm/yarn)
- **Supabase CLI** for local database
- API keys for Anthropic (Claude), Voyage AI, and optionally Google Embeddings

## Setup

<Steps>
  <Step title="Clone and install">
    ```bash
    git clone https://github.com/your-org/bitbit.git
    cd bitbit/personal-assistant
    pnpm install
    ```
  </Step>
  <Step title="Configure environment">
    Copy the example env file and fill in your API keys:

    ```bash
    cp .env.example .env.local
    ```

    Required variables:

    ```
    ANTHROPIC_API_KEY=sk-ant-...
    VOYAGE_API_KEY=pa-...
    NEXT_PUBLIC_SUPABASE_URL=http://localhost:54321
    NEXT_PUBLIC_SUPABASE_ANON_KEY=...
    SUPABASE_SERVICE_ROLE_KEY=...
    ```

    <Note>
    Optional: Set `GOOGLE_API_KEY` for dual-embedding support (768d multimodal). Without it, the system falls back to Voyage-only embeddings.
    </Note>
  </Step>
  <Step title="Start Supabase locally">
    ```bash
    supabase start
    supabase db reset  # applies all 80+ migrations
    ```

    This creates all tables including `entity_nodes`, `entity_edges`, `event_tuples`, `memory_palace_entries`, and the full application schema.
  </Step>
  <Step title="Start the dev server">
    ```bash
    pnpm dev
    ```

    The app runs at `http://localhost:3000`. The chat endpoint is at `/api/agent/chat`.
  </Step>
  <Step title="Verify it works">
    Open the dashboard and send a test message in the chat panel. You should see SSE events streaming back: `thinking`, `tool_call`, `tool_result`, `message`, and `done`.
  </Step>
</Steps>

## Project Structure

```
personal-assistant/
├── src/app/          # Next.js App Router (pages, API routes, cron)
├── src/lib/          # Core business logic (65+ modules)
│   ├── agent/        # TAOR loop, tools, model routing
│   ├── memory-palace/# Episodic + semantic memory
│   ├── knowledge-graph/ # Entity graph (pgvector)
│   ├── context-assembly/# 4-tier context pipeline
│   ├── rag/          # Hybrid retrieval
│   ├── proactive/    # Signal → Decision → Action
│   └── channels/     # Gmail, Outlook, WhatsApp, etc.
├── src/components/   # 31 domain-specific React component dirs
├── supabase/         # 80+ migrations, seed data
└── .planning/        # Architecture and planning docs
```

<Tip>
Components are organized by **feature domain** (e.g., `components/kanban/`, `components/chat/`) rather than atomic design. This keeps related code co-located.
</Tip>

## Dev Bypass

During development, the chat API supports a dev bypass mode that skips authentication. Set `DEV_BYPASS=true` in your `.env.local` to use a hardcoded user and org ID. This is useful for testing the agent loop without setting up Supabase Auth.

<Warning>
Never enable dev bypass in production. It grants full access without authentication.
</Warning>
